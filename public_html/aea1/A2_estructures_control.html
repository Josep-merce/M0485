1. Estructures de selecció: if / else

Les estructures de selecció permeten prendre decisions sobre quin conjunt d'instruccions cal executar en un punt del programa. Es basen sempre en l'avaluació d'una expressió booleana que retorna true o false.

EstructuraQuan usar-laCamins d'execució
ifExecutar un bloc només si es compleix una condició1 camí opcional
if / elseDos escenaris excloents: o un o l'altre2 camins obligatoris
if / else if / elseMúltiples rangs o casos (nota, temperatura...)N camins, s'executa el primer que compleix
if niatsCondicions que depenen d'altres condicions prèviesJeràrquic

BàsicAplicar un descompte si la compra supera un mínim

2. Estructures de selecció: switch

La sentència switch avalua una expressió de tipus enter, caràcter o text i executa el bloc del case que coincideixi. És més llegible que una cadena llarga de if / else if quan es compara sempre la mateixa variable amb valors concrets.

2.1 Switch tradicional (totes les versions de Java)
BàsicNom del dia de la setmana
public class DiaSemana { public static void main(String[] args) { int dia = 3; switch (dia) { case 1: System.out.println("Dilluns"); break; case 2: System.out.println("Dimarts"); break; case 3: System.out.println("Dimecres"); break; case 4: System.out.println("Dijous"); break; case 5: System.out.println("Divendres"); break; case 6: System.out.println("Dissabte"); break; case 7: System.out.println("Diumenge"); break; default: System.out.println("Dia invàlid"); } } }
MitjàCalculadora simple amb switch (char)
public class Calculadora { public static void main(String[] args) { double a = 10.0; double b = 3.0; char op = '/'; switch (op) { case '+': System.out.println("Suma: " + (a + b)); break; case '-': System.out.println("Resta: " + (a - b)); break; case '*': System.out.println("Producte: " + (a * b)); break; case '/': if (b != 0) { System.out.println("Divisió: " + (a / b)); } else { System.out.println("Error: divisió per zero"); } break; default: System.out.println("Operador desconegut: " + op); } } }
⚠️ Fall-through: si oblides el break en un case, Java continua executant el codi del case següent sense comprovar la condició. Sempre posa break al final de cada cas.
2.2 Switch expression (Java 14+) — sintaxi moderna amb ->
MitjàEstació de l'any amb switch expression
public class EstacioAny { public static void main(String[] args) { int mes = 7; String estacio = switch (mes) { case 12, 1, 2 -> "Hivern"; case 3, 4, 5 -> "Primavera"; case 6, 7, 8 -> "Estiu"; case 9, 10, 11 -> "Tardor"; default -> "Mes invàlid"; }; System.out.println("Mes " + mes + ": " + estacio); } }

3. Estructures de repetició: while i do-while

Els bucles permeten repetir un bloc de codi mentre es compleixi una condició. El while comprova la condició abans de cada iteració. El do-while la comprova després: el cos s'executa almenys una vegada.

Parts del whileDescripció
1. InicialitzacióEstablir les variables de control abans del bucle
2. CondicióExpressió booleana avaluada abans de cada iteració
3. Cos del bucleInstruccions que s'executen quan la condició és true
4. ActualitzacióModificar la variable de control per evitar un bucle infinit
3.1 Esquema de recorregut: processar tots els elements
BàsicCompte enrere fins a zero
public class CompteEnrere { public static void main(String[] args) { int comptador = 5; // 1. Inicialització while (comptador > 0) { // 2. Condició System.out.println("Compte enrere: " + comptador); // 3. Cos comptador--; // 4. Actualització } System.out.println("Enlairament!"); } }
MitjàAcumular la suma amb while — 3 variants (for, while, do-while)
public class SumaAcumulativa { public static void main(String[] args) { // Variant for: quan sabem exactament quantes iteracions int sumaFor = 0; for (int i = 1; i <= 5; i++) { sumaFor += i; } System.out.println("Suma amb for: " + sumaFor); // 15 // Variant while: quan la condició és externa int sumaWhile = 0; int i = 1; while (i <= 5) { sumaWhile += i; i++; } System.out.println("Suma amb while: " + sumaWhile); // 15 // Variant do-while: el cos s'executa almenys una vegada int sumaDoWhile = 0; int j = 1; do { sumaDoWhile += j; j++; } while (j <= 5); System.out.println("Suma amb do-while: " + sumaDoWhile); // 15 } }
3.2 Esquema de cerca: buscar i aturar quan es troba
MitjàBuscar el primer número negatiu d'una seqüència
public class CercaNegatiu { public static void main(String[] args) { double[] notes = {6.5, 8.0, 4.5, 9.0, -1}; // -1 = sentinella int idx = 0; boolean trobat = false; while (notes[idx] != -1 && !trobat) { if (notes[idx] < 5.0) { trobat = true; // trobem suspès → parem } idx++; } System.out.println("Hi ha alguna nota suspesa: " + trobat); } }
3.3 do-while: el cos s'executa sempre almenys una vegada
MitjàValidació de PIN amb 3 intents màxims
public class ValidacioPin { public static void main(String[] args) { final int PIN_CORRECTE = 1234; final int MAX_INTENTS = 3; int intents = 0; boolean accedit = false; do { int pin = 1234; // simulem entrada (RA5 llegirà del teclat) intents++; if (pin == PIN_CORRECTE) { accedit = true; } else { System.out.println("PIN incorrecte. Intents: " + intents); } } while (!accedit && intents < MAX_INTENTS); if (accedit) { System.out.println("Accés concedit."); } else { System.out.println("Compte bloquejat."); } } }
while vs do-while: usa while quan és possible que no calgui executar el cos ni una vegada. Usa do-while quan el cos sempre s'ha d'executar almenys una vegada (ex: validar una dada fins que sigui correcta).

4. Estructures de repetició: for

El for s'usa quan sabem exactament quantes iteracions cal fer. Integra en una sola línia la inicialització del comptador, la condició i l'actualització.

Sintaxi:
for (inicialització ; condició ; actualització) { cos }

La inicialització s'executa una sola vegada al principi. La condició es comprova abans de cada iteració. L'actualització s'executa automàticament al final de cada iteració.
BàsicImprimir números parells i senars del 1 al 20
public class ParellsSenars { public static void main(String[] args) { // Números parells del 1 al 20 (pas de 2 en 2) System.out.print("Parells: "); for (int i = 2; i <= 20; i += 2) { System.out.print(i + " "); } System.out.println(); // Números senars del 1 al 20 System.out.print("Senars: "); for (int i = 1; i <= 20; i += 2) { System.out.print(i + " "); } System.out.println(); } }
MitjàTaula de multiplicar del 7
public class TaulaMultiplicar { public static void main(String[] args) { int taula = 7; for (int i = 1; i <= 10; i++) { System.out.println(taula + " x " + i + " = " + (taula * i)); } } }

5. Bucles imbricats

La imbricació consisteix a incloure un bucle dins d'un altre. El bucle intern s'executa completament per a cada iteració del bucle extern.

MitjàTriangle d'asteriscos (bucles dependents)
public class TriangleAsteriscos { public static void main(String[] args) { int n = 5; for (int i = 1; i <= n; i++) { for (int j = 1; j <= i; j++) { System.out.print("* "); } System.out.println(); } // Sortida: * // * * // * * * // * * * * // * * * * * } }
AvançatRectangle ple i buit d'asteriscos
public class Rectangle { public static void main(String[] args) { int base = 10; int altura = 5; // Rectangle ple for (int i = 0; i < altura; i++) { for (int j = 0; j < base; j++) { System.out.print("* "); } System.out.println(); } System.out.println(); // Rectangle buit (només el contorn) for (int i = 0; i < altura; i++) { for (int j = 0; j < base; j++) { if (i == 0 || i == altura - 1 || j == 0 || j == base - 1) { System.out.print("* "); } else { System.out.print(" "); } } System.out.println(); } } }

6. Sentències de salt: break i continue

break atura el bucle completament. continue salta a la iteració següent sense executar la resta del cos.

SentènciaEfecteQuan usar-la
breakSurt immediatament del bucle (o del switch)Quan has trobat el que buscaves
continueSalta el reste del cos i passa a la iteració següentQuan vols saltar un element concret
Bàsicbreak i continue — comparació directa
public class BreakContinue { public static void main(String[] args) { // break: surt del bucle quan i == 5 for (int i = 1; i <= 10; i++) { if (i == 5) break; System.out.print(i + " "); // 1 2 3 4 } System.out.println("← break"); // continue: salta els parells for (int i = 1; i <= 10; i++) { if (i % 2 == 0) continue; System.out.print(i + " "); // 1 3 5 7 9 } System.out.println("← continue"); // for(;;) — bucle infinit controlat per break int n = 0; for (;;) { if (n == 5) break; System.out.print(n + " "); // 0 1 2 3 4 n++; } System.out.println("← for(;;)"); } }

8. Control d'excepcions

Una excepció és un error en temps d'execució: divisió per zero, conversió de text invàlid, accés a una posició inexistent... Sense gestió, el programa s'atura. Java permet capturar-les amb try-catch-finally.

BlocQuan s'executa
try { }Sempre. Conté el codi que podria llançar una excepció
catch (Tipus e) { }Només si s'ha produït l'excepció del tipus indicat
finally { }Sempre, tant si hi ha excepció com si no (opcional)
BàsicCapturar una conversió de text invàlida
public class ParseSegur { public static void main(String[] args) { String text = "vint-i-dos"; try { int edat = Integer.parseInt(text); System.out.println("Edat: " + edat); } catch (NumberFormatException e) { System.out.println("Error: '" + text + "' no és un número."); } finally { System.out.println("[finally] Sempre s'executa."); } System.out.println("El programa continua..."); } }
MitjàMúltiples catch i creació d'excepció pròpia
class NotaInvalidaException extends Exception { public NotaInvalidaException(String msg) { super(msg); } } public class ValidadorNota { public static void validar(double nota) throws NotaInvalidaException { if (nota < 0 || nota > 10) { throw new NotaInvalidaException("Nota fora de rang: " + nota); } System.out.println("Nota vàlida: " + nota); } public static void main(String[] args) { try { validar(8.5); validar(11.0); // llança NotaInvalidaException } catch (NotaInvalidaException e) { System.out.println("Excepció: " + e.getMessage()); } } }
Excepció freqüentCausa habitual
ArithmeticExceptionDivisió per zero
NumberFormatExceptionInteger.parseInt("abc")
ArrayIndexOutOfBoundsExceptionAccés a índex inexistent
NullPointerExceptionUsar un objecte que val null

9. Assercions

Una asserció verifica que una condició que hauria de ser sempre certa realment ho és durant el desenvolupament. Si és false, el programa llança un AssertionError.

⚠️ Les assercions estan desactivades per defecte. Cal activar-les: java -ea NomClasse o a NetBeans: Project Properties → Run → VM Options → -ea.
MitjàAssercions com a precondicions i postcondicions
public class AssercioMitjana { public static double calcularMitjana(double suma, int count) { // Precondició: no podem dividir per zero assert count > 0 : "count ha de ser > 0, però val: " + count; double mitjana = suma / count; // Postcondició: la mitjana ha de ser un valor vàlid de nota assert mitjana >= 0 && mitjana <= 10 : "Mitjana fora de rang: " + mitjana; return mitjana; } public static void main(String[] args) { double resultat = calcularMitjana(46.5, 6); System.out.println("Mitjana: " + resultat); } }
💡 Assercions vs excepcions: les assercions són per a errors de programació (condicions que mai haurien de passar si el codi és correcte). Les excepcions són per a errors en temps d'execució causats per l'usuari o l'entorn. Mai uses assercions per validar l'entrada de l'usuari.

10. Documentació del codi

Comentar i documentar el codi és una part essencial del desenvolupament professional. No es tracta d'explicar el que fa cada línia (el codi ja ho mostra), sinó el per què de les decisions preses: per a que serveix un bucle, quina precondició té un bloc try-catch, o quin comportament es valida amb una asserció.

Bones pràctiques de documentació en estructures de control

EstructuraQuè cal documentarExemple de bon comentari
if / elseLa condició de negoci que representa// Apliquem el descompte soci si ha comprat més de 100 €
while / forInvariant del bucle i condició de sortida// Recorrem fins trobar el primer suspès o arribar al sentinella -1
try-catchQuina operació pot fallar i per quin motiu// L'usuari pot introduir text no numèric → capturem NumberFormatException
assertQuina precondició o postcondició es verifica// Precondició: count > 0 per evitar divisió per zero
MitjàCodi ben documentat: bucle de cerca amb excepció i asserció
public class CercaDocumentada { public static void main(String[] args) { // Seqüència de notes (finalitza amb el sentinella -1) double[] notes = {6.5, 8.0, 4.5, 9.0, -1}; // Recorrem totes les notes fins al sentinella buscant la primera suspesa int idx = 0; boolean trobat = false; while (notes[idx] != -1 && !trobat) { // Una nota és suspesa si és estrictament menor que 5 if (notes[idx] < 5.0) { trobat = true; } idx++; } // Postcondició: idx és vàlid (no ha sobrepassat els límits de l'array) assert idx <= notes.length : "idx fora de rang: " + idx; System.out.println("Hi ha alguna nota suspesa: " + trobat); // Conversió segura: si el text no és numèric, informem sense aturar el programa String entrada = "7.5"; try { double nota = Double.parseDouble(entrada); System.out.println("Nota llegida: " + nota); } catch (NumberFormatException e) { // L'entrada no és un número vàlid → mostrem l'error i continuem System.out.println("Error: '" + entrada + "' no és una nota vàlida."); } } }
💡 Regla pràctica: un bon comentari respon a "per què s'ha pres aquesta decisió?", no a "què fa aquesta línia?". Si el codi és prou clar, no cal comentar-lo; si la lògica no és òbvia (condicions complexes, bucles amb sentinella, try-catch específics), un comentari curt val molt.

11. Exercicis pràctics

Exercicis individuals ordenats de menor a major dificultat. Llegeix l'enunciat, dissenya l'algorisme en paper abans de codificar, i comprova el resultat amb diversos valors de prova. Prem el botó per veure la solució quan hagis acabat.

📌 En els exercicis que usen Scanner per llegir del teclat, pots substituir-lo per ara amb valors declarats directament al codi. Quan arribi l'RA5 podràs connectar-los a la lectura real del teclat.
Ex. 1 Parell o senar amb operació Bàsic

Llegim un número enter. Si és parell el multiplica per 2 i li suma 3. Si és senar el multiplica per 3 i li resta 4. Informa de l'operació feta i el resultat.

  • Declara int numero = 7; (prova també amb un parell).
  • Usa if / else i l'operador mòdul % per determinar parell o senar.
  • Mostra quina operació s'ha fet i el resultat final.
Bàsic CA: 3.1 · 3.5
✅ Solució
public class ParellSenar { public static void main(String[] args) { int numero = 7; if (numero % 2 == 0) { int resultat = numero * 2 + 3; System.out.println("Parell → " + numero + " * 2 + 3 = " + resultat); } else { int resultat = numero * 3 - 4; System.out.println("Senar → " + numero + " * 3 - 4 = " + resultat); } } }
Ex. 2 Taula de multiplicar amb for Bàsic

Genera les taules de multiplicar del 2 al 9 usant bucles imbricats.

  • Bucle extern: recorre les taules del 2 al 9.
  • Bucle intern: recorre els multiplicadors de l'1 al 10.
  • Format de sortida: 7 x 3 = 21
  • Separa cada taula amb una línia en blanc.
Bàsic CA: 3.2 · 3.5
✅ Solució
public class TaulesMultiplicar { public static void main(String[] args) { for (int taula = 2; taula <= 9; taula++) { System.out.println("── Taula del " + taula + " ──"); for (int i = 1; i <= 10; i++) { System.out.println(taula + " x " + i + " = " + (taula * i)); } System.out.println(); } } }
Ex. 3 Sistema de qualificació de notes Bàsic

Donada una nota entre 0 i 10, mostra la qualificació equivalent: A (9–10), B (8–8.9), C (7–7.9), D (6–6.9), F (0–5.9). Si la nota és fora de rang, mostra un missatge d'error.

  • Declara double nota = 8.5;
  • Usa if / else if / else per classificar.
  • Prova amb valors com 10, 5, -1 i 11.
Bàsic CA: 3.1 · 3.5
✅ Solució
public class SistemaQualificacions { public static void main(String[] args) { double nota = 8.5; String qualificacio; if (nota >= 9 && nota <= 10) { qualificacio = "A - Excel·lent"; } else if (nota >= 8 && nota < 9) { qualificacio = "B - Notable alt"; } else if (nota >= 7 && nota < 8) { qualificacio = "C - Notable"; } else if (nota >= 6 && nota < 7) { qualificacio = "D - Aprovat"; } else if (nota >= 0 && nota < 6) { qualificacio = "F - Suspès"; } else { qualificacio = "Nota incorrecta (fora de rang)"; } System.out.println(nota + " → " + qualificacio); } }
Ex. 4 Estació de l'any amb switch expression Mitjà

Donat un número de mes (1–12), mostra l'estació de l'any corresponent usant la nova sintaxi switch amb ->.

  • Declara int mes = 7;
  • Usa switch expression amb múltiples valors per case.
  • Inclou un default per valors fora de rang.
  • Ampliació: afegeix una variable booleana esEstiu que sigui true si el mes és juny, juliol o agost.
Mitjà CA: 3.1 · 3.5
✅ Solució
public class EstacioAny { public static void main(String[] args) { int mes = 7; String estacio = switch (mes) { case 12, 1, 2 -> "Hivern"; case 3, 4, 5 -> "Primavera"; case 6, 7, 8 -> "Estiu"; case 9, 10, 11 -> "Tardor"; default -> "Mes invàlid"; }; boolean esEstiu = (mes >= 6 && mes <= 8); System.out.println("Mes " + mes + ": " + estacio); System.out.println("És estiu: " + esEstiu); } }
Ex. 5 Recorregut i cerca: màxim, mínim i primer múltiple de 5 Mitjà

Donada una seqüència de números enters positius acabada en -1, fes dos programes separats:

  • Recorregut: llegeix tots els valors i mostra el màxim i el mínim. Usa Integer.MIN_VALUE i Integer.MAX_VALUE per inicialitzar.
  • Cerca: para quan trobis el primer múltiple de 5. Usa una variable booleana trobat. Si no n'hi ha cap, informa-ho.
  • Seqüència de prova: {12, 4, 37, 8, 25, 3, 50, -1}
Mitjà CA: 3.2 · 3.3 · 3.5
✅ Solució
public class RecorregutCerca { public static void main(String[] args) { int[] nums = {12, 4, 37, 8, 25, 3, 50, -1}; // ── RECORREGUT: màxim i mínim ── int max = Integer.MIN_VALUE; int min = Integer.MAX_VALUE; int i = 0; while (nums[i] != -1) { if (nums[i] > max) max = nums[i]; if (nums[i] < min) min = nums[i]; i++; } System.out.println("Màxim: " + max + " · Mínim: " + min); // ── CERCA: primer múltiple de 5 ── int j = 0; boolean trobat = false; while (nums[j] != -1 && !trobat) { if (nums[j] % 5 == 0) { System.out.println("Primer múltiple de 5: " + nums[j]); trobat = true; } j++; } if (!trobat) System.out.println("No hi ha cap múltiple de 5."); } }
Ex. 6 Reserva d'hotel — if/else niats i constants Mitjà

Un hotel té dues tarifes: sense vistes al mar (150,50 €/nit) i amb vistes al mar (190,50 €/nit). Calcula el cost total d'una reserva validant que els dies siguin positius.

  • Declara les tarifes com a constants final.
  • Valida que dies > 0; si no, mostra un missatge d'error.
  • Usa if / else per aplicar la tarifa correcta.
  • Mostra el resum complet: client, dies, vistes al mar, cost total.

El programa genera un número secret entre 1 i 50. El jugador té 5 intents per endevinar-lo. Després de cada intent, el programa indica si el número secret és major o menor.

  • Simula els intents amb un array de valors: int[] intents = {25, 40, 35, 38, 37};

import java.util.Random; public class JocEndevinar { public static void main(String[] args) { Random random = new Random(); int secret = random.nextInt(50) + 1; final int MAX_INTENTS = 5; int[] proves = {25, 40, 35, 38, 37}; // simulem intents int intents = 0; boolean encertat = false; while (intents < MAX_INTENTS && !encertat) { int intent = proves[intents]; intents++; if (intent == secret) { encertat = true; } else if (intent < secret) { System.out.println(intent + " → massa baix"); } else { System.out.println(intent + " → massa alt"); } } if (encertat) { System.out.println("Encertat en " + intents + " intents!"); } else { System.out.println("Has esgotat els intents.
