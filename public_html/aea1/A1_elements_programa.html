1. Comentaris

Els comentaris són fragments de text que Java ignora completament durant l'execució. Serveixen per explicar el codi als humans: companys, professors o el teu futur jo. Un bon comentari explica el per què, no el que fa el codi (que ja es llegeix directament).

TipusSintaxiQuan usar-lo
Una sola línia// text fins al final de la líniaNotes curtes, explicar una instrucció concreta
Múltiples línies/* text que pot ocupar diverses línies */Explicar un bloc de codi o desactivar codi temporalment
Javadoc/** text */ just abans d'una classe o mètodeDocumentació automàtica amb l'eina javadoc
BàsicComentari de línia i inline
// Programa que saluda l'usuari per pantalla public class Salutacio { public static void main(String[] args) { System.out.println("Hola, món!"); // mostra un missatge per pantalla } }
MitjàComentari de bloc per explicar un càlcul
public class CalcularPreu { public static void main(String[] args) { /* Calculem el preu final d'un article aplicant el descompte del 10% i afegint el 21% d'IVA al preu resultant */ double preuBase = 49.99; double descompte = preuBase * 0.10; double preuFinal = (preuBase - descompte) * 1.21; System.out.println(preuFinal); } }
AvançatComentari Javadoc documentant una classe i un mètode
/** * Gestiona la informació bàsica d'un alumne. * @author Joan Puig * @version 1.0 */ public class Alumne { /** * Calcula la nota mitjana de l'alumne. * @param nota1 primera nota (0-10) * @param nota2 segona nota (0-10) * @return la mitjana aritmètica de les dues notes */ public static double calcularMitjana(double nota1, double nota2) { return (nota1 + nota2) / 2; } }
💡 Bon vs mal comentari: // sumem 32 → mal comentari (diu el que és obvi). // convertim Celsius a Fahrenheit → bon comentari (explica el perquè). A NetBeans, Ctrl+/ comenta o descomenta la línia seleccionada ràpidament.

2. Tipus de dades

Java és un llenguatge fortament tipat: cada variable ha de tenir un tipus declarat que determina quins valors pot emmagatzemar. Hi ha dos grans grups: primitius (valors simples guardats directament en memòria) i de referència (apunten a un objecte en el heap).

int32 bits−2.147.483.648 a 2.147.483.647int edat = 25;
long64 bits±9,2 × 10¹⁸long poblacio = 8_000_000_000L;
float32 bits~7 decimals significatiusfloat alçada = 1.75f;
double64 bits~15 decimals significatiusdouble pi = 3.14159265;

Tipus de referència principals:
• String → cadena de text. Sembla un primitiu però és un objecte: String nom = "Anna";
• Arrays, classes i interfícies → els veurem en profunditat als RA4 i RA6.
Classes envoltants (Wrapper classes)

Per a cada tipus primitiu existeix una classe envoltant que el tracta com a objecte i ofereix mètodes addicionals. Les usarem sobretot per convertir tipus.

3. Literals

Un literal és el valor fix escrit directament al codi font. Quan escrivim 42, 3.14, true o "Hola" al codi, aquells valors són literals. A diferència d'una variable (que pot canviar), un literal és sempre el mateix valor fix.

int Literals enters Per defecte, qualsevol nombre enter és int. Si supera el rang cal afegir L al final per indicar long. Es pot usar _ per llegibilitat.
Exemples: 42 · -7 · 1_000_000 · 9L
dbl Literals decimals Per defecte és double. Per a float cal la f al final. S'accepta notació científica amb e.
Exemples: 3.14 · 9.99f · 1.5e10
chr Literals de caràcter Sempre entre cometes simples. Exactament un sol caràcter. Java el guarda com un nombre Unicode internament.
Exemples: 'A' · '7' · '@' · ' '
bool Literals booleants Només existeixen dos literals: true i false. Sempre en minúscules — True o TRUE donaran error.
Exemples: true · false
str Literals de cadena Sempre entre cometes dobles. Pot tenir qualsevol longitud, inclosa la cadena buida "". "42" (text) i 42 (nombre) són coses completament diferents.
Exemples: "Joan" · "Hola!" · ""
null Literal null Indica l'absència de valor en un tipus de referència. Només es pot assignar a variables de tipus Object, String, etc., mai a primitius.
Exemple: String nom = null;
BàsicLiterals de tots els tipus en acció
public class Literals { public static void main(String[] args) { int vides = 3; double preu = 19.99; char inicial = 'J'; boolean estaObert = true; String botiga = "La Mercè Market"; System.out.println(vides); System.out.println(preu); System.out.println(inicial); System.out.println(estaObert); System.out.println(botiga); } }
⚠️ Trampes freqüents: escriure 3.14 en una variable float sense la f dona error de compilació. Escriure True o False en un boolean també dona error.

4. Variables

Una variable és un espai en memòria amb nom que emmagatzema un valor que pot canviar durant l'execució del programa. Té tres elements: un tipus, un nom i un valor.

Sintaxi de declaració i inicialització:
tipusDada nomVariable; → declaració sense valor inicial
tipusDada nomVariable = valor; → declaració amb inicialització simultània
nomVariable = nouValor; → modificació d'un valor existent (sense repetir el tipus)
BàsicDeclarar, inicialitzar i modificar variables
public class Variables { public static void main(String[] args) { // Declaració i inicialització simultànies int edat = 25; double salari = 1500.50; boolean eActiu = true; String nom = "Anna"; // Declaració separada de la inicialització char genere; genere = 'F'; System.out.println(edat); // 25 // Modificar el valor (sense repetir el tipus!) edat = 30; System.out.println(edat); // 30 } }
MitjàVariables per a una reserva de vol
public class ReservaVol { public static void main(String[] args) { String destí = "Barcelona → Tokyo"; int numPassatgers = 2; double preuPerPersona = 874.50; boolean equipatgeExtra = false; char classe = 'E'; // E=Econòmica, B=Business double totalReserva = numPassatgers * preuPerPersona; System.out.println("Destí: " + destí); System.out.println("Total: " + totalReserva + " €"); System.out.println("Classe: " + classe); } }

Inferència de tipus: la paraula clau var

A partir de Java 10, podem usar var i el compilador infereix el tipus a partir del valor inicial. Útil quan el tipus és obvi; no és adequat quan dificulta la lectura.

MitjàÚs de var per a inferència de tipus
public class TipusVar { public static void main(String[] args) { var nom = "Laia"; // inferit com a String var edat = 22; // inferit com a int var preu = 9.99; // inferit com a double var esMembre = true; // inferit com a boolean // INCORRECTE: var sense valor inicial — no compila // var x; → error: cannot infer type System.out.println(nom + " · edat: " + edat); } }
💡 var no és un tipus dinàmic: una vegada inferit, el tipus és fix i no es pot canviar. Usa var quan el tipus sigui completament obvi del valor inicial; en cas de dubte, escriu el tipus explícitament.

public class Botiga { public static void main(String[] args) { // Constants: valors que mai canvien en aquest programa final double IVA = 0.21; final double DESCOMPTE_SOCI = 0.10; final int DIES_DEVOLUCIO = 30; // Variables: valors que sí canvien double preuBase = 79.95; boolean esSoci = true; if (esSoci) preuBase -= preuBase * DESCOMPTE_SOCI; double preuFinal = preuBase * (1 + IVA); System.out.println("Preu final: " + preuFinal + " €"); System.out.println("Termini devolució: " + DIES_DEVOLUCIO + " dies"); // IVA = 0.18; ← error de compilació! no es pot reassignar } }

public class ConfiguracioJoc { public static void main(String[] args) { final int VIDES_INICIALS = 3; final int PUNTS_PER_NIVELL = 100; final double VELOCITAT_BASE = 1.5; final String NOM_JOC = "SpaceQuest"; int vides = VIDES_INICIALS; int punts = 0; int nivell = 1; // El jugador passa de nivell punts += PUNTS_PER_NIVELL; nivell++; System.out.println(NOM_JOC + " · Nivell: " + nivell + " · Punts: " + punts); } }

6. Operadors i expressions

Un operador és un símbol que indica a Java que ha de realitzar una operació sobre un o més valors (operands). El resultat d'aplicar un operador s'anomena expressió.

CategoriaOperadorsExemple
Aritmètics+ - * / %5 + 3 → 8
Assignació= += -= *= /= %=a += 5
Comparació== != > < >= <=a > b → true/false
Lògics&& || !a && b → true/false
Unaris+ - ++ --++a, a++
Ternari? :(a > b) ? a : b
6.1 Operadors aritmètics
⚠️ Atenció amb la divisió entera: 5 / 3 = 1, no 1.66. Java descarta els decimals quan tots dos operands són enters. Per obtenir decimals cal que almenys un sigui double: 5.0 / 3 = 1.666...
BàsicOperadors aritmètics bàsics
public class OperadorsAritmeticos { public static void main(String[] args) { int a = 10, b = 3; System.out.println(a + b); // 13 — suma System.out.println(a - b); // 7 — resta System.out.println(a * b); // 30 — multiplicació System.out.println(a / b); // 3 — divisió ENTERA (descarta .33) System.out.println(a % b); // 1 — mòdul: resta de 10/3 } }
MitjàCàlcul del preu amb descompte i IVA
public class CalculPreu { public static void main(String[] args) { double preuBase = 120.0; double descompte = preuBase * 0.15; // 15% → 18.0 double preuNet = preuBase - descompte; // 102.0 double iva = preuNet * 0.21; // 21.42 double preuFinal = preuNet + iva; // 123.42 System.out.println("Preu final: " + preuFinal + " €"); } }
AvançatMòdul aplicat: convertir minuts a hores i minuts
public class ConvertirMinuts { public static void main(String[] args) { int minuts = 137; int hores = minuts / 60; // divisió entera → 2 int minutsRst = minuts % 60; // mòdul → 17 System.out.println(minuts + " minuts = " + hores + "h " + minutsRst + "min"); // 137 minuts = 2h 17min // Comprovar si un any és de traspàs (divisible per 4) int any = 2024; boolean esTraspos = (any % 4 == 0); System.out.println(any + " és traspàs: " + esTraspos); // true } }
6.2 Operadors unaris (++ i --)
BàsicPre-increment vs post-increment
public class OperadorsUnaris { public static void main(String[] args) { int a = 5, resultat; // Pre-increment: primer incrementa, després usa el valor resultat = ++a; System.out.println(resultat); // 6 System.out.println(a); // 6 a = 5; // reiniciem // Post-increment: primer usa el valor, després incrementa resultat = a++; System.out.println(resultat); // 5 (valor original) System.out.println(a); // 6 (ja incrementat) } }
MitjàComptador de visitants i d'errors
public class Comptadors { public static void main(String[] args) { int visitants = 0; int errors = 3; visitants++; // entra un visitant → 1 visitants++; // entra un altre → 2 errors--; // resolem un error → 2 System.out.println("Visitants: " + visitants); System.out.println("Errors pendents: " + errors); } }
6.3 Operadors d'assignació composta
BàsicAcumular i reduir valors
public class AssignacioComposta { public static void main(String[] args) { int puntuacio = 100; puntuacio += 50; // puntuacio = puntuacio + 50 → 150 puntuacio -= 30; // puntuacio = puntuacio - 30 → 120 puntuacio *= 2; // puntuacio = puntuacio * 2 → 240 puntuacio /= 4; // puntuacio = puntuacio / 4 → 60 puntuacio %= 7; // puntuacio = puntuacio % 7 → 4 System.out.println(puntuacio); // 4 } }
MitjàAcumular el cost d'una comanda
public class TotalComanda { public static void main(String[] args) { double totalComanda = 0.0; totalComanda += 12.50; // pa i brioixeria totalComanda += 8.75; // làctics totalComanda += 24.99; // carn totalComanda *= 1.04; // taxa d'enviament (4%) System.out.println("Total amb enviament: " + totalComanda + " €"); } }
6.4 Operadors de comparació

Comparen dos valors i sempre retornen un boolean: true o false. Són la base de totes les condicions i decisions d'un programa.

BàsicComparar edat i nota
public class Comparacions { public static void main(String[] args) { int edat = 17; double nota = 4.8; System.out.println(edat >= 18); // false — és menor d'edat System.out.println(nota == 5.0); // false System.out.println(nota < 5.0); // true — suspès System.out.println(nota != 10.0); // true } }
⚠️ No confondre == amb =: edat == 18 pregunta si és igual a 18 (expressió booleana). edat = 18 assigna el valor 18 a la variable (modifica el valor). Confondre'ls és un dels errors més freqüents en Java.
MitjàComparació de temperatura i aforament
public class ControlAcces { public static void main(String[] args) { double tempActual = 38.2; int aforament = 150; int personesAvui = 148; boolean teFebre = tempActual > 37.5; // true boolean salaPlena = personesAvui >= aforament; // false boolean quedaLloc = personesAvui != aforament; // true System.out.println("Té febre: " + teFebre); System.out.println("Sala plena: " + salaPlena); System.out.println("Queda lloc: " + quedaLloc); } }

OperadorNomRetorna true quan...
&&AND (i)TOTS DOS operands són true

BàsicCondicions simples amb AND, OR i NOT
public class OperadorsLogics { public static void main(String[] args) { boolean teCarnet = true; boolean esMajor = true; boolean estaCansat = false; System.out.println(teCarnet && esMajor); // true — pot conduir System.out.println(teCarnet || esMajor); // true — almenys un cert System.out.println(!estaCansat); // true — no està cansat System.out.println(teCarnet && !estaCansat); // true — pot conduir i descansa } }
MitjàCondicions d'accés a una plataforma
public class AccesPlataforma { public static void main(String[] args) { boolean teCompte = true; boolean contraCorrecta = false; boolean esBlocat = false; int intents = 2; // Pot accedir si té compte, contrasenya correcta i no és bloquejat boolean potAccedir = teCompte && contraCorrecta && !esBlocat; System.out.println("Accés permès: " + potAccedir); // false // Avís si li queden pocs intents O la contrasenya és incorrecta boolean mostrarAvis = (intents <= 3) || !contraCorrecta; System.out.println("Mostrar avís: " + mostrarAvis); // true } }
AvançatValidació d'un formulari d'inscripció
public class ValidacioInscripcio { public static void main(String[] args) { int edat = 22; boolean teNIF = true; boolean haPagat = true; boolean placesDisponibles = false; // Pot inscriure's si: és major, té NIF, ha pagat i hi ha places boolean potInscriure = (edat >= 18) && teNIF && haPagat && placesDisponibles; System.out.println("Inscripció OK: " + potInscriure); // false // Necessita ajuda si és menor O no ha pagat O no hi ha places boolean necessitaAjuda = (edat < 18) || !haPagat || !placesDisponibles; System.out.println("Necessita ajuda: " + necessitaAjuda); // true } }
6.6 Concatenació de cadenes

L'operador + amb String concatena (uneix) cadenes. Quan es mesclen nombres i cadenes, l'ordre dels operands importa molt.

BàsicConcatenar nom i dades personals
public class Concatenacio { public static void main(String[] args) { String nom = "Marta"; int edat = 23; double nota = 8.5; boolean aprovat = true; System.out.println("Nom: " + nom); System.out.println("Edat: " + edat + " anys"); System.out.println("Nota: " + nota + " · Aprovat: " + aprovat); } }
MitjàLa trampa de l'ordre: suma vs concatenació
public class OrdreConcatenacio { public static void main(String[] args) { int a = 5, b = 3; // Cadena primer → sempre concatena d'esquerra a dreta System.out.println("Resultat: " + a + b); // "Resultat: 53" ← atenció! System.out.println("Resultat: " + (a + b)); // "Resultat: 8" ← correcte // Nombres primer → primer suma, després concatena System.out.println(a + b + " és la suma"); // "8 és la suma" System.out.println(a + (b + " és la suma")); // "53 és la suma" ← atenció! } }
AvançatResum de comanda formatat
public class ResumComanda { public static void main(String[] args) { String producte = "Portàtil HP 15"; int quantitat = 2; double preu = 649.99; boolean estocOk = true; double total = quantitat * preu; System.out.println("─── Resum de la comanda ───"); System.out.println("Producte: " + producte); System.out.println("Quantitat: " + quantitat + " unitats"); System.out.println("Preu unit: " + preu + " €"); System.out.println("TOTAL: " + total + " €"); System.out.println("En estoc: " + estocOk); } }
💡 Precedència dels operadors (de major a menor): Unaris (++ --) → Aritmètics (* / %) → Aritmètics (+ -) → Comparació (> < >= <=) → Igualtat (== !=) → AND lògic (&&) → OR lògic (||) → Assignació (= +=...). Usa parèntesis per forçar l'ordre quan calgui.

7. Conversions de tipus

De vegades cal convertir un valor d'un tipus a un altre. Hi ha tres situacions principals.

TipusNomCal escriure res?Pot perdre dades?
ImplícitaWidening (eixamplar)No, Java ho fa solNo
ExplícitaCasting (estrenyir)Sí, cal posar (tipus)Sí (truncament, desbordament)
String ↔ primitiuParsing / valueOfSí, cal usar mètodesPossible (NumberFormatException)
Jerarquia de tipus numèrics (sentit del widening):
byte → short → int → long → float → double
Les fletxes indiquen el sentit de la conversió implícita. En sentit contrari cal casting explícit.
7.1 Conversió implícita (Widening)
BàsicCadena de widenings automàtics
public class Widening { public static void main(String[] args) { byte b = 42; short s = b; // byte → short (automàtic) int i = s; // short → int (automàtic) long l = i; // int → long (automàtic) float f = l; // long → float (automàtic) double d = f; // float → double (automàtic) System.out.println(d); // 42.0 } }
MitjàWidening en operació aritmètica mixta
public class WideningMixt { public static void main(String[] args) { int alumnes = 30; double nota = 7.5; // Java converteix alumnes a double automàticament double sumaNotes = alumnes * nota; // 225.0 // char → int: Java usa el codi Unicode char lletra = 'A'; int codi = lletra; // 65 (codi Unicode de 'A') System.out.println(sumaNotes); System.out.println(codi); // 65 } }
7.2 Casting explícit (Narrowing)
BàsicSintaxi del casting i truncament de decimals
public class Casting { public static void main(String[] args) { double preu = 19.99; int preuEnt = (int) preu; // cal el (int) explícit System.out.println(preuEnt); // 19 — els decimals es perden (trunca, no arrodoneix!) double resultat = 7.0 / 2.0; // 3.5 int enter = (int) resultat; System.out.println(enter); // 3 (trunca, no 4!) } }
MitjàCasting en càlculs de notes i codis
public class CastingNotes { public static void main(String[] args) { // Convertir una nota decimal a enter per mostrar-la double notaReal = 8.7; int notaEntera = (int) notaReal; // 8 (no 9!) System.out.println("Nota: " + notaEntera + "/10"); // Convertir un codi Unicode en caràcter int codi = 65; char lletra = (char) codi; System.out.println("Lletra: " + lletra); // A } }
AvançatDesbordament i arrodoniment correcte
public class CastingAvançat { public static void main(String[] args) { // Desbordament: int → byte (byte té rang -128 a 127) int gran = 130; byte petit = (byte) gran; System.out.println(petit); // -126 (desbordament! resultat inesperat) // Si necessites arrodonir (no truncar), usa Math.round() double nota = 8.7; int rodona = (int) Math.round(nota); System.out.println(rodona); // 9 (arrodonit correctament) } }
⚠️ El casting trunca, no arrodoneix: (int) 9.99 dona 9, no 10. Si necessites arrodonir, usa Math.round().
