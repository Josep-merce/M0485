1. Herència: la paraula clau extends

L'herència permet crear una classe nova basada en una d'existent, reutilitzant el seu codi i afegint noves característiques. La classe de la qual s'hereta s'anomena superclasse; la que hereta, subclasse.

Regla "és un": Usa herència quan pots dir que la subclasse "és un" tipus de la superclasse. Un Gos és un Animal. Un Cotxe és un Vehicle. Si no pots dir "és un", probablement cal composició (que veurem a A2).
public class Animal { String nom; int edat; Animal(String nom, int edat) { this.nom = nom; this.edat = edat; } void descriure() { System.out.println("Animal: " + nom + ", edat: " + edat); } } public class Gos extends Animal { // Gos hereta d'Animal String raca; Gos(String nom, int edat, String raca) { super(nom, edat); // crida al constructor d'Animal this.raca = raca; } void bordar() { System.out.println(nom + " diu: Bup!"); // nom és heretat d'Animal } }

2. La paraula clau super()

super() crida el constructor de la superclasse des del constructor de la subclasse. Ha de ser sempre la primera instrucció del constructor. Si la superclasse no té constructor sense paràmetres, és obligatori cridar super(...) explícitament.

Error típic: Oblidar super() quan la superclasse té constructor amb paràmetres. El compilador donarà error: "There is no default constructor available in [superclasse]".

3. Modificador final

La paraula clau final permet bloquejar l'herència o la sobreescriptura de mètodes i classes:

Aplicat aEfecteExemple
ClasseNo es pot heretar d'ellapublic final class String
MètodeLes subclasses no el poden sobreescriurepublic final void calcular()
VariableEl seu valor no es pot canviar (constant)final int MAX = 100;

4. Sobreescriptura de mètodes (@Override)

Quan una subclasse defineix un mètode amb el mateix nom, paràmetres i tipus de retorn que un de la superclasse, el sobreescriu. L'anotació @Override ho indica explícitament al compilador i evita errors de tipografia.

public class Figura { double area() { return 0; } } public class Rectangle extends Figura { double base, altura; Rectangle(double base, double altura) { this.base = base; this.altura = altura; } @Override double area() { // sobreescriu el mètode de Figura return base * altura; } }
Bona pràctica: Usa sempre @Override. Si la signatura no coincideix exactament amb cap mètode de la superclasse, el compilador t'avisarà en lloc de crear un mètode nou per error.

5. Polimorfisme dinàmic

El polimorfisme permet que una variable de tipus superclasse apunti a un objecte d'una subclasse. En temps d'execució, Java executa el mètode de la classe real de l'objecte, no el de la variable de referència.

Figura[] formes = { new Rectangle(5, 3), new Triangle(4, 6), new Cercle(2) }; for (Figura f : formes) { // Java crida al area() correcte de cada subclasse → polimorfisme System.out.println("Àrea: " + f.area()); }

6. instanceof i càsting d'objectes

Amb una variable de la superclasse podem saber quina subclasse és l'objecte real usant instanceof, i accedir als membres específics fent un càsting explícit.

Figura f = new Rectangle(5, 3); if (f instanceof Rectangle) { Rectangle r = (Rectangle) f; // càsting explícit System.out.println("Base: " + r.base); }
Atenció: Un càsting incorrecte llança ClassCastException en temps d'execució. Comprova sempre amb instanceof abans de fer el càsting.

7. Documentació amb Javadoc

Javadoc és l'estàndard de Java per documentar classes i mètodes. S'escriu amb comentaris /** ... */ just abans de la classe o mètode. A partir d'ara és obligatori en tots els exercicis.

EtiquetaDescripcióExemple
@paramDescriu un paràmetre d'entrada@param nom el nom de l'empleat
@returnDescriu el valor de retorn@return el salari en euros
@throwsDocumenta les excepcions possibles@throws IllegalArgumentException si negatiu
@authorAutor de la classe@author Nom Cognom
/** * Representa un empleat a temps complet. * * @author Nom Cognom * @version 1.0 */ public class EmpleatTempsComplet extends Empleat { /** * Crea un empleat a temps complet. * @param nom el nom de l'empleat * @param id identificador únic * @param salari salari mensual en euros */ public EmpleatTempsComplet(String nom, int id, double salari) { super(nom, id); this.salariMensual = salari; } /** * Retorna el salari mensual fix. * @return el salari en euros */ @Override public double calcularSalari() { return salariMensual; } }

8. Exercicis

Ex 1 Figures geomètriques

Crea una jerarquia bàsica de figures geomètriques amb herència i sobreescriptura.

  • Classe Figura: atributs color i nom (String). Constructor i mètode descriure(). Mètode area() que retorni 0.
  • Classe Rectangle extends Figura: afegeix base i altura (double). Constructor que cridi super(). Sobreescriu area().
  • Classe Cercle extends Figura: afegeix radi (double). Sobreescriu area() (Math.PI * radi * radi).
  • Al main: crea un array de Figura amb un rectangle i un cercle. Recorre'l i imprimeix descriure() i l'àrea de cada un.
  • Afegeix Javadoc a totes les classes i mètodes.
Bàsic CA: 7.1 · 7.3 · 7.4 · 7.5 · 7.8
✅ Solució
/** Classe base per a figures geomètriques. */ public class Figura { String nom, color; /** @param nom nom de la figura * @param color color de la figura */ Figura(String nom, String color) { this.nom = nom; this.color = color; } /** Imprimeix el nom i el color de la figura. */ void descriure() { System.out.println(nom + " [" + color + "]"); } /** @return l'àrea de la figura */ double area() { return 0; } } /** Rectangle que hereta de Figura. */ public class Rectangle extends Figura { double base, altura; Rectangle(String color, double base, double altura) { super("Rectangle", color); this.base = base; this.altura = altura; } @Override double area() { return base * altura; } } /** Cercle que hereta de Figura. */ public class Cercle extends Figura { double radi; Cercle(String color, double radi) { super("Cercle", color); this.radi = radi; } @Override double area() { return Math.PI * radi * radi; } } public class Main { public static void main(String[] args) { Figura[] formes = { new Rectangle("blau", 5.0, 3.0), new Cercle("vermell", 4.0) }; for (Figura f : formes) { f.descriure(); System.out.println(" Àrea: " + f.area()); } } }
Ex 2 Zoo polimòrfic

Practica sobreescriptura i polimorfisme amb animals i instanceof.

  • Classe Animal: nom i edat. Mètodes ferSoroll() (imprimeix "...") i mostrar().
  • Subclasses Gat, Gos i Ocell: sobreescriuen ferSoroll() amb el so correcte i mostrar() afegint una característica pròpia.
  • Al main: crea un ArrayList<Animal> amb almenys 5 animals (barreja de tipus). Recorre la llista, crida mostrar() i ferSoroll().
  • Compta quants Gat hi ha a la llista usant instanceof i imprimeix el total.
  • Javadoc obligatori a totes les classes.

import java.util.ArrayList; /** Classe base que representa un animal genèric. */ public class Animal { String nom; int edat; /** @param nom el nom de l'animal @param edat l'edat en anys */ Animal(String nom, int edat) { this.nom = nom; this.edat = edat; } /** Emet el so de l'animal. */ void ferSoroll() { System.out.println(nom + ": ..."); } /** Mostra informació de l'animal. */ void mostrar() { System.out.println("Animal: " + nom + ", edat: " + edat); } } public class Gat extends Animal { Gat(String n, int e) { super(n, e); } @Override void ferSoroll() { System.out.println(nom + ": Miol!"); } @Override void mostrar() { System.out.println("[Gat] " + nom + ", edat: " + edat); } } public class Gos extends Animal { Gos(String n, int e) { super(n, e); } @Override void ferSoroll() { System.out.println(nom + ": Bup!"); } @Override void mostrar() { System.out.println("[Gos] " + nom + ", edat: " + edat); } } public class Ocell extends Animal { Ocell(String n, int e) { super(n, e); } @Override void ferSoroll() { System.out.println(nom + ": Piu!"); } @Override void mostrar() { System.out.println("[Ocell] " + nom + ", edat: " + edat); } } public class Main { public static void main(String[] args) { ArrayList<Animal> zoo = new ArrayList<>(); zoo.add(new Gat("Whiskers", 2)); zoo.add(new Gos("Rex", 4)); zoo.add(new Gat("Tom", 3)); zoo.add(new Ocell("Piu", 1)); zoo.add(new Gos("Laika", 5)); int numGats = 0; for (Animal a : zoo) { a.mostrar(); a.ferSoroll(); if (a instanceof Gat) numGats++; } System.out.println("\nTotal de gats: " + numGats); } }
Ex 3 Gestió d'empleats i projectes

Sistema de gestió d'empleats amb herència, sobreescriptura i polimorfisme complet.

  • Classe Empleat: nom (String) i id (int). Mètode calcularSalari() retorna 0.0. Mètode mostrarInfo().
  • Classe EmpleatTempsComplet extends Empleat: afegeix salariMensual (double). Sobreescriu els dos mètodes.
  • Classe EmpleatTempsParcial extends Empleat: afegeix horesTreballades (int) i tarifaHora (double). Sobreescriu els dos mètodes.
  • Classe Projecte: nom i un ArrayList<Empleat>. Mètodes afegirEmpleat() i mostrarEmpleats().
  • Al main: crea 2 empleats de cada tipus, 2 projectes, assigna'ls i mostra la suma total de salaris. Javadoc obligatori.
