1. Classes abstractes

Una classe abstracta és una classe que no es pot instanciar directament. Serveix de plantilla: defineix mètodes comuns (concrets) i declara mètodes abstractes que cada subclasse ha d'implementar obligatòriament.

Quan usar una classe abstracta? Quan diverses subclasses comparteixen atributs i codi base, però cadascuna té una implementació diferent d'algun mètode. Exemple: tots els pagaments tenen un import, però cada tipus de pagament es processa de forma diferent.
public abstract class Pagament { protected double import_; protected String data; Pagament(double import_, String data) { this.import_ = import_; this.data = data; } // Mètode abstracte: cada subclasse OBLIGATÒRIAMENT l'ha d'implementar public abstract void processar(); // Mètode concret: compartit per totes les subclasses public void mostrarImport() { System.out.println("Data: " + data + " | Import: " + import_ + "€"); } } public class Targeta extends Pagament { String numTargeta; Targeta(double imp, String data, String num) { super(imp, data); this.numTargeta = num; } @Override public void processar() { System.out.println("Càrrec de " + import_ + "€ a targeta " + numTargeta); } } // Això donaria ERROR de compilació: // Pagament p = new Pagament(50, "avui"); ← no es pot instanciar

2. Interfícies

Una interfície és un contracte: defineix quins mètodes ha d'implementar una classe, sense dir com. La gran avantatge és que una classe pot implementar múltiples interfícies, cosa que no és possible amb l'herència de classes.

public interface Volador { void volar(); // abstracte per defecte default void aterrar() { // mètode per defecte (Java 8+) System.out.println("Aterrant..."); } } public interface Nedador { void nedar(); } // Una classe pot implementar múltiples interfícies alhora public class Anec extends Animal implements Volador, Nedador { Anec(String nom) { super(nom, 1); } @Override public void volar() { System.out.println(nom + " vola!"); } @Override public void nedar() { System.out.println(nom + " neda!"); } @Override void ferSoroll() { System.out.println(nom + ": Quac!"); } }

3. Diferències: classe abstracta vs interfície

CaracterísticaClasse abstractaInterfície
InstanciacióNoNo
Herència / implementacióNomés una (extends)Múltiples (implements)
Atributs d'instànciaSíNo (només constants)
ConstructorsSíNo
Mètodes concretsSíSí, des de Java 8 (default)
Ús principalJerarquia "és un" amb codi compartitContracte de comportament transversal
Quan usar cada un? Usa classe abstracta quan les subclasses comparteixen atributs i codi base (Animals, Pagaments). Usa interfície quan vols definir un comportament que classes no relacionades poden compartir (Volador, Comparable, Serializable).

4. Composició: "té un" vs "és un"

L'herència modela la relació "és un". La composició modela "té un": una classe conté una instància d'una altra classe com a atribut i delega part del comportament.

// ❌ Herència mal aplicada: Motor NO és un Cotxe public class CotxeMalament extends Motor { ... } // ✅ Composició: un Cotxe TÉ UN Motor public class Motor { int cilindrada; void arrancar() { System.out.println("Motor arrancant..."); } } public class Cotxe extends Vehicle { // herència OK: Cotxe és un Vehicle private Motor motor; // composició: Cotxe té un Motor Cotxe(String marca, int any, int cil) { super(marca, any); this.motor = new Motor(cil); } void engegar() { motor.arrancar(); // delega al Motor System.out.println(marca + " llest per circular"); } }
Principi de disseny: "Prefereix composició sobre herència" quan no hi hagi una relació jeràrquica clara. La composició és menys acoblada i més fàcil de canviar.

5. Exercicis

Ex 1 Sistema de pagaments

Implementa un sistema de pagaments usant una classe abstracta.

  • Classe abstracta Pagament: import_ (double) i data (String). Mètode abstracte processar(). Mètode concret mostrarImport().
  • Subclasses Efectiu, Targeta (amb numTargeta) i Transferencia (amb iban). Cadascuna implementa processar().
  • Al main: crea un array de Pagament amb els tres tipus, crida processar() per a cada un i suma l'import total.
Bàsic CA: 7.2 · 7.4 · 7.5 · 7.7 · 7.9
✅ Solució
public abstract class Pagament { protected double import_; protected String data; Pagament(double imp, String data) { this.import_ = imp; this.data = data; } public abstract void processar(); public void mostrarImport() { System.out.println("Data: " + data + " | " + import_ + "€"); } } public class Efectiu extends Pagament { Efectiu(double imp, String data) { super(imp, data); } @Override public void processar() { System.out.println("[EFECTIU] " + import_ + "€ el " + data); } } public class Targeta extends Pagament { String numTargeta; Targeta(double imp, String data, String num) { super(imp, data); this.numTargeta = num; } @Override public void processar() { System.out.println("[TARGETA] " + import_ + "€ → " + numTargeta); } } public class Transferencia extends Pagament { String iban; Transferencia(double imp, String data, String iban) { super(imp, data); this.iban = iban; } @Override public void processar() { System.out.println("[TRANSF.] " + import_ + "€ → IBAN: " + iban); } } public class Main { public static void main(String[] args) { Pagament[] pagaments = { new Efectiu(50.0, "10/01/2025"), new Targeta(120.0, "11/01/2025", "**** 4321"), new Transferencia(800.0, "12/01/2025", "ES12 3456 7890") }; double total = 0; for (Pagament p : pagaments) { p.processar(); total += p.import_; } System.out.println("\nTotal: " + total + "€"); } }
Ex 2 Joc d'armes (interfícies + classes abstractes + composició)

Exercici integrador que combina tots els conceptes de l'AEA3: herència, classes abstractes, interfícies i composició.

  • Interfície Armable: mètode equiparPersonatge(Personatge p).
  • Classe Personatge (composició): té puntsVida, puntsAtac (double) i 4 booleans de protecció/atac. Conté un ArrayList<Armable>. Mètodes afegirArma(), equipar() i mostrarEstat().
  • Classe abstracta ArmaDefensa implements Armable: té percentatgeVida. Implementa equiparPersonatge() incrementant puntsVida i cridant el mètode abstracte activarProteccio().
  • Classe abstracta ArmaAtac implements Armable: té percentatgeAtac. Implementa equiparPersonatge() incrementant puntsAtac i cridant el mètode abstracte activarAtac().
  • Subclasses: Escut i Armadura d'ArmaDefensa; Espasa i Arc d'ArmaAtac.
  • Al main: crea un personatge, afegeix les 4 armes, equipa'l i mostra l'estat abans i després.
Avançat CA: 7.2 · 7.4 · 7.5 · 7.7 · 7.9 · 7.10
✅ Solució
import java.util.ArrayList; public interface Armable { void equiparPersonatge(Personatge p); } public class Personatge { double puntsVida = 100, puntsAtac = 10; boolean proteccioArquer, proteccioCosACos, atacArquer, atacCosACos; ArrayList<Armable> armes = new ArrayList<>(); void afegirArma(Armable a) { armes.add(a); } void equipar() { for (Armable a : armes) a.equiparPersonatge(this); } void mostrarEstat() { System.out.println("Vida: " + puntsVida + " | Atac: " + puntsAtac); System.out.println("Prot. arquer: " + proteccioArquer + " | cos a cos: " + proteccioCosACos); System.out.println("Atac arquer: " + atacArquer + " | cos a cos: " + atacCosACos); } } public abstract class ArmaDefensa implements Armable { protected double percentatgeVida; ArmaDefensa(double pct) { this.percentatgeVida = pct; } public abstract void activarProteccio(Personatge p); @Override public void equiparPersonatge(Personatge p) { p.puntsVida *= (1 + percentatgeVida); activarProteccio(p); } } public abstract class ArmaAtac implements Armable { protected double percentatgeAtac; ArmaAtac(double pct) { this.percentatgeAtac = pct; } public abstract void activarAtac(Personatge p); @Override public void equiparPersonatge(Personatge p) { p.puntsAtac *= (1 + percentatgeAtac); activarAtac(p); } } public class Escut extends ArmaDefensa { Escut() { super(0.10); } @Override public void activarProteccio(Personatge p) { p.proteccioArquer = true; } } public class Armadura extends ArmaDefensa { Armadura() { super(0.50); } @Override public void activarProteccio(Personatge p) { p.proteccioCosACos = true; } } public class Espasa extends ArmaAtac { Espasa() { super(0.40); } @Override public void activarAtac(Personatge p) { p.atacCosACos = true; } } public class Arc extends ArmaAtac { Arc() { super(0.15); } @Override public void activarAtac(Personatge p) { p.atacArquer = true; } } public class Main { public static void main(String[] args) { Personatge heroi = new Personatge(); heroi.afegirArma(new Escut()); heroi.afegirArma(new Armadura()); heroi.afegirArma(new Espasa()); heroi.afegirArma(new Arc()); System.out.println("=== Abans d'equipar ==="); heroi.mostrarEstat(); heroi.equipar(); System.out.println("\n=== Després d'equipar ==="); heroi.mostrarEstat(); } }
