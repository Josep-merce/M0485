1. Bases de dades orientades a objectes

En les bases de dades relacionals (MySQL, PostgreSQL…) les dades s'emmagatzemen en taules, i cal traduir els objectes Java a files i columnes. Quan les classes Java són complexes —amb herència, col·leccions i relacions entre objectes— aquesta traducció es complica i genera molt codi repetitiu.

Les bases de dades orientades a objectes (BDOO) solucionen aquest problema: permeten emmagatzemar directament els objectes Java sense cap traducció. L'objecte es guarda tal com és a memòria i es recupera en el mateix estat.

CaracterísticaBD Relacional (MySQL)BD Orientada a Objectes (ObjectDB)
Unitat d'emmagatzematgeFiles i columnesObjectes Java directament
Traducció necessàriaSí (Object-Relational Mapping)No — l'objecte es guarda tal qual
ConsultesSQLJPQL (Java Persistence Query Language)
Suport herènciaComplex (diverses estratègies)Natural — les subclasses s'emmagatzemen directament
Cas d'ús idealDades tabulars, reportsModels OO complexos, prototips ràpids

2. ObjectDB i JPA

ObjectDB és un sistema gestor de bases de dades orientades a objectes (SGBDOO) natiu de Java. Implementa directament l'estàndard JPA (Java Persistence API), la mateixa API que s'usa en Java EE per a la persistència.

Això vol dir que el codi que escrivim per a ObjectDB és estàndard JPA: si en el futur volguéssim canviar a Hibernate + MySQL o EclipseLink + PostgreSQL, el codi de les classes d'entitat i les consultes JPQL es podria reutilitzar en gran part.

📌 JPA és un estàndard, ObjectDB és una implementació: JPA defineix les anotacions (@Entity, @Id…) i les interfícies (EntityManager, EntityManagerFactory…). ObjectDB és el motor que les implementa per a bases de dades d'objectes. Altres implementacions de JPA populars són Hibernate, EclipseLink i OpenJPA.

3. Instal·lació a NetBeans

Per treballar amb ObjectDB només calen dos fitxers .jar que trobareu al Drive del curs:

  • objectdb-2.9.5.jar — el motor de la base de dades ObjectDB
  • javax.persistence-api-2.2.jar — les interfícies i anotacions estàndard JPA

Per afegir-los a un projecte NetBeans:

  1. Creeu una carpeta lib/ a l'arrel del vostre projecte NetBeans.
  2. Copieu els dos fitxers .jar dins de lib/.
  3. A NetBeans, botó dret sobre Libraries → Add JAR/Folder.
  4. Seleccioneu cada .jar usant la ruta relativa (no absoluta).
  5. Comproveu que apareixen a la llista de llibreries del projecte.
💡 Ruta relativa: Useu sempre ruta relativa per als .jar. D'aquesta manera el projecte funcionarà en qualsevol ordinador sense reconfigurar res. Si useu una ruta absoluta tipus C:\Users\..., el projecte no s'obrirà a d'altres màquines.

4. Explorar bases de dades amb Database Explorer

ObjectDB inclou una eina gràfica, el Database Explorer, que permet inspeccionar el contingut d'un fitxer .odb sense necessitat d'escriure codi.

Per obrir-lo, des de la carpeta on heu descomprimit ObjectDB executeu:

// Executeu des de la carpeta d'instal·lació d'ObjectDB: java -jar objectdb/bin/explorer.jar

Al Drive del curs trobareu dos fitxers de base de dades de mostra:

  • point.odb — base de dades senzilla amb objectes Punt (x, y)
  • world.odb — base de dades amb dades de països, ciutats i idiomes
⚠️ No podeu tenir la BD oberta alhora: Si teniu el fitxer .odb obert al Database Explorer i intenteu executar el vostre programa Java, obtindreu un error de bloqueig. Tanqueu la connexió a l'Explorer (Connection → Disconnect) abans d'executar el programa, i viceversa.
📌 Format del fitxer: Una base de dades ObjectDB és un únic fitxer .odb. No cal instal·lar cap servidor ni servei: simplement indiqueu la ruta del fitxer al vostre programa i ObjectDB s'encarrega de tot. Això la fa ideal per a aplicacions d'escriptori i prototips.

5. Estructura bàsica d'un programa JPA

Tot programa que usa JPA segueix el mateix esquelet. Les tres classes fonamentals són:

  • EntityManagerFactory — es crea una vegada i gestiona la connexió amb la BD
  • EntityManager — s'usa per fer operacions (guardar, cercar, eliminar, consultar)
  • EntityTransaction — agrupa les operacions en una transacció atòmica
import javax.persistence.EntityManager; import javax.persistence.EntityManagerFactory; import javax.persistence.Persistence; public class Main { public static void main(String[] args) { // 1. Obrir la "fàbrica" (connexió amb la BD) EntityManagerFactory emf = Persistence.createEntityManagerFactory("$objectdb/db/prova.odb"); // 2. Obtenir un EntityManager (sessió de treball) EntityManager em = emf.createEntityManager(); // 3. Iniciar la transacció em.getTransaction().begin(); // 4. Fer operacions (guardar, cercar, actualitzar, eliminar) // ... codi de les operacions // 5. Confirmar la transacció em.getTransaction().commit(); // 6. Tancar en ordre invers a l'obertura em.close(); emf.close(); } }
⚠️ Ordre de tancament: Sempre tanqueu primer l'EntityManager i després l'EntityManagerFactory. Si no tanqueu l'EMF, el fitxer .odb pot quedar bloquejat i el proper cop que intenteu obrir-lo obtindreu un error.

Connexió directa vs persistence.xml

Hi ha dues maneres d'indicar a JPA quina base de dades cal obrir:

MètodeCodiQuan usar-la
Connexió directa createEntityManagerFactory("$objectdb/db/fitxer.odb") Proves ràpides, exemples senzills
persistence.xml createEntityManagerFactory("NomUnitat") + fitxer XML Projectes reals, configurable sense recompilar

El fitxer persistence.xml

El fitxer persistence.xml s'ha de col·locar a src/META-INF/persistence.xml. Defineix una unitat de persistència amb el nom i la URL de la base de dades:

<?xml version="1.0" encoding="UTF-8"?> <persistence version="2.1" xmlns="http://xmlns.jcp.org/xml/ns/persistence" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/persistence http://xmlns.jcp.org/xml/ns/persistence/persistence_2_1.xsd"> <persistence-unit name="ProvaUnit"> <provider>com.objectdb.jpa.Provider</provider> <properties> <property name="javax.persistence.jdbc.url" value="$objectdb/db/prova.odb"/> </properties> </persistence-unit> </persistence>
💡 La variable $objectdb: ObjectDB interpreta $objectdb com la carpeta d'instal·lació d'ObjectDB. Si voleu que el fitxer .odb es creï en una ruta concreta del vostre projecte, useu una ruta relativa com ara db/prova.odb.

6. Exercicis

E1 Primera connexió amb ruta directa Bàsic

Crea un projecte NetBeans nou i configura'l per usar ObjectDB. L'objectiu és únicament obrir la connexió amb la base de dades i tancar-la correctament, sense emmagatzemar cap objecte.

  1. Crea un projecte Java nou a NetBeans anomenat PrimerObjectDB.
  2. Afegeix els dos .jar d'ObjectDB a la carpeta lib/ del projecte i registra'ls a Libraries.
  3. Escriu una classe Main que obri una EntityManagerFactory amb ruta directa "$objectdb/db/prova.odb", crei un EntityManager i els tanqui en ordre invers.
  4. Mostra "Connexió oberta correctament!" i "Connexió tancada." per consola.
  5. Executa el programa i comprova que s'ha creat el fitxer prova.odb.
Bàsic CA 8.1 · 8.3
Solució — Primera connexió amb ruta directa
import javax.persistence.EntityManager; import javax.persistence.EntityManagerFactory; import javax.persistence.Persistence; public class Main { public static void main(String[] args) { EntityManagerFactory emf = Persistence.createEntityManagerFactory("$objectdb/db/prova.odb"); EntityManager em = emf.createEntityManager(); System.out.println("Connexió oberta correctament!"); em.close(); emf.close(); System.out.println("Connexió tancada."); } }
E2 Connexió via persistence.xml Mitjà

Refés l'exercici anterior però ara usant un fitxer persistence.xml per configurar la base de dades. Afegeix un bloc try-finally per garantir el tancament fins i tot si es produeix un error.

  1. Crea la carpeta src/META-INF/ dins del teu projecte NetBeans.
  2. Crea el fitxer persistence.xml amb una unitat de persistència anomenada "ProvaUnit" que apunti a db/prova2.odb.
  3. Modifica la classe Main perquè usi Persistence.createEntityManagerFactory("ProvaUnit").
  4. Embolica l'obertura i el tancament en un bloc try-finally. Declara emf i em com a null abans del try i comprova que no siguin null al finally abans de tancar.
Mitjà CA 8.2 · 8.3
Solució — Connexió via persistence.xml
// src/META-INF/persistence.xml <?xml version="1.0" encoding="UTF-8"?> <persistence version="2.1" xmlns="http://xmlns.jcp.org/xml/ns/persistence" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/persistence http://xmlns.jcp.org/xml/ns/persistence/persistence_2_1.xsd"> <persistence-unit name="ProvaUnit"> <provider>com.objectdb.jpa.Provider</provider> <properties> <property name="javax.persistence.jdbc.url" value="db/prova2.odb"/> </properties> </persistence-unit> </persistence>
// Main.java import javax.persistence.EntityManager; import javax.persistence.EntityManagerFactory; import javax.persistence.Persistence; public class Main { public static void main(String[] args) { EntityManagerFactory emf = null; EntityManager em = null; try { emf = Persistence.createEntityManagerFactory("ProvaUnit"); em = emf.createEntityManager(); System.out.println("Connexió establerta via persistence.xml!"); } finally { if (em != null) em.close(); if (emf != null) emf.close(); System.out.println("Connexió tancada."); } } }
E3 Explorar world.odb amb JPQL Avançat

Obre la base de dades world.odb del Drive del curs amb el Database Explorer i, a continuació, escriu un programa Java per fer consultes bàsiques sobre el seu contingut.

  1. Obre world.odb amb el Database Explorer d'ObjectDB. Navega per les classes emmagatzemades i anota els noms de les classes i les seves propietats principals.
  2. Crea un projecte NetBeans nou i connecta't a world.odb via ruta directa.
  3. Usa em.createQuery("SELECT COUNT(c) FROM Country c", Long.class).getSingleResult() per comptar el nombre de països i mostra'l per consola.
  4. Usa una consulta JPQL per obtenir els 5 primers noms de país en ordre alfabètic i mostra'ls.
  5. Usa ORDER BY c.population DESC i setMaxResults(1) per obtenir el país amb més població.
Avançat CA 8.1 · 8.2 · 8.3
Solució — Consultes sobre world.odb
import javax.persistence.*; import java.util.List; public class Main { public static void main(String[] args) { EntityManagerFactory emf = null; EntityManager em = null; try { emf = Persistence.createEntityManagerFactory( "$objectdb/db/world.odb"); em = emf.createEntityManager(); // Nombre total de països long total = em.createQuery( "SELECT COUNT(c) FROM Country c", Long.class) .getSingleResult(); System.out.println("Total de països: " + total); // 5 primers en ordre alfabètic List<String> noms = em.createQuery( "SELECT c.name FROM Country c ORDER BY c.name ASC", String.class) .setMaxResults(5) .getResultList(); System.out.println("Primers 5 països: " + noms); // País amb més població String mesPoble = em.createQuery( "SELECT c.name FROM Country c ORDER BY c.population DESC", String.class) .setMaxResults(1) .getSingleResult(); System.out.println("País amb més població: " + mesPoble); } finally { if (em != null) em.close(); if (emf != null) emf.close(); } } }
