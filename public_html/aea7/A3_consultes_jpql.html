1. Recuperació per clau primària: find()

El mètode find() recupera un objecte directament per la seva clau primària. És la forma més eficient quan es coneix l'ID:

// Recupera el Producte amb id = 1L Producte p = em.find(Producte.class, 1L); if (p != null) { System.out.println(p.getNom() + " — " + p.getPreu() + " €"); } else { System.out.println("Producte no trobat"); }
📌 find() no necessita transacció: find() no modifica la BD, per tant no requereix transacció activa. Retorna null si no existeix cap objecte amb aquella clau primària.

2. Consultes JPQL

JPQL (Java Persistence Query Language) és el llenguatge de consultes de JPA. S'assembla a SQL però opera sobre classes i atributs Java, no sobre taules i columnes.

SQLJPQL equivalent
SELECT * FROM producteSELECT p FROM Producte p
SELECT * FROM producte WHERE preu > 50SELECT p FROM Producte p WHERE p.preu > 50
SELECT * FROM producte ORDER BY preu ASCSELECT p FROM Producte p ORDER BY p.preu ASC
SELECT COUNT(*) FROM producteSELECT COUNT(p) FROM Producte p
UPDATE producte SET preu = preu * 0.9UPDATE Producte p SET p.preu = p.preu * 0.9
DELETE FROM producte WHERE disponible = 0DELETE FROM Producte p WHERE p.disponible = false
// JPQL bàsic: tots els productes List<Producte> llista = em .createQuery("SELECT p FROM Producte p", Producte.class) .getResultList(); for (Producte p : llista) { System.out.println(p.getNom()); } // JPQL amb filtre List<Producte> cars = em .createQuery("SELECT p FROM Producte p WHERE p.preu > 50", Producte.class) .getResultList(); // JPQL amb ORDER BY List<Producte> ordenats = em .createQuery("SELECT p FROM Producte p ORDER BY p.preu ASC", Producte.class) .getResultList();
Paràmetres de consulta

Les consultes amb paràmetres eviten problemes de seguretat (injecció) i milloren la llegibilitat:

// Paràmetres de NOM (recomanats) List<Producte> resultat = em .createQuery( "SELECT p FROM Producte p WHERE p.disponible = :disp AND p.preu < :maxPreu", Producte.class) .setParameter("disp", true) .setParameter("maxPreu", 100.0) .getResultList(); // Paràmetres ORDINALS (posicionals, menys llegibles) List<Producte> res2 = em .createQuery( "SELECT p FROM Producte p WHERE p.preu BETWEEN ?1 AND ?2", Producte.class) .setParameter(1, 20.0) .setParameter(2, 80.0) .getResultList();
@NamedQuery: consultes predefinides a l'entitat

Les consultes amb nom es defineixen directament a la classe d'entitat amb l'anotació @NamedQuery. S'executen de manera més eficient perquè es compilen una sola vegada:

@Entity @NamedQuery( name = "Producte.findDisponibles", query = "SELECT p FROM Producte p WHERE p.disponible = true ORDER BY p.preu" ) public class Producte implements Serializable { /* ... */ } // Ús de la named query List<Producte> disp = em .createNamedQuery("Producte.findDisponibles", Producte.class) .getResultList();

3. Actualització d'objectes

Actualització transparent (objecte managed)

Si modifiques un objecte en estat managed dins d'una transacció activa, JPA detecta el canvi automàticament:

em.getTransaction().begin(); Producte p = em.find(Producte.class, 1L); p.setPreu(79.99); // Canvi detectat automàticament p.setEstoc(20); em.getTransaction().commit(); // El canvi es desa sense cridar cap mètode addicional
Actualització amb merge() (objecte detached)

Si l'objecte s'ha obtingut en una sessió anterior (EntityManager tancat), cal usar merge():

// Sessió 1: obtenim i tanquem Producte p = em.find(Producte.class, 1L); em.close(); // p queda en estat DETACHED // Modifiquem el producte desconnectat p.setPreu(55.00); // Sessió 2: fusionem amb merge() EntityManager em2 = emf.createEntityManager(); em2.getTransaction().begin(); em2.merge(p); // Fusiona l'objecte detached amb la BD em2.getTransaction().commit();

4. Eliminació d'objectes

// Eliminació explícita d'un objecte em.getTransaction().begin(); Producte p = em.find(Producte.class, 3L); if (p != null) { em.remove(p); // Marca per eliminar; efectiu en fer commit } em.getTransaction().commit(); // Eliminació massiva amb JPQL (DELETE) em.getTransaction().begin(); int eliminats = em .createQuery("DELETE FROM Producte p WHERE p.disponible = false") .executeUpdate(); System.out.println("Eliminats: " + eliminats); em.getTransaction().commit();
⚠️ remove() requereix objecte managed: No pots cridar remove() sobre un objecte detached. Primer cal fer find() per obtenir-lo en estat managed, i llavors eliminar-lo.

5. Exercicis

E1 Consultes JPQL sobre Producte Bàsic

Partint del projecte de productes de l'A2 (amb almenys 5 productes a la BD), implementa les consultes següents.

  1. Recupera i mostra tots els productes amb find() usant els IDs 1 a 5 en un bucle.
  2. Llista tots els productes disponibles (disponible = true) ordenats per preu ascendent.
  3. Mostra el nombre total de productes amb una consulta agregada (COUNT).
  4. Mostra el preu màxim i el preu mínim dels productes disponibles.
Bàsic CA 8.4 · 8.7
Solució — Consultes JPQL sobre Producte
// 1. find() en bucle for (long i = 1; i <= 5; i++) { Producte p = em.find(Producte.class, i); if (p != null) System.out.println(i + ": " + p.getNom()); } // 2. Disponibles per preu List<Producte> disp = em .createQuery( "SELECT p FROM Producte p WHERE p.disponible = true ORDER BY p.preu ASC", Producte.class) .getResultList(); disp.forEach(p -> System.out.println(p.getNom() + ": " + p.getPreu() + "€")); // 3. COUNT long total = em .createQuery("SELECT COUNT(p) FROM Producte p", Long.class) .getSingleResult(); System.out.println("Total: " + total); // 4. MAX i MIN double maxPreu = em .createQuery("SELECT MAX(p.preu) FROM Producte p WHERE p.disponible = true", Double.class) .getSingleResult(); double minPreu = em .createQuery("SELECT MIN(p.preu) FROM Producte p WHERE p.disponible = true", Double.class) .getSingleResult(); System.out.printf("Màx: %.2f€ Mín: %.2f€%n", maxPreu, minPreu);
E2 Actualitzar i eliminar productes Mitjà

Amplia el projecte anterior implementant operacions d'actualització i eliminació sobre la BD de productes.

  1. Aplica un 10% de descompte a tots els productes amb preu superior a 80€ usant una consulta UPDATE JPQL.
  2. Recupera el primer producte amb find(), canvia el seu estoc a 50 i actualitza'l (actualització transparent).
  3. Elimina tots els productes no disponibles (disponible = false) amb una consulta DELETE JPQL.
  4. Mostra la llista final de productes per verificar els canvis.

Solució — Actualitzar i eliminar productes
em.getTransaction().begin(); // 1. Descompte del 10% als productes > 80€ int actualitzats = em .createQuery("UPDATE Producte p SET p.preu = p.preu * 0.9 WHERE p.preu > 80") .executeUpdate(); System.out.println("Productes amb descompte: " + actualitzats); // 2. Actualització transparent (cal refrescar el context) em.clear(); // Netegem el context per veure els valors actualitzats Producte p = em.find(Producte.class, 1L); if (p != null) p.setEstoc(50); // 3. Eliminar no disponibles int eliminats = em .createQuery("DELETE FROM Producte p WHERE p.disponible = false") .executeUpdate(); System.out.println("Eliminats: " + eliminats); em.getTransaction().commit(); // 4. Llistar resultat final em.createQuery("SELECT p FROM Producte p ORDER BY p.preu", Producte.class) .getResultList() .forEach(prod -> System.out.printf("%-20s %.2f€ estoc:%d%n", prod.getNom(), prod.getPreu(), prod.getEstoc()));
E3 @NamedQuery i consultes agregades avançades Avançat

Afegeix @NamedQuery a l'entitat Producte i implementa un informe de resum de l'estoc.

  1. Afegeix dues @NamedQuery a la classe Producte: una per buscar per nom (amb paràmetre :nom) i una altra per buscar per rang de preus (:min i :max).
  2. Usa les named queries per cercar productes que continguin "a" al nom i productes entre 30€ i 90€.
  3. Implementa un informe d'estoc amb JPQL que mostri per cada producte: nom, preu, estoc i el valor total d'estoc (preu * estoc). Ordena per valor total descendent.
Avançat CA 8.4 · 8.7
Solució — @NamedQuery i informe d'estoc
// Definició a la classe Producte @Entity @NamedQueries({ @NamedQuery( name = "Producte.findByNom", query = "SELECT p FROM Producte p WHERE p.nom LIKE :nom" ), @NamedQuery( name = "Producte.findByRangPreu", query = "SELECT p FROM Producte p WHERE p.preu BETWEEN :min AND :max ORDER BY p.preu" ) }) public class Producte implements Serializable { /* ... */ }
// Ús de les named queries List<Producte> ambA = em .createNamedQuery("Producte.findByNom", Producte.class) .setParameter("nom", "%a%") .getResultList(); List<Producte> rang = em .createNamedQuery("Producte.findByRangPreu", Producte.class) .setParameter("min", 30.0).setParameter("max", 90.0) .getResultList(); // Informe d'estoc amb valor total List<Object[]> informe = em .createQuery( "SELECT p.nom, p.preu, p.estoc, p.preu * p.estoc " + "FROM Producte p ORDER BY p.preu * p.estoc DESC", Object[].class) .getResultList(); System.out.printf("%-20s %8s %6s %10s%n", "Producte", "Preu", "Estoc", "Valor"); for (Object[] row : informe) { System.out.printf("%-20s %7.2f€ %6d %9.2f€%n", row[0], row[1], row[2], row[3]); }
