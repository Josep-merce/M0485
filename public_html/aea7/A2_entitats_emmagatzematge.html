1. Classe d'entitat JPA

Una classe d'entitat és una classe Java normal que JPA pot emmagatzemar a la base de dades. Ha de complir tres requisits: estar anotada amb @Entity, implementar Serializable, i tenir un constructor sense paràmetres.

import java.io.Serializable; import javax.persistence.*; @Entity public class Punt implements Serializable { @Id @GeneratedValue private Long id; private double x; private double y; public Punt() {} // Constructor sense paràmetres — obligatori per JPA public Punt(double x, double y) { this.x = x; this.y = y; } public double getX() { return x; } public double getY() { return y; } }
⚠️ Error freqüent — oblidar @GeneratedValue: Si defines @Id però oblides @GeneratedValue i no assignes el valor manualment, tots els objectes tindran id = null i cada nou objecte sobreescriurà l'anterior. Obre l'Explorer: si només queda un objecte a la BD, és aquest l'error.
AnotacióSignificat
@EntityMarca la classe com a persistent. Sense ella, JPA llança PersistenceException.
@IdCamp clau primària. Obligatori per JPA. Identificador únic de cada objecte.
@GeneratedValueLa BD genera automàticament el valor de l'ID. Evita que els objectes es sobreescriguin.

2. Cicle de vida d'un objecte d'entitat

Un objecte d'entitat passa per quatre estats al llarg de la seva vida:

EstatDescripcióCom s'hi arriba
NewRecién creat amb new. No té ID, no és gestionat per l'EM.new Punt(...)
ManagedAssociat a l'EntityManager. Els canvis es sincronitzen automàticament.em.persist(p) o em.find(...)
DetachedHa estat gestionat però ja no ho és (EM tancat).em.close()
RemovedMarcat per eliminar. S'esborra al commit().em.remove(p)
// NEW: objecte creat però no persistent Punt p = new Punt(3.0, 4.0); em.getTransaction().begin(); em.persist(p); // MANAGED: ara és gestionat per l'EntityManager em.getTransaction().commit(); em.close(); // DETACHED: l'EntityManager s'ha tancat
📌 Actualització transparent: Si modifiques un objecte en estat managed dins d'una transacció activa, JPA detecta el canvi automàticament i actualitza la BD al fer commit(). No cal cridar cap mètode d'actualització explícit.

3. Emmagatzematge massiu

Quan cal emmagatzemar milers d'objectes, mantenir-los tots al context de persistència (en memòria) consumeix molta RAM. La solució és buidar periòdicament el context amb flush() + clear():

em.getTransaction().begin(); for (int i = 0; i < 100000; i++) { Punt p = new Punt(i * 0.1, i * 0.2); em.persist(p); if (i % 10000 == 0) { em.flush(); // Envia els canvis a la BD (temporalment) em.clear(); // Buida el context de persistència de memòria } } em.getTransaction().commit();
💡 flush() vs commit(): flush() envia els canvis a la BD però dins la mateixa transacció — si no es fa commit(), es perden. commit() confirma definitivament tots els canvis.

4. Exercicis

E1 Entitat Punt i 10 punts aleatoris Bàsic

Crea la classe d'entitat Punt i un programa que emmagatzemi 10 punts amb coordenades aleatòries a una base de dades ObjectDB.

  1. Crea la classe Punt amb atributs x i y (double), anotada com a @Entity amb @Id @GeneratedValue.
  2. Crea el fitxer persistence.xml incloent la classe Punt a la unitat "PuntsPU".
  3. Al main, persisteix 10 objectes Punt amb Math.random() per a les coordenades.
  4. Obre l'Explorer i verifica que hi ha exactament 10 punts emmagatzemats.
Bàsic CA 8.4 · 8.5
Solució — Entitat Punt i 10 punts aleatoris
// src/META-INF/persistence.xml <persistence version="2.1" xmlns="http://xmlns.jcp.org/xml/ns/persistence"> <persistence-unit name="PuntsPU"> <provider>com.objectdb.jpa.Provider</provider> <class>programa.Punt</class> <properties> <property name="javax.persistence.jdbc.url" value="db/punts.odb"/> </properties> </persistence-unit> </persistence>
// Punt.java import java.io.Serializable; import javax.persistence.*; @Entity public class Punt implements Serializable { @Id @GeneratedValue private Long id; private double x; private double y; public Punt() {} public Punt(double x, double y) { this.x = x; this.y = y; } @Override public String toString() { return "Punt(" + x + ", " + y + ")"; } }
// Main.java import javax.persistence.*; public class Main { public static void main(String[] args) { EntityManagerFactory emf = Persistence.createEntityManagerFactory("PuntsPU"); EntityManager em = emf.createEntityManager(); em.getTransaction().begin(); for (int i = 0; i < 10; i++) { Punt p = new Punt(Math.random() * 100, Math.random() * 100); em.persist(p); System.out.println("Persistit: " + p); } em.getTransaction().commit(); em.close(); emf.close(); System.out.println("10 punts emmagatzemats."); } }
E2 Entitat Producte amb camps variats Mitjà

Crea una entitat Producte amb atributs de tipus variats i desa un conjunt de productes d'una botiga.

  1. Crea la classe Producte amb: Long id (autogenerat), String nom, double preu, int estoc, boolean disponible.
  2. Afegeix el persistence.xml amb una unitat "ProductesPU" que inclogui la classe Producte.
  3. Persisteix 5 productes: Teclat (89.99, 15), Ratólí (34.50, 32), Monitor (249.00, 0), Auriculars (59.95, 8), Webcam (45.00, 0). Marca com a disponible=false els de estoc 0.
  4. Comprova amb l'Explorer que tots els productes apareixen amb els valors correctes.
  5. Executa el programa un segon cop. Quants productes hi ha ara a la BD? Per què?
Mitjà CA 8.4 · 8.5 · 8.6
Solució — Entitat Producte
// src/META-INF/persistence.xml <persistence version="2.1" xmlns="http://xmlns.jcp.org/xml/ns/persistence"> <persistence-unit name="ProductesPU"> <provider>com.objectdb.jpa.Provider</provider> <class>model.Producte</class> <properties> <property name="javax.persistence.jdbc.url" value="db/productes.odb"/> </properties> </persistence-unit> </persistence>
@Entity public class Producte implements Serializable { @Id @GeneratedValue private Long id; private String nom; private double preu; private int estoc; private boolean disponible; public Producte() {} public Producte(String nom, double preu, int estoc) { this.nom = nom; this.preu = preu; this.estoc = estoc; this.disponible = estoc > 0; } // getters... }
em.getTransaction().begin(); em.persist(new Producte("Teclat mecànic", 89.99, 15)); em.persist(new Producte("Ratólí sense fil", 34.50, 32)); em.persist(new Producte("Monitor 24\", 249.00, 0)); em.persist(new Producte("Auriculars", 59.95, 8)); em.persist(new Producte("Webcam HD", 45.00, 0)); em.getTransaction().commit(); // Al segon cop hi haurà 10 productes: @GeneratedValue // genera IDs nous cada vegada i ObjectDB acumula.
E3 Emmagatzematge massiu de 50.000 punts Avançat

Emmagatzema 50.000 objectes Punt usant flush() + clear() cada 5.000 insercions i mesura el temps d'execució.

  1. Reutilitza el projecte de l'E1. Elimina la BD des de l'Explorer abans d'executar.
  2. Insereix 50.000 punts en un bucle. Aplica flush() + clear() cada 5.000 iteracions.
  3. Mesura el temps total d'execució amb System.currentTimeMillis().
  4. Comprova amb l'Explorer que hi ha exactament 50.000 objectes.
  5. Opcional: Prova sense flush/clear per comparar el temps i l'ús de memòria.
Avançat CA 8.6
Solució — Emmagatzematge massiu 50.000 punts
public class Main { public static void main(String[] args) { EntityManagerFactory emf = Persistence.createEntityManagerFactory("PuntsPU"); EntityManager em = emf.createEntityManager(); long inici = System.currentTimeMillis(); em.getTransaction().begin(); for (int i = 0; i < 50000; i++) { em.persist(new Punt(i * 0.01, i * 0.02)); if (i % 5000 == 0 && i > 0) { em.flush(); em.clear(); System.out.println("Processats: " + i); } } em.getTransaction().commit(); long fi = System.currentTimeMillis(); System.out.println("Temps total: " + (fi - inici) + " ms"); em.close(); emf.close(); } }
