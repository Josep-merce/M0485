1. Estructura i components d'una classe Java

Una classe ben dissenyada té una estructura clara i previsible. En Java, la convenció és organitzar els membres en aquest ordre: constants, atributs, constructors, getters/setters i mètodes de negoci. Aquesta organització fa el codi molt més fàcil de llegir i mantenir.

public class NomClasse { // 1. Constants (public static final) public static final double IVA = 0.21; // 2. Atributs (private — principi d'encapsulament) private String nom; private double preu; // 3. Constructors public NomClasse(String nom, double preu) { this.nom = nom; this.preu = preu; } // 4. Getters i setters public String getNom() { return nom; } public void setNom(String nom) { this.nom = nom; } public double getPreu() { return preu; } public void setPreu(double preu) { this.preu = preu; } // 5. Mètodes de negoci public double preuAmbIva() { return preu * (1 + IVA); } // 6. toString (per mostrar l'objecte com a text) @Override public String toString() { return "NomClasse{nom='" + nom + "', preu=" + preu + "}}"; } }

2. Encapsulament: protegir les dades

L'encapsulament consisteix a amagar els atributs d'una classe fent-los private i controlar l'accés des de fora mitjançant mètodes públics (getters i setters). Això permet validar les dades abans d'acceptar-les i canviar la implementació interna sense afectar qui usa la classe.

Sense encapsulament ❌Amb encapsulament ✅
producte.preu = -5; (accepta valors absurds)producte.setPreu(-5); → el setter valida i rebutja
Qualsevol classe pot modificar qualsevol atribut directamentÚnicament els mètodes autoritzats poden modificar l'estat intern
Si canvies el nom de l'atribut, cal canviar-ho a tot el codiEl canvi intern no afecta qui crida el getter/setter
2.1 Classe sense encapsulament vs. amb encapsulament
BàsicPersona sense encapsulament — atributs accessibles des de fora
// ❌ SENSE encapsulament: qualsevol pot modificar els atributs directament public class PersonaMal { String nom; // visible des de fora int edat; // visible des de fora public static void main(String[] args) { PersonaMal p = new PersonaMal(); p.nom = "Laia"; p.edat = -50; // Java ho accepta! No hi ha cap validació System.out.println(p.edat); // -50 } }
BàsicPersona amb encapsulament — atributs privats, accés controlat
// ✅ AMB encapsulament: els atributs són privats, els setters validen public class Persona { private String nom; private int edat; public Persona(String nom, int edat) { this.nom = nom; setEdat(edat); // reutilitzem el setter per validar des del constructor } public String getNom() { return nom; } public void setNom(String nom) { this.nom = nom; } public int getEdat() { return edat; } public void setEdat(int edat) { if (edat < 0 || edat > 150) { System.out.println("Error: edat no vàlida (" + edat + ")"); return; } this.edat = edat; } @Override public String toString() { return nom + " (" + edat + " anys)"; } public static void main(String[] args) { Persona p = new Persona("Laia", 20); System.out.println(p); // Laia (20 anys) p.setEdat(-50); // Error: edat no vàlida (-50) System.out.println(p.getEdat()); // 20 (no ha canviat) } }
✅ Bones pràctiques amb getters i setters: crida el setter des del constructor (com a l'exemple) per no duplicar la lògica de validació. Si un atribut mai ha de canviar un cop creat l'objecte, posa-li getter però no setter — o fes-lo final.

3. Modificadors de visibilitat

Java ofereix quatre nivells de visibilitat per controlar qui pot accedir a un atribut o mètode. Triar el nivell correcte és fonamental per al bon disseny d'una classe.

ModificadorMateixa classeMateix paquetSubclasseQualsevolQuan usar-lo
private✅❌❌❌Atributs d'instància (encapsulament)
(sense modificador)✅✅❌❌Rarament; accés per paquet
protected✅✅✅❌Atributs que han d'heretar les subclasses
public✅✅✅✅Constructors, getters/setters i mètodes de la API pública
💡 Regla d'or: fes els atributs private i els mètodes que han de ser accessibles des de fora, public. Si en el futur necessites que una subclasse accedeixi a un atribut directament, pots canviar a protected. Mai facis els atributs public.

4. Exemple complet: classe Producte ben dissenyada

Veiem com queda una classe completa seguint totes les bones pràctiques: atributs privats, constructors, getters i setters amb validació, mètodes de negoci i toString().

MitjàProducte.java — classe completa amb encapsulament i validació
public class Producte { // Constant de classe public static final double IVA_GENERAL = 0.21; // Atributs privats private String nom; private double preu; private int estoc; // Constructors public Producte(String nom, double preu, int estoc) { this.nom = nom; setPreu(preu); setEstoc(estoc); } public Producte(String nom, double preu) { this(nom, preu, 0); } // Getters i setters public String getNom() { return nom; } public void setNom(String nom) { this.nom = nom; } public double getPreu() { return preu; } public void setPreu(double preu) { if (preu < 0) { System.out.println("Error: preu negatiu"); return; } this.preu = preu; } public int getEstoc() { return estoc; } public void setEstoc(int estoc) { if (estoc < 0) { System.out.println("Error: estoc negatiu"); return; } this.estoc = estoc; } // Mètodes de negoci public double preuAmbIva() { return preu * (1 + IVA_GENERAL); } public boolean teEstoc() { return estoc > 0; } public void vendre(int unitats) { if (unitats > estoc) { System.out.println("Error: no hi ha prou estoc"); return; } estoc -= unitats; System.out.printf("Venut %d × %s. Estoc restant: %d%n", unitats, nom, estoc); } @Override public String toString() { return String.format("Producte{nom='%s', preu=%.2f €, estoc=%d}", nom, preu, estoc); } public static void main(String[] args) { Producte p = new Producte("Cafè mòlt", 3.50, 10); System.out.println(p); System.out.printf("Preu amb IVA: %.2f €%n", p.preuAmbIva()); p.vendre(3); p.vendre(20); // error: no hi ha prou estoc p.setPreu(-1); // error: preu negatiu System.out.println(p); } }

5. El mètode toString()

toString() és un mètode especial heretat de la classe Object (la superclasse de totes les classes Java). Quan imprimeixes un objecte amb System.out.println(obj), Java crida automàticament obj.toString(). Si no el sobreescrius, imprimeix quelcom com Producte@1a2b3c4d — que no és útil per a ningú.

BàsicAmb i sense toString() — la diferència
public class Punt { private double x, y; public Punt(double x, double y) { this.x = x; this.y = y; } public double getX() { return x; } public double getY() { return y; } // Sense aquest mètode, println imprimiria: Punt@6d06d69c @Override public String toString() { return "(" + x + ", " + y + ")"; } public static void main(String[] args) { Punt p1 = new Punt(3, 4); Punt p2 = new Punt(0, 0); System.out.println(p1); // (3.0, 4.0) System.out.println(p2); // (0.0, 0.0) System.out.println("Punt creat: " + p1); // concatenació també crida toString() } }

6. Interacció entre objectes

Els objectes no viuen aïllats: es relacionen entre ells. Un objecte pot tenir un altre objecte com a atribut (composició), o rebre'n un com a paràmetre. Aquesta és la base per construir sistemes complexos a partir de classes senzilles.

MitjàAdreça i Persona — un objecte dins d'un altre
public class Adreca { private String carrer; private String ciutat; private String codiPostal; public Adreca(String carrer, String ciutat, String codiPostal) { this.carrer = carrer; this.ciutat = ciutat; this.codiPostal = codiPostal; } public String getCarrer() { return carrer; } public String getCiutat() { return ciutat; } public String getCodiPostal() { return codiPostal; } @Override public String toString() { return carrer + ", " + codiPostal + " " + ciutat; } } public class Persona { private String nom; private int edat; private Adreca adreca; // un objecte com a atribut public Persona(String nom, int edat, Adreca adreca) { this.nom = nom; this.edat = edat; this.adreca = adreca; } public String getNom() { return nom; } public int getEdat() { return edat; } public Adreca getAdreca() { return adreca; } @Override public String toString() { return nom + " (" + edat + " anys) — " + adreca; } public static void main(String[] args) { Adreca a1 = new Adreca("C/ Balmes 42", "Barcelona", "08007"); Persona p1 = new Persona("Laia", 20, a1); System.out.println(p1); System.out.println("Viu a: " + p1.getAdreca().getCiutat()); } }
📌 En tots els exercicis, els valors estan declarats al codi. Quan arribi l'RA5 podràs llegir-los del teclat amb Scanner.

Ex. 1 Classe Alumne encapsulada Bàsic

Redissenya la classe Alumne de l'A1 seguint ara les bones pràctiques d'encapsulament: atributs privats, getters, setters amb validació i toString().

  • Atributs private: nom (String), edat (int), notaMitjana (double).
  • Constructor complet que usi els setters per validar.
  • setEdat: rebutja valors fora de [0, 120].
  • setNotaMitjana: rebutja valors fora de [0.0, 10.0].
  • toString() que retorni: Alumne{nom='Laia', edat=19, nota=7.80}
  • Prova: crea alumnes vàlids i intenta posar una edat de -5 i una nota d'11.
Bàsic CA: 4.1 · 4.2 · 4.3 · 4.4
✅ Solució
public class Alumne { private String nom; private int edat; private double notaMitjana; public Alumne(String nom, int edat, double notaMitjana) { this.nom = nom; setEdat(edat); setNotaMitjana(notaMitjana); } public String getNom() { return nom; } public void setNom(String nom) { this.nom = nom; } public int getEdat() { return edat; } public void setEdat(int edat) { if (edat < 0 || edat > 120) { System.out.println("Error: edat no vàlida (" + edat + ")"); return; } this.edat = edat; } public double getNotaMitjana() { return notaMitjana; } public void setNotaMitjana(double nota) { if (nota < 0.0 || nota > 10.0) { System.out.println("Error: nota no vàlida (" + nota + ")"); return; } this.notaMitjana = nota; } @Override public String toString() { return String.format("Alumne{nom='%s', edat=%d, nota=%.2f}", nom, edat, notaMitjana); } public static void main(String[] args) { Alumne a1 = new Alumne("Laia", 19, 7.8); Alumne a2 = new Alumne("Marc", -5, 11.0); // edat i nota invàlides System.out.println(a1); System.out.println(a2); } }
Ex. 2 Classe Compte bancari encapsulada Mitjà

Redissenya la classe Compte de l'A1 seguint encapsulament complet. Ara el saldo no ha de tenir setter públic: s'ha de modificar únicament a través dels mètodes ingressar i retirar.

  • Atributs private: titular (String), numeroCom (String), saldo (double).
  • Constructor complet. El saldo inicial ha de ser ≥ 0.
  • Getters per als tres atributs. Cap setter per a saldo — és de lectura.
  • Mètode ingressar(double impo): valida que l'import sigui positiu.
  • Mètode retirar(double impo): valida que l'import sigui positiu i que hi hagi saldo.
  • toString(): Compte{titular='Laia', número='ES12-0049', saldo=350.00 €}
  • Prova: saldo inicial 500 €, ingressa 200 €, retira 350 €, intenta retirar 500 €.
Mitjà CA: 4.1 · 4.2 · 4.3 · 4.4 · 4.5 · 4.6
✅ Solució
public class Compte { private String titular; private String numeroCom; private double saldo; public Compte(String titular, String numeroCom, double saldoInicial) { this.titular = titular; this.numeroCom = numeroCom; this.saldo = (saldoInicial >= 0) ? saldoInicial : 0; } public String getTitular() { return titular; } public String getNumeroCom() { return numeroCom; } public double getSaldo() { return saldo; } // No hi ha setSaldo() — el saldo és de lectura des de fora public void ingressar(double impo) { if (impo <= 0) { System.out.println("Error: import ha de ser positiu"); return; } saldo += impo; System.out.printf("Ingrés %.2f €. Saldo: %.2f €%n", impo, saldo); } public void retirar(double impo) { if (impo <= 0) { System.out.println("Error: import ha de ser positiu"); return; } if (impo > saldo) { System.out.println("Error: saldo insuficient"); return; } saldo -= impo; System.out.printf("Retirada %.2f €. Saldo: %.2f €%n", impo, saldo); } @Override public String toString() { return String.format("Compte{titular='%s', número='%s', saldo=%.2f €}", titular, numeroCom, saldo); } public static void main(String[] args) { Compte c = new Compte("Laia Puig", "ES12-0049", 500.0); System.out.println(c); c.ingressar(200); c.retirar(350); c.retirar(500); System.out.println(c); } }
Ex. 3 Sistema de reserva d'hotel Mitjà

Crea dues classes relacionades: Habitacio i Reserva. Una reserva conté una habitació i un client.

  • Habitacio: atributs numero (int), tipusHabitacio (String: "individual"/"doble"/"suite"), preuPerNit (double), disponible (boolean). Getters/setters. toString().
  • Reserva: atributs client (String), habitacio (Habitacio), nitsEstada (int). Constructor. Getter per tots. Mètode calcularTotal() que retorni habitacio.getPreuPerNit() * nitsEstada. toString() complet.
  • Al main: crea dues habitacions, fes dues reserves i mostra'n el total a pagar.

public class Habitacio { private int numero; private String tipusHabitacio; private double preuPerNit; private boolean disponible; public Habitacio(int numero, String tipus, double preu) { this.numero = numero; this.tipusHabitacio = tipus; this.preuPerNit = preu; this.disponible = true; } public int getNumero() { return numero; } public String getTipusHabitacio() { return tipusHabitacio; } public double getPreuPerNit() { return preuPerNit; } public boolean isDisponible() { return disponible; } public void setDisponible(boolean d) { this.disponible = d; } @Override public String toString() { return String.format("Hab.%d (%s) — %.2f €/nit — %s", numero, tipusHabitacio, preuPerNit, disponible ? "disponible" : "ocupada"); } } public class Reserva { private String client; private Habitacio habitacio; private int nitsEstada; public Reserva(String client, Habitacio habitacio, int nits) { this.client = client; this.habitacio = habitacio; this.nitsEstada = nits; habitacio.setDisponible(false); } public String getClient() { return client; } public Habitacio getHabitacio() { return habitacio; } public int getNitsEstada() { return nitsEstada; } public double calcularTotal() { return habitacio.getPreuPerNit() * nitsEstada; } @Override public String toString() { return String.format("Reserva[%s, %s, %d nits → %.2f €]", client, habitacio.getTipusHabitacio(), nitsEstada, calcularTotal()); } public static void main(String[] args) { Habitacio h1 = new Habitacio(101, "doble", 95.0); Habitacio h2 = new Habitacio(201, "suite", 180.0); Reserva r1 = new Reserva("Laia Puig", h1, 3); Reserva r2 = new Reserva("Marc Vila", h2, 2); System.out.println(r1); System.out.println(r2); System.out.println(h1); // ara apareix com a "ocupada" } }
Ex. 4 Gestió de la biblioteca (adaptat) Avançat

Implementa la classe Llibre amb encapsulament complet. Practica la gestió d'estat intern amb lògica de negoci (préstec) i validació de la valoració.

  • Atributs private: titol, autor (String); anyPublicacio (int); enPrestec (boolean, per defecte false); valoracioMitjana (double); nombreValoracions (int).
  • Constructor: titol, autor, any. La resta d'atributs s'inicialitzen als valors lògics.
  • Getters per a tots. Setters only per a titol i autor.
  • Mètode prestar(): si no està en préstec, el marca com a prestat. Si ja ho està, informa.
  • Mètode retornar(): marca el llibre com a disponible.
  • Mètode afegirValoracio(int nova): valida [1,5], actualitza la mitjana ((valoracioMitjana * nombreValoracions + nova) / (nombreValoracions + 1)).
  • toString() complet. Crea tres llibres, presta'ls i valora'ls.
Avançat CA: 4.1–4.6 · Font: Exercici Biblioteca.pdf (Canela)
✅ Solució
public class Llibre { private String titol; private String autor; private int anyPublicacio; private boolean enPrestec; private double valoracioMitjana; private int nombreValoracions; public Llibre(String titol, String autor, int any) { this.titol = titol; this.autor = autor; this.anyPublicacio = any; this.enPrestec = false; this.valoracioMitjana = 0.0; this.nombreValoracions = 0; } public String getTitol() { return titol; } public void setTitol(String titol) { this.titol = titol; } public String getAutor() { return autor; } public void setAutor(String autor) { this.autor = autor; } public int getAnyPublicacio() { return anyPublicacio; } public boolean isEnPrestec() { return enPrestec; } public double getValoracioMitjana() { return valoracioMitjana; } public int getNombreValoracions() { return nombreValoracions; } public void prestar() { if (enPrestec) { System.out.println("'" + titol + "' ja està prestat"); return; } enPrestec = true; System.out.println("'" + titol + "' prestat correctament"); } public void retornar() { enPrestec = false; System.out.println("'" + titol + "' retornat. Ja disponible."); } public void afegirValoracio(int nova) { if (nova < 1 || nova > 5) { System.out.println("Error: valoració ha d'estar entre 1 i 5"); return; } valoracioMitjana = (valoracioMitjana * nombreValoracions + nova) / (nombreValoracions + 1); nombreValoracions++; } @Override public String toString() { return String.format("'%s' (%s, %d) — ★%.1f (%d val.) — %s", titol, autor, anyPublicacio, valoracioMitjana, nombreValoracions, enPrestec ? "prestat" : "disponible"); } public static void main(String[] args) { Llibre l1 = new Llibre("La metamorfosi", "Kafka", 1915); Llibre l2 = new Llibre("Solitud", "Víctor Català", 1905); l1.prestar(); l1.prestar(); // ja prestat l1.afegirValoracio(5); l1.afegirValoracio(4); l2.afegirValoracio(6); // error: fora de rang System.out.println(l1); System.out.println(l2); } }
