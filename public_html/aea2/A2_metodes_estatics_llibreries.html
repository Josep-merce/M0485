1. Mètodes estàtics: crides sense crear un objecte

Un mètode estàtic pertany a la classe, no a un objecte concret. Pots cridar-lo directament amb el nom de la classe, sense haver de fer cap new. S'indica amb la paraula clau static a la signatura.

Mètode d'instànciaMètode estàtic
Pertany a un objecte concretPertany a la classe
Cal crear l'objecte primer: new Classe()Es crida directament: Classe.metode()
Pot accedir als atributs de l'objecteNo pot accedir als atributs d'instància
Exemple: p1.mostrarInfo()Exemple: Math.sqrt(25)
1.1 Primer mètode estàtic propi
BàsicMètodes estàtics d'utilitat a la classe Calculadora
public class Calculadora { // Mètode estàtic: no necessita objecte, pertany a la classe public static double sumar(double a, double b) { return a + b; } public static double restar(double a, double b) { return a - b; } public static double multiplicar(double a, double b) { return a * b; } public static double dividir(double a, double b) { if (b == 0) { System.out.println("Error: divisió per zero"); return 0; } return a / b; } public static void main(String[] args) { // Es criden amb el nom de la classe, sense new System.out.println(Calculadora.sumar(10, 4)); // 14.0 System.out.println(Calculadora.restar(10, 4)); // 6.0 System.out.println(Calculadora.multiplicar(3, 7)); // 21.0 System.out.println(Calculadora.dividir(15, 4)); // 3.75 } }
💡 Quan usar static? Quan el mètode no depèn de cap atribut d'un objecte concret: càlculs matemàtics, utilitats de text, conversions, validacions. Si el mètode necessita accedir a this o als atributs de la classe, no pot ser estàtic.
1.2 Mètodes estàtics cridant-se entre ells
MitjàClasse Utils — mètodes que es composen entre ells
public class Utils { public static boolean esParell(int n) { return n % 2 == 0; } public static boolean esPrimer(int n) { if (n <= 1) return false; for (int i = 2; i < n; i++) { if (n % i == 0) return false; } return true; } public static int sumaDigits(int n) { int suma = 0; n = Math.abs(n); // treballem amb valor absolut while (n > 0) { suma += n % 10; n /= 10; } return suma; } public static void main(String[] args) { System.out.println(esParell(8)); // true System.out.println(esPrimer(17)); // true System.out.println(esPrimer(12)); // false System.out.println(sumaDigits(347)); // 14 (3+4+7) } }
✅ Dins del main (o d'un altre mètode estàtic de la mateixa classe), pots cridar els mètodes estàtics propis sense el nom de la classe davant: esPrimer(17) en comptes de Utils.esPrimer(17). Des d'una altra classe, cal escriure el nom complet: Utils.esPrimer(17).

2. Pas de paràmetres

Els paràmetres permeten passar dades a un mètode perquè treballi amb elles. Es declaren entre parèntesis a la signatura del mètode i actuen com a variables locals dins del cos. Java passa els tipus primitius per valor: el mètode treballa amb una còpia, i la variable original no canvia.

2.1 Paràmetres de tipus primitiu: pas per valor
BàsicEl mètode rep una còpia — la variable original no canvia
public class PassPerValor { public static void duplicar(int n) { n = n * 2; // modifiquem la còpia local, no l'original System.out.println("Dins del mètode: " + n); } public static void main(String[] args) { int valor = 5; duplicar(valor); System.out.println("Fora del mètode: " + valor); // continua sent 5 } }
⚠️ Pas per valor: quan passes un int, double, boolean o qualsevol altre tipus primitiu, el mètode rep una còpia. Modificar-la dins del mètode no afecta la variable original. Per retornar un resultat cal usar return.
2.2 Múltiples paràmetres i crida en cadena
MitjàClasse Geometria — mètodes estàtics amb múltiples paràmetres
public class Geometria { public static double areaRectangle(double base, double alcada) { return base * alcada; } public static double areaCircle(double radi) { return Math.PI * radi * radi; } public static double hipotenusa(double a, double b) { return Math.sqrt(a * a + b * b); } public static void mostrarResum(double base, double alcada, double radi) { System.out.printf("Àrea rectangle (%.1f x %.1f): %.2f%n", base, alcada, areaRectangle(base, alcada)); System.out.printf("Àrea cercle (r=%.1f): %.2f%n", radi, areaCircle(radi)); System.out.printf("Hipotenusa (%.1f, %.1f): %.2f%n", base, alcada, hipotenusa(base, alcada)); } public static void main(String[] args) { mostrarResum(5.0, 3.0, 4.0); } }

3. Sobrecàrrega de mètodes i constructors

La sobrecàrrega (overloading) permet definir diversos mètodes (o constructors) amb el mateix nom però diferent número o tipus de paràmetres. Java tria automàticament quin usar en funció dels arguments de la crida. Això fa el codi més intuïtiu: no cal inventar noms com sumar2 o sumarDouble.

3.1 Sobrecàrrega de mètodes
BàsicMètode imprimir sobrecarregat per a diversos tipus
public class Impressor { public static void imprimir(int n) { System.out.println("Enter: " + n); } public static void imprimir(double d) { System.out.println("Decimal: " + d); } public static void imprimir(String s) { System.out.println("Text: " + s); } public static void imprimir(int n, String etiqueta) { System.out.println(etiqueta + ": " + n); } public static void main(String[] args) { imprimir(42); // Enter: 42 imprimir(3.14); // Decimal: 3.14 imprimir("Hola"); // Text: Hola imprimir(100, "Puntuació"); // Puntuació: 100 } }
3.2 Sobrecàrrega de constructors
MitjàClasse Producte — tres constructors per a tres casos d'ús
public class Producte { String nom; double preu; int estoc; // Constructor 1: nom i preu (estoc per defecte 0) Producte(String nom, double preu) { this.nom = nom; this.preu = preu; this.estoc = 0; } // Constructor 2: nom, preu i estoc Producte(String nom, double preu, int estoc) { this.nom = nom; this.preu = preu; this.estoc = estoc; } // Constructor 3: crida l'anterior amb this() per evitar duplicació Producte(String nom, double preu, int estoc, double descompte) { this(nom, preu * (1 - descompte), estoc); // reutilitza el constructor 2 } void mostrar() { System.out.println(nom + " | " + preu + " € | estoc: " + estoc); } public static void main(String[] args) { Producte p1 = new Producte("Arròs", 1.80); // estoc = 0 Producte p2 = new Producte("Oli", 4.50, 30); // estoc = 30 Producte p3 = new Producte("Sucre", 2.00, 50, 0.10); // 10% descompte p1.mostrar(); // Arròs | 1.8 € | estoc: 0 p2.mostrar(); // Oli | 4.5 € | estoc: 30 p3.mostrar(); // Sucre | 1.8 € | estoc: 50 } }
💡 this() dins d'un constructor: un constructor pot cridar un altre constructor de la mateixa classe amb this(...). Evita duplicar codi d'inicialització. Ha de ser sempre la primera instrucció del constructor.

4. Llibreries predefinides: Math, Arrays i String

Java inclou milers de classes predefinides agrupades en paquets. Les que veurem aquí estan al paquet java.lang (disponible sempre sense import) o a java.util (cal importar). Totes ofereixen mètodes estàtics molt útils per no reinventar la roda.

4.1 La classe Math
BàsicPrincipals mètodes de Math
public class ExempleMath { public static void main(String[] args) { // Potència i arrel System.out.println(Math.pow(2, 10)); // 1024.0 (2^10) System.out.println(Math.sqrt(144)); // 12.0 // Arrodoniment System.out.println(Math.round(4.6)); // 5 System.out.println(Math.floor(4.9)); // 4.0 (cap avall) System.out.println(Math.ceil(4.1)); // 5.0 (cap amunt) // Valor absolut i max/min System.out.println(Math.abs(-15)); // 15 System.out.println(Math.max(8, 13)); // 13 System.out.println(Math.min(8, 13)); // 8 // Aleatori entre 0.0 i 1.0 (exclòs) System.out.println(Math.random()); // Rang personalitzat [min, max]: (int)(Math.random() * (max-min+1)) + min int dau = (int)(Math.random() * 6) + 1; // [1, 6] System.out.println("Dau: " + dau); // Constants System.out.println(Math.PI); // 3.141592653589793 System.out.println(Math.E); // 2.718281828459045 } }
4.2 La classe Arrays
MitjàOperacions habituals sobre arrays amb java.util.Arrays
import java.util.Arrays; public class ExempleArrays { public static void main(String[] args) { int[] notes = {7, 3, 9, 5, 1, 8}; // Ordenar l'array (modifica l'original) Arrays.sort(notes); System.out.println(Arrays.toString(notes)); // [1, 3, 5, 7, 8, 9] // Buscar un element (l'array ha d'estar ordenat) int pos = Arrays.binarySearch(notes, 7); System.out.println("Posició del 7: " + pos); // 3 // Copiar un array (còpia independent) int[] copia = Arrays.copyOf(notes, notes.length); copia[0] = 99; System.out.println(Arrays.toString(notes)); // l'original no canvia System.out.println(Arrays.toString(copia)); // [99, 3, 5, 7, 8, 9] // Omplir amb un valor int[] zeros = new int[5]; Arrays.fill(zeros, -1); System.out.println(Arrays.toString(zeros)); // [-1, -1, -1, -1, -1] // Comparar dos arrays int[] a = {1, 2, 3}; int[] b = {1, 2, 3}; System.out.println(Arrays.equals(a, b)); // true (== donaria false!) } }
⚠️ Mai comparar arrays amb ==: a == b compara si apunten al mateix lloc de memòria, no si el contingut és igual. Usa sempre Arrays.equals(a, b) per comparar el contingut. El mateix passa amb els Strings: usa .equals().
4.3 Mètodes estàtics útils de String
MitjàConversions entre String i tipus primitius
public class ConversionsString { public static void main(String[] args) { // De String a número (mètodes estàtics de les classes embolcall) int edat = Integer.parseInt("25"); double preu = Double.parseDouble("4.99"); boolean actiu = Boolean.parseBoolean("true"); System.out.println(edat + 1); // 26 (ja és enter, no text) System.out.println(preu * 2); // 9.98 System.out.println(actiu); // true // De número a String String s1 = String.valueOf(42); // "42" String s2 = String.valueOf(3.14); // "3.14" String s3 = Integer.toString(255, 16); // "ff" (base 16) System.out.println(s1 + s2); // "423.14" (concatenació, no suma!) System.out.println(s3); // ff // Format amb printf / String.format String resultat = String.format("Preu: %.2f €, Quantitat: %d", preu, (int)edat); System.out.println(resultat); // Preu: 4.99 €, Quantitat: 25 } }
💡 Classes embolcall (Wrapper classes): cada tipus primitiu té una classe corresponent: int → Integer, double → Double, boolean → Boolean, etc. Aquestes classes ofereixen mètodes estàtics molt útils com parseInt, MAX_VALUE, MIN_VALUE o toBinaryString.

5. Exercicis pràctics

Exercicis individuals ordenats de menor a major dificultat. Pensa primer quins paràmetres necessita cada mètode i quin tipus ha de retornar. Prem el botó per veure la solució quan hagis acabat.

📌 Tots els valors estan declarats al codi. Quan arribi l'RA5 podràs llegir-los del teclat amb Scanner.
Ex. 1 Classe Conversor d'unitats Bàsic

Crea una classe Conversor amb quatre mètodes estàtics per convertir entre unitats habituals.

  • kmAMilles(double km) → retorna km × 0.621371
  • celsiusAFahrenheit(double c) → retorna c × 9/5 + 32
  • eurosADolars(double euros, double canvi) → retorna euros × canvi
  • kgALliures(double kg) → retorna kg × 2.20462
  • Prova: 100 km, 37 °C, 50 € (canvi 1.08), 70 kg.
Bàsic CA: 2.5 · 2.6
✅ Solució
public class Conversor { public static double kmAMilles(double km) { return km * 0.621371; } public static double celsiusAFahrenheit(double c) { return c * 9.0 / 5.0 + 32; } public static double eurosADolars(double euros, double canvi) { return euros * canvi; } public static double kgALliures(double kg) { return kg * 2.20462; } public static void main(String[] args) { System.out.printf("100 km = %.2f milles%n", kmAMilles(100)); System.out.printf("37 °C = %.2f °F%n", celsiusAFahrenheit(37)); System.out.printf("50 € = %.2f $%n", eurosADolars(50, 1.08)); System.out.printf("70 kg = %.2f lliures%n", kgALliures(70)); } }
Ex. 2 Classe Producte amb tres constructors Bàsic

Crea una classe Producte amb sobrecàrrega de constructors per adaptar-se a diferents situacions: quan tens totes les dades, quan no saps l'estoc, i quan el producte és nou i no té preu encara.

  • Atributs: nom (String), preu (double), estoc (int).
  • Constructor 1: nom, preu, estoc (tots els valors).
  • Constructor 2: nom, preu (estoc = 0 per defecte).
  • Constructor 3: nom sol (preu = 0.0, estoc = 0).
  • Mètode mostrar() que imprimeixi les tres dades.
  • Crea un objecte de cada constructor i mostra'n la informació.
Bàsic CA: 2.8
✅ Solució
public class Producte { String nom; double preu; int estoc; Producte(String nom, double preu, int estoc) { this.nom = nom; this.preu = preu; this.estoc = estoc; } Producte(String nom, double preu) { this(nom, preu, 0); // reutilitza el constructor complet } Producte(String nom) { this(nom, 0.0, 0); } void mostrar() { System.out.printf("%-20s | %6.2f € | estoc: %d%n", nom, preu, estoc); } public static void main(String[] args) { Producte p1 = new Producte("Cafè mòlt", 3.50, 25); Producte p2 = new Producte("Llet sencera", 0.85); Producte p3 = new Producte("Nou producte"); p1.mostrar(); p2.mostrar(); p3.mostrar(); } }
Ex. 3 Classe Estadística amb Math Mitjà

Crea una classe Estadistica amb mètodes estàtics que calculin estadístiques bàsiques sobre un array de notes. Usa Math per als càlculs.

  • maxim(double[] v): retorna el valor màxim de l'array.
  • minim(double[] v): retorna el valor mínim.
  • mitjana(double[] v): retorna la mitjana aritmètica.
  • arrodonir(double v, int decimals): retorna el valor arrodonit al nombre de decimals indicat. Pista: Math.round(v * 10^d) / 10^d.
  • Prova amb: {6.5, 8.0, 4.75, 9.0, 7.25, 5.5}. Mostra màxim, mínim i mitjana arrodonida a 2 decimals.
Mitjà CA: 2.5 · 2.6 · 2.7
✅ Solució
public class Estadistica { public static double maxim(double[] v) { double max = v[0]; for (double x : v) { max = Math.max(max, x); } return max; } public static double minim(double[] v) { double min = v[0]; for (double x : v) { min = Math.min(min, x); } return min; } public static double mitjana(double[] v) { double suma = 0; for (double x : v) suma += x; return suma / v.length; } public static double arrodonir(double v, int decimals) { double factor = Math.pow(10, decimals); return Math.round(v * factor) / factor; } public static void main(String[] args) { double[] notes = {6.5, 8.0, 4.75, 9.0, 7.25, 5.5}; System.out.println("Màxim: " + maxim(notes)); System.out.println("Mínim: " + minim(notes)); System.out.println("Mitjana: " + arrodonir(mitjana(notes), 2)); } }
Ex. 4 Caixa registradora Avançat

Dissenya una classe CaixaRegistradora que gestioni les vendes d'una botiga. Ha de tenir un constructor sobrecarregat i mètodes estàtics de càlcul.

  • Atributs: nomBotiga (String), tipusIva (double), totalVendes (double, inicialitzat a 0).
  • Constructor 1: nom i IVA (p.ex. 0.21 per 21%).
  • Constructor 2: nom sol (IVA = 0.21 per defecte).
  • Mètode d'instància vendre(double preu): afegeix el preu a totalVendes i imprimeix el tiquet (preu base, IVA i total amb IVA).
  • Mètode estàtic calcularIva(double base, double tipus): retorna l'import de l'IVA.
  • Mètode d'instància tancament(): mostra el resum del dia: nom botiga, total sense IVA, total IVA i total amb IVA.
  • Prova: crea una caixa, fes 3 vendes (15.00 €, 8.50 €, 32.00 €) i tanca.
Avançat CA: 2.5 · 2.6 · 2.7 · 2.8
✅ Solució
public class CaixaRegistradora { String nomBotiga; double tipusIva; double totalVendes; CaixaRegistradora(String nomBotiga, double tipusIva) { this.nomBotiga = nomBotiga; this.tipusIva = tipusIva; this.totalVendes = 0; } CaixaRegistradora(String nomBotiga) { this(nomBotiga, 0.21); // IVA general per defecte } public static double calcularIva(double base, double tipus) { return base * tipus; } void vendre(double preu) { totalVendes += preu; double iva = calcularIva(preu, tipusIva); double total = preu + iva; System.out.println("--- Tiquet ---"); System.out.printf(" Base: %6.2f €%n", preu); System.out.printf(" IVA: %6.2f €%n", iva); System.out.printf(" TOTAL: %6.2f €%n", total); } void tancament() { double ivaTotal = calcularIva(totalVendes, tipusIva); double totalAmbIva = totalVendes + ivaTotal; System.out.println("\n====== Tancament de caixa ======"); System.out.println("Botiga: " + nomBotiga); System.out.printf("Total sense IVA: %8.2f €%n", totalVendes); System.out.printf("Total IVA (%.0f%%): %7.2f €%n", tipusIva * 100, ivaTotal); System.out.printf("TOTAL AMB IVA: %8.2f €%n", totalAmbIva); } public static void main(String[] args) { CaixaRegistradora caixa = new CaixaRegistradora("Mercat La Mercè"); caixa.vendre(15.00); caixa.vendre(8.50); caixa.vendre(32.00); caixa.tancament(); } }
