1. Què és la Programació Orientada a Objectes?

La Programació Orientada a Objectes (POO) és una manera d'organitzar el codi on tot gira al voltant d'objectes: entitats que combinen dades (el que saben) i comportament (el que poden fer). Va néixer per resoldre un problema real: els programes creixien tant que eren impossibles de mantenir.

PrincipiQuè significaExemple del dia a dia
EncapsulamentAgrupar dades i mètodes en una sola unitat, ocultant els detalls internsCondueixes un cotxe girant el volant, sense saber com funciona el motor per dins
HerènciaUna classe pot aprofitar els atributs i mètodes d'una altraUn GatDomèstic és un Gat, que és un Animal: comparteixen característiques
PolimorfismeEl mateix mètode pot comportar-se diferent segons l'objecte que el cridaEl mètode ferSo() sona diferent si l'executa un gat o un gos
AbstraccióModelar només les característiques rellevants per al problemaUn compte bancari: importa el saldo i les operacions, no el servidor on s'emmagatzema
💡 Java és orientat a objectes per disseny: gairebé tot el codi s'escriu dins de classes. Fins i tot el main() que has estat usant fins ara és un mètode dins d'una classe. A partir d'ara entendràs per què.

2. Classe i objecte: el motlle i les peces

Una classe és el motlle o plantilla: defineix com seran els objectes d'aquell tipus. Un objecte és una instància concreta d'aquella classe, amb els seus propis valors. Pots crear tants objectes com vulguis a partir d'una sola classe, igual que amb un motlle de galetes en pots fer centenars, totes amb la mateixa forma però cadascuna independent.

Classe (el motlle)Objecte (la peça)
Qué ésPlantilla abstracta que existeix al codi fontEntitat concreta que existeix en memòria durant l'execució
ExempleProductep1 → Pa de motlle, 1,20 €
QuantitatUna sola definicióTantes instàncies com calgui
Paraula clau Javaclass Producte { ... }new Producte(...)
2.1 Primera classe Java
BàsicClasse Producte amb atributs i un mètode
// Fitxer: Producte.java public class Producte { // Atributs: les dades que "sap" cada objecte String nom; double preu; int estoc; // Mètode: una acció que pot fer l'objecte void mostrarInfo() { System.out.println("Producte: " + nom); System.out.println("Preu: " + preu + " €"); System.out.println("Estoc: " + estoc + " unitats"); } }
BàsicCrear i usar objectes de la classe Producte
// Fitxer: AppBotiga.java public class AppBotiga { public static void main(String[] args) { // new Producte() crea un objecte nou a memòria Producte p1 = new Producte(); p1.nom = "Pa de motlle"; p1.preu = 1.20; p1.estoc = 50; Producte p2 = new Producte(); p2.nom = "Llet sencera"; p2.preu = 0.85; p2.estoc = 120; // Cada objecte crida el seu propi mètode p1.mostrarInfo(); System.out.println("---"); p2.mostrarInfo(); } }
✅ Convenció de noms a Java: les classes comencen amb majúscula (Producte, AppBotiga). Els objectes i variables amb minúscula (p1, estoc). Els mètodes també amb minúscula en forma verbal (mostrarInfo()). Seguir les convencions fa el codi molt més llegible per a tothom.

3. Mètodes: el comportament dels objectes

Un mètode és un bloc de codi amb nom que defineix una acció que pot fer un objecte. Evita repetir codi, organitza el programa i permet modificar o consultar l'estat de l'objecte des de fora.

Tipus de mètodeRetorna alguna cosa?Rep paràmetres?Exemple
Sense retorn ni paràmetresNo (void)Novoid mostrarInfo()
Amb paràmetres, sense retornNo (void)Sívoid aplicarDescompte(double pct)
Sense paràmetres, amb retornSíNodouble getPreu()
Amb paràmetres i retornSíSídouble calcularTotal(int quantitat)
3.1 Mètodes que retornen un valor
MitjàClasse Producte amb mètodes que calculen i retornen dades
public class Producte { String nom; double preu; int estoc; // Retorna el preu amb IVA del 21% double preuAmbIva() { return preu * 1.21; } // Retorna true si queden unitats disponibles boolean teEstoc() { return estoc > 0; } // Calcula el cost total per a una quantitat donada double calcularTotal(int quantitat) { return preu * quantitat; } public static void main(String[] args) { Producte cafe = new Producte(); cafe.nom = "Cafè mòlt"; cafe.preu = 3.50; cafe.estoc = 25; System.out.println("Preu amb IVA: " + cafe.preuAmbIva() + " €"); System.out.println("Hi ha estoc? " + cafe.teEstoc()); System.out.println("Total 4 unitats: " + cafe.calcularTotal(4) + " €"); } }
⚠️ Errors habituals amb mètodes: oblidar el return en un mètode que no és void genera un error de compilació. Cridar mostrarInfo() sol (sense l'objecte davant) no funciona tret que sigui un mètode estàtic — sempre cal escriure p1.mostrarInfo().

4. Constructors: inicialitzar objectes des del primer moment

Un constructor és un mètode especial que s'executa automàticament quan creem un objecte amb new. El seu objectiu és inicialitzar els atributs a valors correctes des del primer instant, evitant objectes en un estat incoherent (per exemple, un producte sense preu).

Característiques clau:

  • Té el mateix nom que la classe (amb majúscula).
  • No té tipus de retorn (ni tan sols void).
  • S'executa una sola vegada, en el moment de fer new.
  • Si no en defineixes cap, Java n'afegeix un de buit automàticament.
4.1 Constructor per defecte i constructor parametritzat
BàsicClasse Alumne — dos constructors per a dos casos d'ús
public class Alumne { String nom; int edat; double notaMitjana; // Constructor per defecte: inicialitza amb valors "buits" Alumne() { nom = "Desconegut"; edat = 0; notaMitjana = 0.0; } // Constructor parametritzat: rep els valors des de fora Alumne(String nom, int edat, double notaMitjana) { // "this" distingeix l'atribut de la classe del paràmetre this.nom = nom; this.edat = edat; this.notaMitjana = notaMitjana; } void mostrar() { System.out.println(nom + " (" + edat + " anys) — Nota: " + notaMitjana); } public static void main(String[] args) { Alumne a1 = new Alumne(); // constructor per defecte Alumne a2 = new Alumne("Laia", 19, 7.8); // constructor parametritzat Alumne a3 = new Alumne("Marc", 20, 6.2); a1.mostrar(); // Desconegut (0 anys) — Nota: 0.0 a2.mostrar(); // Laia (19 anys) — Nota: 7.8 a3.mostrar(); // Marc (20 anys) — Nota: 6.2 } }
💡 La paraula clau this: quan el nom d'un paràmetre coincideix amb el d'un atribut (tots dos es diuen nom), Java no sap a quin et refereixes. this.nom és l'atribut de l'objecte actual; nom sol és el paràmetre. Sense this, l'atribut mai s'actualitzaria i l'objecte quedaria amb el valor inicial per defecte.
4.2 Exemple complet: classe Temperatura
MitjàConstructor + mètodes de conversió d'unitats
public class Temperatura { double grausCelsius; Temperatura(double grausCelsius) { this.grausCelsius = grausCelsius; } double aFahrenheit() { return grausCelsius * 9.0 / 5.0 + 32; } double aKelvin() { return grausCelsius + 273.15; } void mostrar() { System.out.println(grausCelsius + " °C = " + aFahrenheit() + " °F = " + aKelvin() + " K"); } public static void main(String[] args) { Temperatura t1 = new Temperatura(0); Temperatura t2 = new Temperatura(100); Temperatura t3 = new Temperatura(37); // temperatura corporal t1.mostrar(); // 0.0 °C = 32.0 °F = 273.15 K t2.mostrar(); // 100.0 °C = 212.0 °F = 373.15 K t3.mostrar(); // 37.0 °C = 98.6 °F = 310.15 K } }

5. Instanciar objectes de classes predefinides de Java

Java inclou centenars de classes llestes per usar. Treballar-hi és exactament igual que amb les nostres pròpies: creem un objecte amb new i cridem els seus mètodes amb el punt (.). Les més habituals al principi són String i Random.

5.1 La classe String i els seus mètodes
BàsicPrincipals mètodes de String que usaràs sovint
public class ExempleString { public static void main(String[] args) { String nom = "Institut La Mercè"; // Longitud i accés a caràcters System.out.println(nom.length()); // 17 System.out.println(nom.charAt(0)); // I System.out.println(nom.toUpperCase()); // INSTITUT LA MERCÈ // Cerca i comprovació System.out.println(nom.contains("Mercè")); // true System.out.println(nom.startsWith("Ins")); // true System.out.println(nom.indexOf("La")); // 9 // Transformació System.out.println(nom.replace("Mercè", "Merce")); // Institut La Merce System.out.println(nom.substring(0, 8)); // Institut // Comparació — sempre amb equals(), MAI amb == String a = "hola"; String b = "hola"; System.out.println(a.equals(b)); // true System.out.println(a.equalsIgnoreCase("HOLA")); // true } }
⚠️ Comparar Strings amb == no funciona: == compara si les dues variables apunten al mateix lloc a memòria, no si el text és igual. Per comparar el contingut, usa sempre .equals(). Oblidar-ho és un dels errors més habituals en Java i pot generar resultats inesperats.

5.3 Exemple integrador: classe Persona amb Random
AvançatClasse Persona amb constructor, mètodes i generació aleatòria
import java.util.Random; public class Persona { String nom; int edat; String ciutat; Persona(String nom, int edat, String ciutat) { this.nom = nom; this.edat = edat; this.ciutat = ciutat; } String salutacio() { return "Hola! Soc " + nom + ", tinc " + edat + " anys i visc a " + ciutat + "."; } boolean esMajorEdat() { return edat >= 18; } // Mètode estàtic: no necessita un objecte existent per cridar-lo static Persona generarAleatoria() { Random r = new Random(); String[] noms = {"Laia", "Marc", "Júlia", "Pol", "Neus"}; String[] citats = {"Barcelona", "Girona", "Lleida", "Tarragona"}; String nom = noms[r.nextInt(noms.length)]; int edat = r.nextInt(30) + 16; // [16, 45] String ciutat = citats[r.nextInt(citats.length)]; return new Persona(nom, edat, ciutat); } public static void main(String[] args) { Persona p1 = new Persona("Laia", 20, "Barcelona"); System.out.println(p1.salutacio()); System.out.println("Major d'edat: " + p1.esMajorEdat()); System.out.println("\n--- Persones aleatòries ---"); for (int i = 0; i < 3; i++) { Persona p = Persona.generarAleatoria(); // cridem mètode estàtic System.out.println(p.salutacio()); } } }

Ex. 1 Classe Cotxe Bàsic

Crea una classe Cotxe que representi un vehicle. Cada cotxe ha de guardar la marca, el model i l'any de fabricació. Afegeix un constructor parametritzat i un mètode per mostrar la informació per pantalla.

  • Atributs: marca (String), model (String), any (int).
  • Constructor parametritzat que inicialitzi els tres atributs amb this.
  • Mètode mostrarInfo() que imprimeixi: Cotxe: Seat Ibiza (2019)
  • Al main, crea tres cotxes: Seat Ibiza 2019, Toyota Yaris 2022, Volkswagen Golf 2020, i mostra'n la informació.
Bàsic CA: 2.1 · 2.2 · 2.9
✅ Solució
public class Cotxe { String marca; String model; int any; Cotxe(String marca, String model, int any) { this.marca = marca; this.model = model; this.any = any; } void mostrarInfo() { System.out.println("Cotxe: " + marca + " " + model + " (" + any + ")"); } public static void main(String[] args) { Cotxe c1 = new Cotxe("Seat", "Ibiza", 2019); Cotxe c2 = new Cotxe("Toyota", "Yaris", 2022); Cotxe c3 = new Cotxe("Volkswagen", "Golf", 2020); c1.mostrarInfo(); c2.mostrarInfo(); c3.mostrarInfo(); } }
Ex. 2 Classe Rectangle — àrea i perímetre Bàsic

Crea una classe Rectangle que guardi les mides d'un rectangle i sàpiga calcular la seva àrea i el seu perímetre. Practica mètodes amb retorn.

  • Atributs: base (double), alcada (double).
  • Constructor parametritzat.
  • Mètode area() que retorni base * alcada.
  • Mètode perimetre() que retorni 2 * (base + alcada).
  • Mètode mostrar() que imprimeixi base, alçada, àrea i perímetre.
  • Prova amb tres rectangles: 5×3, 10×4 i 7×7 (quadrat).
Bàsic CA: 2.2 · 2.3 · 2.4
✅ Solució
public class Rectangle { double base; double alcada; Rectangle(double base, double alcada) { this.base = base; this.alcada = alcada; } double area() { return base * alcada; } double perimetre() { return 2 * (base + alcada); } void mostrar() { System.out.println("Base: " + base + " Alçada: " + alcada + " Àrea: " + area() + " Perímetre: " + perimetre()); } public static void main(String[] args) { Rectangle r1 = new Rectangle(5, 3); Rectangle r2 = new Rectangle(10, 4); Rectangle r3 = new Rectangle(7, 7); r1.mostrar(); r2.mostrar(); r3.mostrar(); } }
Ex. 3 Compte bancari Mitjà

Crea una classe Compte que simuli les operacions bàsiques d'un compte bancari. El compte ha de permetre ingressar i retirar diners, controlant que les operacions siguin vàlides.

  • Atributs: titular (String), numeroCom (String), saldo (double).
  • Constructor que rep titular, número de compte i saldo inicial.
  • Mètode ingressar(double impo): suma l'import al saldo. Rebutja imports negatius o zero.
  • Mètode retirar(double impo): resta l'import. Si no hi ha saldo suficient, imprimeix un missatge d'error sense modificar res.
  • Mètode mostrarEstat(): mostra titular, número de compte i saldo.
  • Prova: saldo inicial 500 €, ingressa 200 €, retira 350 €, intenta retirar 500 € (ha de mostrar error).
Mitjà CA: 2.2 · 2.3 · 2.4 · 2.9
✅ Solució
public class Compte { String titular; String numeroCom; double saldo; Compte(String titular, String numeroCom, double saldo) { this.titular = titular; this.numeroCom = numeroCom; this.saldo = saldo; } void ingressar(double impo) { if (impo <= 0) { System.out.println("Error: l'import ha de ser positiu."); return; } saldo += impo; System.out.printf("Ingrés de %.2f €. Nou saldo: %.2f €%n", impo, saldo); } void retirar(double impo) { if (impo <= 0) { System.out.println("Error: l'import ha de ser positiu."); return; } if (impo > saldo) { System.out.println("Error: saldo insuficient. Saldo actual: " + saldo + " €"); return; } saldo -= impo; System.out.printf("Retirada de %.2f €. Nou saldo: %.2f €%n", impo, saldo); } void mostrarEstat() { System.out.println("--- Estat del compte ---"); System.out.println("Titular: " + titular); System.out.println("Número: " + numeroCom); System.out.printf("Saldo: %.2f €%n", saldo); } public static void main(String[] args) { Compte c = new Compte("Laia Puig", "ES12-0049-1234", 500.0); c.mostrarEstat(); c.ingressar(200.0); c.retirar(350.0); c.retirar(500.0); // saldo insuficient c.mostrarEstat(); } }
Ex. 4 Joc de daus: dos jugadors Avançat

Dissenya una classe Jugador per a un joc de daus on dos jugadors competeixen tirant un dau 5 vegades cadascun. Guanya qui acumuli més punts al final.

  • Atributs: nom (String), punts (int inicialitzat a 0), rnd (Random declarat a la classe).
  • Constructor que rep el nom.
  • Mètode tirarDau(): genera un valor entre 1 i 6 amb Random, l'afegeix als punts i el retorna.
  • Mètode mostrarPunts(): imprimeix nom i punts totals.
  • Al main: crea dos jugadors, fes 5 rondes mostrant cada tirada, i al final declara el guanyador. En cas d'empat, informa-ho.
Avançat CA: 2.1 · 2.2 · 2.3 · 2.4 · 2.9
✅ Solució
import java.util.Random; public class Jugador { String nom; int punts; Random rnd = new Random(); Jugador(String nom) { this.nom = nom; this.punts = 0; } int tirarDau() { int valor = rnd.nextInt(6) + 1; punts += valor; return valor; } void mostrarPunts() { System.out.println(nom + ": " + punts + " punts"); } public static void main(String[] args) { Jugador j1 = new Jugador("Laia"); Jugador j2 = new Jugador("Marc"); for (int r = 1; r <= 5; r++) { int v1 = j1.tirarDau(); int v2 = j2.tirarDau(); System.out.println("Ronda " + r + ": " + j1.nom + " treu " + v1 + " | " + j2.nom + " treu " + v2); } System.out.println("\n--- Resultat final ---"); j1.mostrarPunts(); j2.mostrarPunts(); if (j1.punts > j2.punts) { System.out.println("Guanyador: " + j1.nom + " 🎉"); } else if (j2.punts > j1.punts) { System.out.println("Guanyador: " + j2.nom + " 🎉"); } else { System.out.println("Empat!"); } } }
