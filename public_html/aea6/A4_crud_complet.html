1. Arquitectura de l'aplicació

En les activitats anteriors hem escrit tot el codi JDBC directament al main(). En una aplicació real, el codi s'organitza en capes. En aquesta activitat aplicarem l'arquitectura que descriu el material del professor, amb tres paquets:

entitats/ Client.java // Classe POJO que representa un client Activitat.java // Classe POJO que representa una activitat persistencia/ Connexio.java // Singleton de connexió (de l'A1) ClientDAO.java // Operacions CRUD sobre Clients ActivitatDAO.java // Operacions CRUD sobre Activitats presentacio/ Main.java // Menú principal i interacció
📌 Patró DAO (Data Access Object): Les classes ClientDAO i ActivitatDAO encapsulen tota la lògica d'accés a la BD. El Main no sap res de JDBC: només crida mètodes del DAO com dao.inserir(client) o dao.llistarTots(). Aquesta separació fa el codi molt més mantenible.

2. Les classes entitat (POJO)

// ── entitats/Client.java ──────────────────────── package entitats; public class Client { private String dni, nom, correu; private int edat; public Client(String dni, String nom, int edat, String correu) { this.dni = dni; this.nom = nom; this.edat = edat; this.correu = correu; } public String getDni() { return dni; } public String getNom() { return nom; } public int getEdat() { return edat; } public String getCorreu() { return correu; } public void setEdat(int edat) { this.edat = edat; } public void setCorreu(String correu) { this.correu = correu; } public String toString() { return String.format("%-12s %-20s %3d %s", dni, nom, edat, correu); } }

3. La classe ClientDAO

La classe DAO conté un mètode per a cada operació CRUD. Cada mètode gestiona la seva pròpia SQLException o la propaga cap amunt, depenent del disseny.

package persistencia; import entitats.Client; import java.sql.*; import java.util.*; public class ClientDAO { // ── CREATE ───────────────────────────────────── public boolean inserir(Client c) { String sql = "INSERT INTO Clients (dni,nom,edat,correu) VALUES (?,?,?,?)"; try (PreparedStatement ps = Connexio.getConnexio().prepareStatement(sql)) { ps.setString(1, c.getDni()); ps.setString(2, c.getNom()); ps.setInt (3, c.getEdat()); ps.setString(4, c.getCorreu()); return ps.executeUpdate() > 0; } catch (SQLException e) { System.out.println("Error inserint client: " + e.getMessage()); return false; } } // ── READ ALL ─────────────────────────────────── public List<Client> llistarTots() { List<Client> llista = new ArrayList<>(); try (Statement st = Connexio.getConnexio().createStatement(); ResultSet rs = st.executeQuery("SELECT * FROM Clients ORDER BY nom")) { while (rs.next()) { llista.add(new Client( rs.getString("dni"), rs.getString("nom"), rs.getInt("edat"), rs.getString("correu"))); } } catch (SQLException e) { System.out.println("Error llegint clients: " + e.getMessage()); } return llista; } // ── UPDATE ───────────────────────────────────── public boolean actualitzar(Client c) { String sql = "UPDATE Clients SET nom=?, edat=?, correu=? WHERE dni=?"; try (PreparedStatement ps = Connexio.getConnexio().prepareStatement(sql)) { ps.setString(1, c.getNom()); ps.setInt(2, c.getEdat()); ps.setString(3, c.getCorreu()); ps.setString(4, c.getDni()); return ps.executeUpdate() > 0; } catch (SQLException e) { System.out.println("Error actualitzant: " + e.getMessage()); return false; } } // ── DELETE ───────────────────────────────────── public boolean eliminar(String dni) { try (PreparedStatement ps = Connexio.getConnexio().prepareStatement( "DELETE FROM Clients WHERE dni = ?")) { ps.setString(1, dni); return ps.executeUpdate() > 0; } catch (SQLException e) { System.out.println("Error eliminant: " + e.getMessage()); return false; } } }

4. Transaccions (nivell avançat)

Quan necessitem que dues o més operacions s'executin com una unitat indivisible (o totes o cap), usem transaccions. Per exemple: inscriure un client a una activitat i decrementar les places disponibles ha de ser atòmic.

Connection con = Connexio.getConnexio(); try { con.setAutoCommit(false); // Desactivem l'autocommit // Operació 1: inscriure el client try (PreparedStatement ps1 = con.prepareStatement( "INSERT INTO Inscripcions (dni, idActivitat) VALUES (?, ?)")) { ps1.setString(1, "12345678A"); ps1.setInt (2, 4); ps1.executeUpdate(); } // Operació 2: decrementar les places try (PreparedStatement ps2 = con.prepareStatement( "UPDATE Activitats SET places = places - 1 WHERE idActivitat = ?")) { ps2.setInt(1, 4); ps2.executeUpdate(); } con.commit(); // Confirmar les dues operacions juntes System.out.println("✓ Inscripció completada."); } catch (SQLException e) { con.rollback(); // Si alguna falla, desfer-ho tot System.out.println("✗ Error. Operació cancel·lada: " + e.getMessage()); } finally { con.setAutoCommit(true); // Restaurar l'estat per defecte }
⚠️ Sempre restaureu l'autocommit! Si no poseu setAutoCommit(true) al bloc finally, totes les operacions posteriors d'aquesta connexió també quedaran pendents de commit, la qual cosa pot causar comportaments inesperats.

  3. Implementa cada opció cridant el mètode corresponent del ClientDAO.
  4. Gestiona el cas en què l'usuari intenti eliminar un DNI que no existeix (mostra un missatge informatiu).

package persistencia; import entitats.Activitat; import java.sql.*; import java.util.*; public class ActivitatDAO { public List<Activitat> llistarTots() { List<Activitat> llista = new ArrayList<>(); try (Statement st = Connexio.getConnexio().createStatement(); ResultSet rs = st.executeQuery( "SELECT * FROM Activitats ORDER BY nom")) { while (rs.next()) { llista.add(new Activitat( rs.getInt("idActivitat"), rs.getString("nom"), rs.getString("nivell"), rs.getInt("places"))); } } catch (SQLException e) { System.out.println(e.getMessage()); } return llista; } public boolean inserir(Activitat a) { String sql = "INSERT INTO Activitats (nom, nivell, places) VALUES (?,?,?)"; try (PreparedStatement ps = Connexio.getConnexio().prepareStatement(sql)) { ps.setString(1, a.getNom()); ps.setString(2, a.getNivell()); ps.setInt (3, a.getPlaces()); return ps.executeUpdate() > 0; } catch (SQLException e) { System.out.println(e.getMessage()); return false; } } public boolean actualitzarPlaces(int id, int places) { try (PreparedStatement ps = Connexio.getConnexio().prepareStatement( "UPDATE Activitats SET places = ? WHERE idActivitat = ?")) { ps.setInt(1, places); ps.setInt(2, id); return ps.executeUpdate() > 0; } catch (SQLException e) { System.out.println(e.getMessage()); return false; } } public boolean eliminar(int id) { try (PreparedStatement ps = Connexio.getConnexio().prepareStatement( "DELETE FROM Activitats WHERE idActivitat = ?")) { ps.setInt(1, id); return ps.executeUpdate() > 0; } catch (SQLException e) { if (e.getErrorCode() == 1451) { System.out.println("No es pot eliminar: l'activitat té inscripcions actives."); } else { System.out.println(e.getMessage()); } return false; } } }
E3 Inscripció atòmica amb transacció Avançat

Implementa el mètode inscriureClient(String dni, int idActivitat) a una nova classe InscripcioDAO que realitzi les dues operacions (INSERT a Inscripcions + UPDATE places) de forma atòmica amb una transacció.

  1. Crea la classe InscripcioDAO al paquet persistencia.
  2. Implementa el mètode inscriureClient que: comprovi si hi ha places disponibles, faci l'INSERT a Inscripcions, faci l'UPDATE decrementant places, i faci commit si tot va bé o rollback si falla qualsevol operació.
  3. Prova el cas d'error: simula un error al segon executeUpdate i comprova que el rollback desfà el primer INSERT.
  4. Afegeix l'opció d'inscriure un client al menú del Main.

Solució — InscripcioDAO.java amb transacció
package persistencia; import java.sql.*; public class InscripcioDAO { public boolean inscriureClient(String dni, int idActivitat) { Connection con = null; try { con = Connexio.getConnexio(); con.setAutoCommit(false); // 1. Comprovar places disponibles try (PreparedStatement ps = con.prepareStatement( "SELECT places FROM Activitats WHERE idActivitat = ?")) { ps.setInt(1, idActivitat); ResultSet rs = ps.executeQuery(); rs.next(); if (rs.getInt("places") <= 0) { System.out.println("L'activitat és plena."); con.setAutoCommit(true); return false; } } // 2. Crear la inscripció try (PreparedStatement ps = con.prepareStatement( "INSERT INTO Inscripcions (dni, idActivitat) VALUES (?, ?)")) { ps.setString(1, dni); ps.setInt (2, idActivitat); ps.executeUpdate(); } // 3. Decrementar places try (PreparedStatement ps = con.prepareStatement( "UPDATE Activitats SET places = places - 1 WHERE idActivitat = ?")) { ps.setInt(1, idActivitat); ps.executeUpdate(); } con.commit(); // Tot correcte: confirmar System.out.println("✓ Inscripció completada correctament."); return true; } catch (SQLException e) { try { if (con != null) con.rollback(); } catch (SQLException ex) {} System.out.println("✗ Error. Operació revertida: " + e.getMessage()); return false; } finally { try { if (con != null) con.setAutoCommit(true); } catch (SQLException ex) {} } } }
