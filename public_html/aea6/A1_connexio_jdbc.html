1. Què és JDBC?

Quan una aplicació Java ha d'interactuar amb una base de dades, necessita un mecanisme que faci de pont entre el codi Java i el sistema gestor de bases de dades (SGBD). Aquest pont és JDBC (Java Database Connectivity), una API estàndard de Java que permet connectar-se a qualsevol SGBD (MySQL, PostgreSQL, Oracle, SQLite…) de manera independent del proveïdor.

Gràcies a JDBC podem fer des de Java totes les operacions habituals sobre una base de dades: connectar-nos, executar consultes SQL, llegir resultats i tancar la connexió.

2. Arquitectura per capes

Una aplicació Java ben estructurada que usa JDBC segueix una arquitectura de tres capes. Cada capa té una responsabilitat clara i no "sap" com funcionen les altres per dins:

[ Capa de presentació (GUI / consola) ]
↓ crida mètodes del servei
[ Capa de lògica de negoci (serveis) ]
↓ crida mètodes de persistència
[ Capa de persistència (JDBC) ]
↓ SQL
[ Base de dades relacional (MySQL) ]

En aquesta AEA treballarem principalment la capa de persistència, que és on s'usa JDBC per fer les operacions de connexió, consulta i modificació de dades.

3. El driver JDBC

Per poder connectar-nos a MySQL des de Java necessitem afegir al projecte el connector MySQL, que és un fitxer .jar que actua com a driver JDBC. Al Drive del curs trobareu el fitxer mysql-connector-j-9.6.0.jar.

📌 Versions antigues vs actuals: En versions antigues de Java calia registrar el driver explícitament amb Class.forName("com.mysql.jdbc.Driver"). Amb els connectors actuals (versió 6+) ja no cal fer-ho: simplement afegiu el .jar al projecte i el driver es registra sol.

Per afegir el connector a NetBeans de forma que el projecte sigui portable:

  1. Copieu el fitxer .jar dins d'una carpeta lib/ a l'arrel del vostre projecte NetBeans.
  2. A NetBeans, botó dret sobre Libraries → Add JAR/Folder → seleccioneu el fitxer amb ruta relativa.
  3. Comproveu que apareix a la llista de llibreries del projecte.
💡 Ruta relativa: Si useu ruta relativa per afegir el .jar, el projecte funcionarà en qualsevol ordinador sense haver de tornar a configurar res. Eviteu rutes absolutes tipus C:\Users\....

4. Connexió amb l'SGBD

Un cop tenim el driver al projecte, podem establir la connexió. La classe clau és DriverManager, que s'encarrega de crear un objecte Connection a partir d'una URL, un usuari i una contrasenya.

L'URL de connexió per a MySQL té aquest format:

// Format general jdbc:mysql://adreça_servidor:port/nom_base_de_dades // Exemple per a servidor local jdbc:mysql://localhost:3306/libros

El codi per establir la connexió és el següent:

import java.sql.Connection; import java.sql.DriverManager; import java.sql.SQLException; public class Main { public static void main(String[] args) { String url = "jdbc:mysql://localhost:3306/libros"; String usuari = "root"; String pass = ""; try (Connection con = DriverManager.getConnection(url, usuari, pass)) { System.out.println("Connexió establerta correctament!"); } catch (SQLException e) { System.out.println("Error de connexió: " + e.getMessage()); } } }
📌 try-with-resources: Obrir la connexió dins del try(...) garanteix que es tancarà automàticament quan el bloc acabi, fins i tot si es produeix un error. No cal cridar con.close() explícitament. És la forma recomanada.
⚠️ SQLException: Si la connexió falla, getConnection() llança una SQLException. Les causes més habituals són: XAMPP no arrenccat, URL incorrecta, base de dades inexistent, o usuari/contrasenya incorrectes. Sempre gestioneu aquest error.

5. La classe Connexio (patró Singleton)

En projectes reals no volem crear una connexió nova cada vegada que accedim a la BD. La solució és encapsular la connexió en una classe separada que asseguri que només existeix una única instància de la connexió (patró Singleton).

package persistencia; import java.sql.Connection; import java.sql.DriverManager; import java.sql.SQLException; public class Connexio { final static String url = "jdbc:mysql://localhost:3306/libros"; final static String usuari = "root"; final static String pass = ""; static Connection con = null; private Connexio() { // Constructor privat: impedeix instanciar la classe des de fora } public static Connection getConnexio() throws SQLException { if (con == null) { con = DriverManager.getConnection(url, usuari, pass); } return con; } }

D'aquesta manera, qualsevol part del programa pot cridar Connexio.getConnexio() i sempre obtindrà la mateixa connexió, en lloc de crear-ne una de nova cada vegada.

6. Exercicis

E1 Primera connexió a MySQL Bàsic

Crea un projecte NetBeans nou, afegeix el connector mysql-connector-j-9.6.0.jar a les llibreries i escriu un programa que intenti connectar-se a la base de dades libros i mostri per pantalla si la connexió ha estat correcta o ha fallat.

  1. Crea el projecte PrimerJDBC a NetBeans.
  2. Crea la carpeta lib/ dins el projecte i copia-hi el .jar del connector.
  3. Afegeix el .jar a les llibreries del projecte amb ruta relativa.
  4. Executa el fitxer libros.sql a MySQL (via phpMyAdmin o des de NetBeans) per crear la BD.
  5. Escriu el codi de connexió amb try-with-resources i catch(SQLException).
  6. Prova el cas d'error: atura XAMPP i torna a executar el programa. Observa el missatge d'error.
Bàsic CA 9.1 · 9.2
Solució — Primera connexió a MySQL
import java.sql.Connection; import java.sql.DriverManager; import java.sql.SQLException; public class PrimerJDBC { public static void main(String[] args) { String url = "jdbc:mysql://localhost:3306/libros"; String usuari = "root"; String pass = ""; try (Connection con = DriverManager.getConnection(url, usuari, pass)) { System.out.println("✓ Connexió establerta correctament!"); System.out.println(" BD: " + con.getCatalog()); System.out.println(" Driver: " + con.getMetaData().getDriverName()); } catch (SQLException e) { System.out.println("✗ Error de connexió: " + e.getMessage()); } } }
E2 Classe Connexio per al Gimnàs Mitjà

Adapta la classe Connexio del patró Singleton perquè apunti a la base de dades Gimnas, i crea una classe Main que la faci servir per verificar la connexió i mostri informació de la BD.

  1. Crea un projecte nou GimnasJDBC amb dos paquets: persistencia i presentacio.
  2. Importa el fitxer Gimnas.sql a MySQL per crear la base de dades.
  3. Crea la classe Connexio al paquet persistencia apuntant a Gimnas.
  4. Crea la classe Main al paquet presentacio que cridi Connexio.getConnexio() i mostri: nom de la BD (getCatalog()), versió del servidor (getMetaData().getDatabaseProductVersion()) i si la connexió és vàlida (isValid(1)).

Solució — Classe Connexio per al Gimnàs
// ── paquet persistencia ───────────────────────── package persistencia; import java.sql.Connection; import java.sql.DriverManager; import java.sql.SQLException; public class Connexio { private static final String URL = "jdbc:mysql://localhost:3306/Gimnas"; private static final String USUARI = "root"; private static final String PASS = ""; private static Connection con = null; private Connexio() {} public static Connection getConnexio() throws SQLException { if (con == null || !con.isValid(1)) { con = DriverManager.getConnection(URL, USUARI, PASS); } return con; } } // ── paquet presentacio ───────────────────────── package presentacio; import java.sql.Connection; import java.sql.SQLException; import persistencia.Connexio; public class Main { public static void main(String[] args) { try { Connection con = Connexio.getConnexio(); System.out.println("Base de dades : " + con.getCatalog()); System.out.println("Versió servidor: " + con.getMetaData().getDatabaseProductVersion()); System.out.println("Connexió vàlida: " + con.isValid(1)); } catch (SQLException e) { System.out.println("Error: " + e.getMessage()); } } }
E3 Gestió robusta d'errors de connexió Avançat

Millora la classe Connexio del Gimnàs per gestionar de manera informativa els tres errors de connexió més habituals: servidor aturat, base de dades inexistent i contrasenya incorrecta.

  1. Partint de la solució de l'E2, amplia el bloc catch(SQLException) de la classe Main.
  2. Usa e.getErrorCode() i/o e.getSQLState() per distingir els casos:
    • Codi 0 / SQLState 08S01: servidor aturat o no accessible.
    • Codi 1049: base de dades desconeguda.
    • Codi 1045: accés denegat (usuari/contrasenya incorrectes).
  3. Mostra un missatge específic i comprensible per a cada cas.
  4. Prova cada cas modificant la URL, el nom de la BD o la contrasenya.

Usuari o contrasenya incorrectes."); } else { System.out.println("✗ Error inesperat (codi " + codi + "): " + e.getMessage()); } }
