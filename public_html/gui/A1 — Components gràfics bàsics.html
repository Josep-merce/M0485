1. Swing i NetBeans: visió general

Swing és la biblioteca de Java per crear interfícies gràfiques d'escriptori. Tots els components hereten de JComponent i comencen amb J (JFrame, JButton, JLabel...). NetBeans ofereix un editor visual de formularis que genera el codi automàticament mentre dissenyes arrossegant components.

Flux de treball a NetBeans: crear projecte Java → afegir un JFrame Form → arrossegar components des de la Paleta a l'àrea de disseny → editar propietats al panell dret → fer doble clic al component per afegir-hi el codi de l'esdeveniment.
1.1 Eines de l'IDE
EinaFunció
PaletaLlista tots els components Swing disponibles per arrossegar al formulari
Àrea de dissenyVista visual del formulari; cada canvi es reflecteix al codi
Editor de propietatsPermet canviar les propietats de cada component (text, mida, font, color...)
Vista de codiEl codi generat per NetBeans; la secció Generated Code no s'ha d'editar manualment

2. Components principals

2.1 JFrame — la finestra principal

JFrame és la finestra contenidora principal. La nostra classe n'hereta i afegeix els components al seu interior. La propietat defaultCloseOperation s'ha de posar a EXIT_ON_CLOSE perquè el programa s'aturi en tancar la finestra.

// Per mostrar el formulari des del main: public static void main(String[] args) { // Executa la creació de la finestra al fil de l'Event Dispatch Thread (EDT) java.awt.EventQueue.invokeLater(() -> { new MevaFinestra().setVisible(true); }); }

3. Layout managers

Els layout managers controlen com es distribueixen els components dins d'un contenidor. NetBeans usa per defecte GroupLayout, però per als exercicis escrits a mà els layouts bàsics són més pràctics.

LayoutComportamentQuan usar-lo
FlowLayoutComponents en fila, s'ajusten quan no hi capGrups de botons horitzontals
BorderLayout5 zones: NORTH, SOUTH, EAST, WEST, CENTEREstructures principals de finestra
GridLayout(f,c)Quadrícula de f files i c columnes, mida igualFormularis amb camps alineats
GroupLayoutControl precís horitzontal i vertical (NetBeans)Formularis complexos generats per IDE
nullPosició absoluta amb setBounds(x,y,w,h)Prototips ràpids (no recomanat)
// Exemple de BorderLayout amb panells JFrame finestra = new JFrame("Exemple BorderLayout"); finestra.setLayout(new BorderLayout()); JLabel titol = new JLabel("Títol", SwingConstants.CENTER); JButton boto = new JButton("Acceptar"); JTextArea area = new JTextArea(); finestra.add(titol, BorderLayout.NORTH); finestra.add(area, BorderLayout.CENTER); finestra.add(boto, BorderLayout.SOUTH); finestra.setSize(400, 300); finestra.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); finestra.setVisible(true);

4. Exercicis

E1 Formulari de dades personals Bàsic

Crea un formulari visual a NetBeans que reculli dades personals d'un usuari seguint el PDF de l'exemple pràctic de Canela.

  1. Crea un projecte nou a NetBeans i afegeix un JFrame Form.
  2. Afegeix dos JPanel amb vora de tipus TitledBorder: "Dades personals" i "Correu electrònic".
  3. Dins el primer, afegeix JLabel i JTextField per al nom, cognoms i DNI.
  4. Afegeix un JComboBox amb les quatre províncies catalanes.
  5. Afegeix un botó "Netejar" que buidi tots els camps.
Bàsic CA 5.6
Solució — Formulari de dades personals
// Neteja dels camps — mètode associat al botó "Netejar" private void btnNetejarActionPerformed(ActionEvent evt) { tfNom.setText(""); tfCognoms.setText(""); tfDNI.setText(""); comboProvinciа.setSelectedIndex(0); tfNom.requestFocus(); }
E2 Comanda de productes amb JComboBox Mitjà

Implementa el formulari de comanda de productes de l'Exercici 5 de Canela: JComboBox editable, JRadioButton, JCheckBox i JTextArea de resultats.

  1. Crea un formulari de "Comanda de productes" basant-te en el PDF de l'exercici 5 de Canela.
  2. Inclou un JComboBox editable amb tres productes inicials (Pa, Llet, Fruita).
  3. Afegeix JRadioButton per al tipus de venda (Unitats / Pes), agrupats en un ButtonGroup.
  4. Afegeix un JCheckBox "En oferta" i un JTextField per a la quantitat.
  5. El botó "Afegir" ha d'afegir la comanda a un JTextArea amb el format: producte – quantitat – tipus – oferta.

E3 Registre d'alumnes amb validació Avançat

Crea un formulari de registre d'alumnes complet amb validació de tots els camps, camp NIF amb màscara i exportació en format CSV al JTextArea.

  1. Crea un formulari complet de registre d'alumnes amb: nom, cognoms, NIF (camp JFormattedTextField amb màscara ########U), correu, nota (0-10) i curs (JComboBox).
  2. Afegeix validació al botó "Registrar": camp buit mostra un JOptionPane d'error; nota fora del rang 0-10 mostra avís.
  3. Si la validació passa, afegeix l'alumne a un JTextArea de resultats en format CSV i neteja el formulari.
  4. El botó "Esborrar resultats" ha de netejar el JTextArea.
Avançat CA 5.6

// Validació i registre d'alumne private void btnRegistrarActionPerformed(ActionEvent evt) { String nom = tfNom.getText().trim(); String cognoms = tfCognoms.getText().trim(); String nif = tfNIF.getText().trim(); String correu = tfCorreu.getText().trim(); String cursStr = (String) comboCurs.getSelectedItem(); double nota; try { nota = Double.parseDouble(tfNota.getText().trim()); } catch (NumberFormatException e) { JOptionPane.showMessageDialog(this, "La nota ha de ser un número."); return; } if (nom.isEmpty() || cognoms.isEmpty() || nif.isEmpty()) { JOptionPane.showMessageDialog(this, "Tots els camps són obligatoris."); return; } if (nota < 0 || nota > 10) { JOptionPane.showMessageDialog(this, "La nota ha d'estar entre 0 i 10."); return; } taResultats.append(String.format("%s;%s;%s;%s;%.1f;%s%n", nom, cognoms, nif, correu, nota, cursStr)); // Netejar formulari tfNom.setText(""); tfCognoms.setText(""); tfNIF.setText(""); tfCorreu.setText(""); tfNota.setText(""); comboCurs.setSelectedIndex(0); tfNom.requestFocus(); }
