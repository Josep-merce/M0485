1. El Java Collections Framework

El Java Collections Framework (JCF) és un conjunt d'interfícies i classes que proporcionen estructures de dades dinàmiques. A diferència dels arrays, les col·leccions creixen i decreixen en temps d'execució i ofereixen moltes operacions ja implementades (ordenació, cerca, filtratge...).

Totes les col·leccions es declaren amb el tipus de l'element entre <>. Això és la genericitat en acció: el compilador sap quin tipus espera i detecta errors en temps de compilació.

InterfícieCaracterístiquesImplementacions típiques
List<E>Ordenada per posició, permet repetits, indexadaArrayList, LinkedList
Set<E>No permet duplicats, no té ordre garantit (depèn)HashSet, TreeSet, LinkedHashSet
Map<K,V>Parells clau→valor, claus úniquesHashMap, TreeMap
Bona pràctica: declara la variable amb la interfície i instancia amb la classe concreta. Així pots canviar la implementació en un sol punt sense tocar la resta del codi:
List<String> noms = new ArrayList<>();
List<String> noms = new LinkedList<>();  ← canvi mínim

2. List: ArrayList i LinkedList

Una List és una col·lecció ordenada per posició que permet elements repetits. ArrayList és la millor opció per a accessos aleatoris ràpids; LinkedList és millor per a insercions i esborrades al mig.

2.1 Operacions bàsiques amb ArrayList
import java.util.ArrayList; import java.util.List; List<String> fruites = new ArrayList<>(); // Afegir elements fruites.add("poma"); fruites.add("pera"); fruites.add("mango"); fruites.add(1, "kiwi"); // inserir a la posició 1 // Llegir elements System.out.println(fruites.get(0)); // "poma" System.out.println(fruites.size()); // 4 System.out.println(fruites.contains("pera")); // true System.out.println(fruites.indexOf("mango")); // 2 // Eliminar elements fruites.remove("kiwi"); // per valor fruites.remove(0); // per posició // Recorregut (for-each) for (String f : fruites) { System.out.println(f); } // Ordenar fruites.sort(Comparator.naturalOrder());
2.2 Iterator — eliminar mentre recorrem

Mai eliminis elements d'una col·lecció dins d'un bucle for-each normal; causarà ConcurrentModificationException. Usa un Iterator o el mètode removeIf:

import java.util.Iterator; Iterator<String> it = fruites.iterator(); while (it.hasNext()) { String f = it.next(); if (f.startsWith("p")) { it.remove(); // elimina de forma segura } } // Alternativa més concisa amb removeIf (Java 8+) fruites.removeIf(f -> f.startsWith("p"));
2.3 ArrayList vs LinkedList
OperacióArrayListLinkedList
Accés per índex (get(i))✅ O(1) molt ràpid⚠️ O(n) lent
Inserció al mig⚠️ O(n) lent (cal desplaçar)✅ O(1) molt ràpid
Afegir al final✅ ràpid (amortitzat)✅ ràpid
Ús de memòriaMenys (array intern)Més (nodes amb punters)

3. Set: HashSet i TreeSet

Un Set no permet elements duplicats. És útil quan necessites garantir la unicitat d'un conjunt de valors (p. ex. paraules úniques, categories, DNIs...).

ClasseOrdre dels elementsQuan usar-la
HashSetCap ordre garantitQuan la velocitat és prioritat i l'ordre no importa
LinkedHashSetOrdre d'insercióQuan cal unicitat i recordar l'ordre d'inserció
TreeSetOrdre natural o per comparadorQuan cal unicitat i els elements ordenats
import java.util.HashSet; import java.util.TreeSet; import java.util.Set; // HashSet: sense ordre Set<String> colors = new HashSet<>(); colors.add("vermell"); colors.add("blau"); colors.add("vermell"); // duplicat: ignorat System.out.println(colors.size()); // 2 // TreeSet: elements ordenats alfabèticament Set<String> ordenat = new TreeSet<>(colors); System.out.println(ordenat); // [blau, vermell] // TreeSet amb objectes propis: cal un Comparator Set<Alumne> alumnes = new TreeSet<>( Comparator.comparing(Alumne::getNota) ); alumnes.add(new Alumne("Anna", 8.5)); alumnes.add(new Alumne("Marc", 7.0)); alumnes.add(new Alumne("Laia", 9.0)); // Recorrem ordenats per nota for (Alumne a : alumnes) System.out.println(a);

4. Map: HashMap i TreeMap

Un Map associa claus úniques a valors. És com un diccionari: donada una clau, obtens el valor corresponent en temps constant (amb HashMap).

import java.util.HashMap; import java.util.Map; Map<String, Integer> edats = new HashMap<>(); // Inserir parells clau→valor edats.put("Anna", 22); edats.put("Marc", 19); edats.put("Laia", 25); edats.put("Anna", 23); // sobreescriu el valor d'Anna // Llegir System.out.println(edats.get("Marc")); // 19 System.out.println(edats.containsKey("Laia")); // true System.out.println(edats.getOrDefault("Jordi", 0)); // 0 (no existeix) // Recórrer totes les entrades (clau + valor) for (Map.Entry<String, Integer> entrada : edats.entrySet()) { System.out.println(entrada.getKey() + " → " + entrada.getValue()); } // Mètodes útils edats.putIfAbsent("Anna", 99); // no sobreescriu perquè ja existeix edats.remove("Marc"); System.out.println(edats.size()); // 2
HashMap vs TreeMap: HashMap no garanteix cap ordre de les claus (màxim rendiment). TreeMap manté les claus ordenades per ordre natural o per un Comparator.

5. Genericitat

Els genèrics permeten escriure classes, interfícies i mètodes que funcionen amb qualsevol tipus, garantint la seguretat de tipus en temps de compilació. Ja els has vist als contenidors (List<String>, Map<K,V>); ara aprendrem a crear-ne de propis.

5.1 Classe genèrica
// Classe Caixa que pot contenir qualsevol tipus T public class Caixa<T> { private T contingut; public void set(T contingut) { this.contingut = contingut; } public T get() { return contingut; } } // Ús: el compilador comprova els tipus Caixa<String> c1 = new Caixa<>(); c1.set("Hola"); String text = c1.get(); // no cal cast Caixa<Integer> c2 = new Caixa<>(); c2.set(42); // c2.set("text"); → ERROR en compilació!
5.2 Convenis de noms
SímbolSignificatExemple
TTipus genèric generalCaixa<T>
EElement (per a col·leccions)ArrayList<E>
KClau (key)Map<K, V>
VValor (value)Map<K, V>
?Tipus desconegut (comodí)List<?>
5.3 Mètode genèric
// Mètode que imprimeix qualsevol llista independentment del tipus static <T> void imprimirLlista(List<T> llista) { for (T element : llista) { System.out.print(element + " "); } System.out.println(); } // Mètode genèric amb límit superior: T ha de ser Number o subclasse static <T extends Number> double sumarLlista(List<T> llista) { double suma = 0; for (T n : llista) suma += n.doubleValue(); return suma; } // Ús imprimirLlista(List.of("a", "b", "c")); imprimirLlista(List.of(1, 2, 3)); System.out.println(sumarLlista(List.of(1.5, 2.0, 3.5))); // 7.0
Restricció important: els genèrics no accepten tipus primitius directament. Usa les classes wrapper: Integer en lloc de int, Double en lloc de double, etc. Java fa l'autoboxing automàticament quan assignes valors literals.

6. Stream API

Un Stream és una seqüència d'elements sobre la qual podem aplicar operacions de forma declarativa (dient què volem, no com). No modifica la col·lecció original i s'estructura sempre en tres fases:

Font (col·lecció / array) → Intermèdies (0..N) → Terminal (1)
Regles fonamentals dels Streams:
· Les operacions intermèdies no executen res per si soles: només descriuen el procés.
· L'operació terminal dispara tota la cadena i tanca el Stream.
· Un Stream només es pot usar una vegada: un cop consumit no es pot reutilitzar.
· El Stream no modifica mai la col·lecció original.
6.1 Operacions intermèdies
OperacióDescripcióExemple
filter(predicat)Manté els elements que compleixen la condició.filter(p -> p.getEdat() >= 18)
map(funció)Transforma cada element.map(p -> p.getNom())
sorted()Ordena per ordre natural.sorted()
sorted(comparador)Ordena per criteri personalitzat.sorted(Comparator.comparing(Persona::getEdat))
distinct()Elimina duplicats.distinct()
limit(n)Pren els primers n elements.limit(5)
skip(n)Descarta els primers n elements.skip(2)
6.2 Operacions terminals
OperacióDescripcióExemple
forEach(acció)Executa una acció per cada element.forEach(System.out::println)
collect(collector)Acumula els resultats en una col·lecció.collect(Collectors.toList())
count()Retorna el nombre d'elements.count()
min(comparador)Retorna l'element mínim (Optional).min(Comparator.naturalOrder())
max(comparador)Retorna l'element màxim (Optional).max(Comparator.naturalOrder())
allMatch(predicat)Tots compleixen la condició?.allMatch(p -> p.getEdat() > 0)
anyMatch(predicat)Algun compleix la condició?.anyMatch(p -> p.getNom().equals("Anna"))
noneMatch(predicat)Cap compleix la condició?.noneMatch(p -> p.getEdat() < 0)
6.3 Exemple complet amb classe Persona
import java.util.*; import java.util.stream.*; public class ExempleStream { public static void main(String[] args) { List<Persona> persones = List.of( new Persona("Anna", 22, "F"), new Persona("Marc", 17, "M"), new Persona("Laia", 30, "F"), new Persona("Jordi", 15, "M"), new Persona("Marta", 28, "F") ); // Noms de les persones majors d'edat, ordenats alfabèticament List<String> majors = persones.stream() .filter(p -> p.getEdat() >= 18) .map(Persona::getNom) .sorted() .collect(Collectors.toList()); System.out.println(majors); // [Anna, Laia, Marta] // Quantes persones hi ha de cada sexe? Map<String, Long> perSexe = persones.stream() .collect(Collectors.groupingBy(Persona::getSexe, Collectors.counting())); System.out.println(perSexe); // {F=3, M=2} // La persona més jove persones.stream() .min(Comparator.comparing(Persona::getEdat)) .ifPresent(p -> System.out.println("Més jove: " + p.getNom())); // Jordi // Totes les persones són majors d'edat? boolean totesMajors = persones.stream() .allMatch(p -> p.getEdat() >= 18); System.out.println(totesMajors); // false } }

  2. Mostra el segon i el penúltim element.
  3. Afegeix un nou nom a la posició 2.
  4. Elimina tots els noms que tinguin menys de 4 caràcters usant removeIf.
  5. Ordena la llista alfabèticament i mostra-la.
Bàsic CA 6.6 — ArrayList, iteradors
✅ Solució
import java.util.*; public class GestioAlumnes { public static void main(String[] args) { List<String> alumnes = new ArrayList<>( List.of("Anna", "Marc", "Laia", "Pol", "Marta", "Eva") ); System.out.println("Segon: " + alumnes.get(1)); System.out.println("Penúltim: " + alumnes.get(alumnes.size() - 2)); alumnes.add(2, "Jordi"); System.out.println("Après afegir: " + alumnes); alumnes.removeIf(nom -> nom.length() < 4); System.out.println("Sense noms curts: " + alumnes); alumnes.sort(Comparator.naturalOrder()); System.out.println("Ordenada: " + alumnes); } }
Ex. 2 Paraules úniques d'un text Bàsic

  1. Divideix la cadena en paraules amb split(" ").
  2. Afegeix-les totes a un TreeSet<String> (que eliminarà duplicats i les ordenarà).
  3. Mostra el conjunt i la seva mida.
Bàsic CA 6.6 — TreeSet, unicitat
✅ Solució
import java.util.*; public class ParaulesUniques { public static void main(String[] args) { String text = "poma pera poma kiwi pera taronja kiwi poma"; String[] paraules = text.split(" "); Set<String> uniques = new TreeSet<>(); for (String p : paraules) uniques.add(p); System.out.println("Paraules úniques: " + uniques); System.out.println("Total: " + uniques.size()); } } // Sortida: // Paraules úniques: [kiwi, pera, poma, taronja] // Total: 4
Ex. 3 Diccionari d'ocurrències amb HashMap Mitjà

Donada la mateixa cadena de l'exercici anterior, compta quantes vegades apareix cada paraula usant un HashMap<String, Integer>.

  1. Per a cada paraula, si ja és al mapa incrementa el comptador; si no, afegeix-la amb valor 1. Pots usar getOrDefault per simplificar.
  2. Mostra totes les parelles paraula → nombre d'ocurrències.
  3. Mostra quina paraula apareix més vegades.
Mitjà CA 6.6 — HashMap, entrySet
✅ Solució
import java.util.*; public class Ocurrencies { public static void main(String[] args) { String[] paraules = "poma pera poma kiwi pera taronja kiwi poma".split(" "); Map<String, Integer> comptador = new HashMap<>(); for (String p : paraules) { comptador.put(p, comptador.getOrDefault(p, 0) + 1); } // Mostrar totes les ocurrències for (Map.Entry<String, Integer> e : comptador.entrySet()) { System.out.println(e.getKey() + " → " + e.getValue()); } // Paraula més freqüent String maxParaula = comptador.entrySet().stream() .max(Map.Entry.comparingByValue()) .get().getKey(); System.out.println("Més freqüent: " + maxParaula + " (" + comptador.get(maxParaula) + "x)"); } }
Ex. 4 Classe genèrica Parell<A, B> Mitjà

Crea una classe genèrica Parell<A, B> que emmagatzemi dos valors de tipus potencialment diferents. Implementa-la i demostra el seu ús.

  1. La classe tindrà dos atributs genèrics primer (de tipus A) i segon (de tipus B), constructor, getters i toString().
  2. Al main, crea: un Parell<String, Integer> per a nom i edat, un Parell<String, Double> per a producte i preu.
  3. Crea un mètode genèric estàtic bescanviar(Parell<A,B>) que retorni un nou Parell<B,A> amb els elements intercanviats.
Mitjà CA 6.4 — Classes genèriques
✅ Solució
public class Parell<A, B> { private A primer; private B segon; public Parell(A primer, B segon) { this.primer = primer; this.segon = segon; } public A getPrimer() { return primer; } public B getSegon() { return segon; } @Override public String toString() { return "(" + primer + ", " + segon + ")"; } // Mètode genèric estàtic que bescanvia els dos elements public static <A, B> Parell<B, A> bescanviar(Parell<A, B> p) { return new Parell<>(p.getSegon(), p.getPrimer()); } } public class Main { public static void main(String[] args) { Parell<String, Integer> persona = new Parell<>("Anna", 22); Parell<String, Double> producte = new Parell<>("Teclat", 45.99); System.out.println(persona); // (Anna, 22) System.out.println(producte); // (Teclat, 45.99) Parell<Integer, String> bescanviat = Parell.bescanviar(persona); System.out.println(bescanviat); // (22, Anna) } }
Ex. 5 Processament de persones amb Streams Mitjà

Donada una llista de 6 persones (nom, edat, sexe), resol les consultes usant exclusivament Streams.

  1. Llista els noms de les dones ordenats alfabèticament.
  2. Calcula l'edat mitjana de tots els homes.
  3. Comprova si hi ha alguna persona menor de 18 anys.
  4. Recull en un TreeSet les edats de totes les persones (sense repetits i ordenades).
Mitjà CA 6.7 — Stream: filter, map, collect, anyMatch
✅ Solució
import java.util.*; import java.util.stream.*; // Classe Persona (simplificada) class Persona { private String nom; private int edat; private String sexe; public Persona(String nom, int edat, String sexe) { this.nom = nom; this.edat = edat; this.sexe = sexe; } public String getNom() { return nom; } public int getEdat() { return edat; } public String getSexe() { return sexe; } } public class StreamPersones { public static void main(String[] args) { List<Persona> llista = List.of( new Persona("Anna", 22, "F"), new Persona("Marc", 17, "M"), new Persona("Laia", 30, "F"), new Persona("Jordi", 15, "M"), new Persona("Marta", 28, "F"), new Persona("Pau", 34, "M") ); // 1. Noms de les dones, ordenats List<String> dones = llista.stream() .filter(p -> p.getSexe().equals("F")) .map(Persona::getNom) .sorted() .collect(Collectors.toList()); System.out.println("Dones: " + dones); // [Anna, Laia, Marta] // 2. Edat mitjana dels homes double mitjanaHomes = llista.stream() .filter(p -> p.getSexe().equals("M")) .mapToInt(Persona::getEdat) .average() .orElse(0); System.out.printf("Edat mitjana homes: %.1f%n", mitjanaHomes); // 22.0 // 3. Hi ha algun menor? boolean hiHaMinor = llista.stream() .anyMatch(p -> p.getEdat() < 18); System.out.println("Hi ha menors: " + hiHaMinor); // true // 4. Edats úniques i ordenades Set<Integer> edats = llista.stream() .map(Persona::getEdat) .collect(Collectors.toCollection(TreeSet::new)); System.out.println("Edats: " + edats); // [15, 17, 22, 28, 30, 34] } }
Ex. 6 Sistema d'inventari amb genèrics (pràctica integradora) Avançat

Implementa un sistema d'inventari genèric seguint l'enunciat del fitxer Tema3_ExerciciGenerics.pdf del Drive. El sistema ha de gestionar productes de tipus diferent (Llibre, Electrodomèstic) a través d'una classe genèrica Inventari<T extends Producte>.

  1. Crea la interfície Producte amb getId(), getNom() i getPreu().
  2. Implementa les classes Llibre (atribut extra: autor) i Electrodomestic (atribut extra: marca).
  3. Crea la classe genèrica Inventari<T extends Producte> amb mètodes per afegir, consultar, eliminar i mostrar tots els productes. Llença IllegalArgumentException per a productes duplicats o inexistents.
  4. Al main, crea un inventari de llibres i un d'electrodomèstics, demostra totes les operacions i gestiona els errors.
Nota: Pots executar el fitxer GestioProductesArrays.jar del Drive per veure un exemple del resultat esperat amb arrays. L'exercici d'inventari genèric és equivalent però usant col·leccions i genèrics.
Avançat CA 6.4 · CA 6.6 · CA 6.10 — Genèrics + col·leccions + excepcions
✅ Solució (estructura principal)
// Interfície Producte public interface Producte { String getId(); String getNom(); double getPreu(); } // Classe Llibre public class Llibre implements Producte { private String id, nom, autor; private double preu; public Llibre(String id, String nom, String autor, double preu) { this.id = id; this.nom = nom; this.autor = autor; this.preu = preu; } public String getId() { return id; } public String getNom() { return nom; } public double getPreu(){ return preu; } @Override public String toString() { return "Llibre["+nom+", "+autor+", "+preu+"€]"; } } // Classe genèrica Inventari public class Inventari<T extends Producte> { private List<T> elements = new ArrayList<>(); public void afegir(T producte) { for (T e : elements) if (e.getId().equals(producte.getId())) throw new IllegalArgumentException("Producte ja existent: " + producte.getId()); elements.add(producte); } public T consultar(String id) { return elements.stream() .filter(e -> e.getId().equals(id)) .findFirst() .orElseThrow(() -> new IllegalArgumentException("No trobat: " + id)); } public void eliminar(String id) { T p = consultar(id); // llança excepció si no existeix elements.remove(p); } public void mostrarTots() { elements.forEach(System.out::println); } } // Main public class Main { public static void main(String[] args) { Inventari<Llibre> inv = new Inventari<>(); inv.afegir(new Llibre("L1", "El Quixot", "Cervantes", 12.50)); inv.afegir(new Llibre("L2", "1984", "Orwell", 9.95)); inv.mostrarTots(); try { inv.afegir(new Llibre("L1", "Duplicat", "?", 0)); // llança excepció } catch (IllegalArgumentException e) { System.out.println("Error: " + e.getMessage()); } System.out.println("Consulta L2: " + inv.consultar("L2")); inv.eliminar("L1"); inv.mostrarTots(); } }
