1. Què és una excepció?

Fins ara, quan un programa trobava un error (dividir per zero, accedir a una posició inexistent d'un array…), s'aturava bruscament. Les excepcions són el mecanisme de Java per detectar aquests errors en temps d'execució i reaccionar de forma controlada, sense que el programa es tanqui de cop.

Quan es produeix un error, Java crea un objecte excepció que conté informació sobre el problema i el llança. El nostre codi pot capturar aquest objecte i decidir què fer.

Dos enfocaments per gestionar errors:
Sense excepcions (prevenció): comproves les condicions prèviament amb if.
Amb excepcions (reacció): deixes que l'error es produeixi i el captures amb try-catch.
Les excepcions permeten separar la lògica de negoci del codi de gestió d'errors, fent el codi més llegible.

2. La jerarquia Throwable

Totes les excepcions i errors de Java hereten de la classe Throwable. Conèixer l'arbre és essencial per entendre quines excepcions cal capturar obligatòriament i quines no.

Throwable ├── Error ← problemes greus de la JVM (no cal capturar) │ ├── OutOfMemoryError │ └── StackOverflowError │ └── Exception ├── IOException ← checked: el compilador obliga a capturar │ └── FileNotFoundException │ └── RuntimeException ← unchecked: no cal capturar obligatòriament ├── ArithmeticException (divisió per zero) ├── NullPointerException (accés a referència null) ├── ArrayIndexOutOfBoundsException ├── NumberFormatException (parseInt("hola")) └── InputMismatchException (Scanner llegeix tipus incorrecte)
TipusHereten deCal capturar?Exemples típics
CheckedException (no Runtime)✅ Sí, el compilador ho exigeixIOException, FileNotFoundException
UncheckedRuntimeException⚠️ Opcional (però recomanable)ArithmeticException, NullPointerException, NumberFormatException
ErrorError❌ No (no recuperables)OutOfMemoryError, StackOverflowError

3. Capturar excepcions: try-catch-finally

3.1 Estructura bàsica
try { // Codi que pot generar una excepció int resultat = 10 / 0; } catch (ArithmeticException e) { // S'executa NOMÉS si es produeix l'excepció indicada System.out.println("Error: " + e.getMessage()); } finally { // S'executa SEMPRE, hi hagi excepció o no System.out.println("Bloc finally executat"); }
3.2 Flux d'execució
SituacióOrdre d'execució
Sense cap excepciótry complet → finally → continua
Excepció capturada pel catchtry fins al punt d'error → catch → finally → continua
Excepció NO capturadatry fins al punt d'error → finally → es propaga cap amunt
3.3 Múltiples catch

Quan un bloc try pot generar diversos tipus d'excepció, es pot afegir un catch per a cada tipus. Java els avalua en ordre i executa el primer que coincideixi. Posa sempre les excepcions més específiques primer.

import java.util.Scanner; import java.util.InputMismatchException; public class MultipleCatch { public static void main(String[] args) { Scanner sc = new Scanner(System.in); try { int num1 = sc.nextInt(); // possible InputMismatchException int num2 = Integer.parseInt("hola"); // possible NumberFormatException System.out.println(num2 / num1); // possible ArithmeticException } catch (ArithmeticException e) { System.out.println("Error aritmètic: " + e.getMessage()); } catch (InputMismatchException e) { System.out.println("Entrada no vàlida (esperava un enter)"); } catch (NumberFormatException e) { System.out.println("Format numèric incorrecte: " + e.getMessage()); } finally { sc.close(); System.out.println("Programa finalitzat correctament"); } } }
Mètodes útils de l'objecte excepció:
e.getMessage() → retorna un text descriptiu de l'error.
e.printStackTrace() → mostra per consola la traça completa d'on s'ha produït l'error. Molt útil per depurar.
3.4 Capturar amb la superclasse Exception

Si poses catch (Exception e) captures qualsevol excepció. És còmode però poc precís: no saps exactament quin error s'ha produït. Usa-ho com a últim recurs o per depurar.

try { int valor1 = 10, valor2 = 0; var resultat = valor1 / valor2; System.out.println("resultat = " + resultat); } catch (Exception e) { System.out.println("Ha ocorregut un error: " + e); }

4. Llançar excepcions: throw i throws

4.1 throw — llançar una excepció manualment

Podem llançar nosaltres mateixos una excepció quan detectem una situació d'error, sense esperar que Java la generi automàticament.

import java.util.InputMismatchException; public class ValidacioMinuts { public static void main(String[] args) { int minuts = 75; // valor invàlid if (minuts < 0 || minuts >= 60) { throw new InputMismatchException("Valor fora de rang: ha de ser entre 0 i 59"); } System.out.println("Minuts vàlids: " + minuts); } }
4.2 throws — declarar que un mètode pot propagar una excepció

Quan un mètode pot llançar una excepció checked i no la captura internament, ha de declarar-ho a la seva capçalera amb throws. Qui cridi al mètode haurà de capturar-la.

// El mètode declara que pot llançar ArithmeticException static void dividir(int n, int m) throws ArithmeticException { if (m == 0) throw new ArithmeticException("No es pot dividir per zero"); System.out.println(n + "/" + m + " = " + (n / m)); } public static void main(String[] args) { try { dividir(10, 2); // 10/2 = 5 → OK dividir(5, 0); // llança excepció } catch (ArithmeticException e) { System.out.println("Capturada al main: " + e.getMessage()); } }
throw vs throws:
throw (sense 's') llança un objecte excepció en un punt concret del codi.
throws (amb 's') declara a la capçalera d'un mètode que pot propagar aquell tipus d'excepció.

5. Excepcions personalitzades

Quan les excepcions predefinides de Java no representen prou bé el nostre error de negoci, podem crear les nostres pròpies. El procés és senzill: creem una classe que hereti de Exception (checked) o de RuntimeException (unchecked).

5.1 Estructura d'una excepció pròpia
// Definim la nostra excepció personalitzada (checked) public class EdatNoValidaException extends Exception { private int edat; public EdatNoValidaException(int edat) { this.edat = edat; } @Override public String toString() { return "Edat no vàlida: " + edat + ". Ha de ser entre 0 i 120."; } }
5.2 Ús de l'excepció personalitzada
public class Persona { private String nom; private int edat; public Persona(String nom, int edat) throws EdatNoValidaException { if (edat < 0 || edat > 120) throw new EdatNoValidaException(edat); this.nom = nom; this.edat = edat; } @Override public String toString() { return nom + " (" + edat + " anys)"; } } public class Main { public static void main(String[] args) { try { Persona p1 = new Persona("Anna", 25); // OK System.out.println(p1); Persona p2 = new Persona("Marc", -5); // llança EdatNoValidaException System.out.println(p2); } catch (EdatNoValidaException e) { System.out.println("Error: " + e); } } } // Sortida: // Anna (25 anys) // Error: Edat no vàlida: -5. Ha de ser entre 0 i 120.
Exception vs RuntimeException: Si la teva excepció hereta de Exception, qui la llanci haurà de declarar-la amb throws i qui la cridi haurà de capturar-la (és checked). Si hereta de RuntimeException, no és obligatori (és unchecked). Per a errors de lògica de negoci (edat invàlida, saldo insuficient…) és recomanable usar Exception per forçar la gestió explícita.

6. Exercicis

Ex. 1 Divisió segura Bàsic

Escriu un programa que demani dos enters per teclat i mostri el resultat de la seva divisió. Captura les excepcions que es puguin produir: divisió per zero i entrada no numèrica.

  1. Usa un bloc try-catch amb múltiples catch: un per ArithmeticException, un per InputMismatchException.
  2. Afegeix un bloc finally que mostri "Programa finalitzat".
  3. Prova el programa introduint: (a) dos enters vàlids, (b) divisor zero, (c) text en lloc d'un nombre.
Bàsic CA 6.10 — try-catch-finally
✅ Solució
import java.util.Scanner; import java.util.InputMismatchException; public class DivisioSegura { public static void main(String[] args) { Scanner sc = new Scanner(System.in); try { System.out.print("Introdueix el dividend: "); int a = sc.nextInt(); System.out.print("Introdueix el divisor: "); int b = sc.nextInt(); int resultat = a / b; System.out.println("Resultat: " + resultat); } catch (ArithmeticException e) { System.out.println("Error: no es pot dividir per zero."); } catch (InputMismatchException e) { System.out.println("Error: cal introduir un número enter."); } finally { sc.close(); System.out.println("Programa finalitzat."); } } }
Ex. 2 Accés segur a un array Bàsic

Declara un array de 5 enters. Demana a l'usuari una posició i mostra el valor. Captura les dues excepcions possibles: índex fora de rang i entrada no numèrica.

  1. Captura ArrayIndexOutOfBoundsException quan la posició no existeixi.
  2. Captura InputMismatchException quan l'usuari no introdueixi un enter.
  3. Usa un bucle do-while perquè el programa repeteixi la pregunta fins que l'entrada sigui vàlida.
Bàsic CA 6.10 — try-catch + excepcions d'array
✅ Solució
import java.util.Scanner; import java.util.InputMismatchException; public class AccessSegurArray { public static void main(String[] args) { int[] dades = { 10, 20, 30, 40, 50 }; Scanner sc = new Scanner(System.in); boolean valid = false; do { System.out.print("Introdueix una posició (0-4): "); try { int pos = sc.nextInt(); System.out.println("Valor a la posició " + pos + ": " + dades[pos]); valid = true; } catch (ArrayIndexOutOfBoundsException e) { System.out.println("Error: posició fora de rang (0-4)."); } catch (InputMismatchException e) { System.out.println("Error: cal introduir un número enter."); sc.next(); // buidem el buffer del Scanner } } while (!valid); sc.close(); } }

  1. Crea la classe EdatNoValidaException que hereti de Exception. Guarda l'edat invàlida com a atribut i sobreescriu toString() per mostrar un missatge clar.
  2. Modifica el constructor de Persona per llançar EdatNoValidaException si l'edat no és vàlida.
  3. Al main, crea diverses persones (algunes vàlides, algunes no) i captura l'excepció.

Ex. 4 Gestió d'alumnes amb excepcions de validació Avançat

Crea una classe Alumne amb els atributs nom, cognoms, nif i nota. Implementa una excepció personalitzada AlumneExcepcio que es llanci en els casos indicats.

  1. La nota ha d'estar entre 0 i 10. Si no, llença AlumneExcepcio.
  2. El nom i els cognoms han de començar amb majúscula (usa Character.isUpperCase(nom.charAt(0))). Si no, llença AlumneExcepcio.
  3. Crea un main que intenti crear diversos alumnes, alguns amb dades correctes i altres amb dades incorrectes, i mostri un missatge clar per a cada cas.
Pista: Pots tenir un únic catch (AlumneExcepcio e) si el missatge d'error ja descriu bé el problema. O bé pots fer que AlumneExcepcio guardi el camp que ha fallat com a atribut.
Avançat CA 6.10 — Excepcions personalitzades + validació de dades
✅ Solució
// Fitxer: AlumneExcepcio.java public class AlumneExcepcio extends Exception { private String camp; private String motiu; public AlumneExcepcio(String camp, String motiu) { this.camp = camp; this.motiu = motiu; } @Override public String toString() { return "Error al camp '" + camp + "': " + motiu; } } // Fitxer: Alumne.java public class Alumne { private String nom; private String cognoms; private String nif; private double nota; public Alumne(String nom, String cognoms, String nif, double nota) throws AlumneExcepcio { if (!Character.isUpperCase(nom.charAt(0))) throw new AlumneExcepcio("nom", "la primera lletra ha de ser majúscula"); if (!Character.isUpperCase(cognoms.charAt(0))) throw new AlumneExcepcio("cognoms", "la primera lletra ha de ser majúscula"); if (nota < 0 || nota > 10) throw new AlumneExcepcio("nota", "ha d'estar entre 0 i 10"); this.nom = nom; this.cognoms = cognoms; this.nif = nif; this.nota = nota; } @Override public String toString() { return nom + " " + cognoms + " | NIF: " + nif + " | Nota: " + nota; } } // Fitxer: Main.java public class Main { public static void main(String[] args) { String[][] dades = { { "Anna", "Puig", "12345678A", "8.5" }, { "marc", "Serra", "87654321B", "7.0" }, // nom en minúscula { "Laia", "Roca", "11223344C", "11.0" }, // nota > 10 { "Jordi", "Martí", "44332211D", "9.0" } }; for (String[] d : dades) { try { Alumne a = new Alumne(d[0], d[1], d[2], Double.parseDouble(d[3])); System.out.println("Creat: " + a); } catch (AlumneExcepcio e) { System.out.println("No creat (" + d[0] + "): " + e); } } } }
