1. Expressions regulars

Una expressió regular (o regex) és una seqüència de caràcters que descriu un patró de text. En lloc de cercar un text concret, una regex descriu l'estructura que ha de tenir el text: "exactament 5 dígits", "un email vàlid", "un telèfon espanyol"...

A Java les expressions regulars s'utilitzen principalment per a tres coses: validar que una cadena té un format concret, cercar fragments que compleixen un patró dins d'un text llarg, i substituir parts d'una cadena.

Quan usar regex i quan no:
✅ Usa regex quan el format és complex o té múltiples regles (email, telèfon, codi postal, matrícula).
❌ No cal regex quan n'hi ha prou amb contains(), startsWith(), endsWith() o equals().

2. Sintaxi bàsica

2.1 Metacaracters i classes de caràcters
PatróConcorda ambExemple
.Qualsevol caràcter (excepte salt de línia)"c.t" → "cat", "cut", "c3t"
\dQualsevol dígit (0–9)"\\d" → "5"
\wLletra, dígit o guió baix [a-zA-Z_0-9]"\\w+" → "hola_123"
\sEspai en blanc (espai, tab, salt de línia)"\\s" → " "
\D, \W, \SNegació de \d, \w, \s"\\D" → "a", "!", " "
[abc]Un dels caràcters indicats"[aeiou]" → "a"
[a-z]Qualsevol lletra minúscula"[a-z]" → "m"
[^abc]Qualsevol caràcter que NO sigui a, b o c"[^0-9]" → "x"

2.2 Quantificadors
QuantificadorSignificatExemple
*Zero o més vegades"\\d*" → "", "1", "123"
+Una o més vegades"\\d+" → "1", "123" (no "")
?Zero o una vegada (opcional)"colou?r" → "color" o "colour"
{n}Exactament n vegades"\\d{5}" → "08080"
{n,}Almenys n vegades"\\d{2,}" → "12", "1234"
{n,m}Entre n i m vegades"\\w{3,15}" → entre 3 i 15 caràcters de paraula
Doble barra invertida a Java: A les cadenes Java, \ és el caràcter d'escapament. Per tant, per escriure \d en una regex has d'escriure "\\d". Per escriure un \ literal has d'escriure "\\\\".
2.3 Combinació de patrons — exemples pràctics
// Exactament 5 dígits (codi postal simple) "08080".matches("\\d{5}") // true "1234".matches("\\d{5}") // false (4 dígits) // 3 lletres minúscules seguides de 2 dígits "abc12".matches("[a-z]{3}\\d{2}") // true "ABC12".matches("[a-z]{3}\\d{2}") // false (majúscules) // Nom d'usuari: comença per lletra, 3-15 caràcters [a-zA-Z0-9_] "Usuari123".matches("[a-zA-Z]\\w{2,14}") // true "99User".matches("[a-zA-Z]\\w{2,14}") // false (comença per dígit) // Substituir tots els espais per guions String resultat = "hola món".replaceAll("\\s", "-"); // "hola-món" // Dividir per qualsevol quantitat d'espais en blanc String[] parts = "a b c".split("\\s+"); // ["a", "b", "c"]

3. Les classes Pattern i Matcher

Els mètodes de String (matches, replaceAll, split) serveixen quan volem comprovar tota la cadena o fer substitucions simples. Però quan volem cercar totes les coincidències dins d'un text llarg, necessitem les classes Pattern i Matcher.

Mètode de StringClasses Pattern / Matcher
Comprova tota la cadenaCerca parts del text
Una sola comprovació ràpidaItera sobre totes les coincidències
"08080".matches("\\d{5}")while (matcher.find()) { ... }
3.1 Estructura bàsica
import java.util.regex.Pattern; import java.util.regex.Matcher; // 1. Compilar el patró (reutilitzable) Pattern patro = Pattern.compile("\\d+"); // 2. Aplicar-lo sobre un text String text = "El curs 2024 té 198 hores i 9 RAs"; Matcher m = patro.matcher(text); // 3. Iterar sobre totes les coincidències while (m.find()) { System.out.println("Trobat: " + m.group()); } // Sortida: // Trobat: 2024 // Trobat: 198 // Trobat: 9
3.2 matches() vs find()
Pattern p = Pattern.compile("\\d{4}"); // matches(): comprova si TOTA la cadena compleix el patró p.matcher("2024").matches() // true p.matcher("l'any 2024").matches() // false (hi ha més text) // find(): cerca si el patró apareix EN ALGUN PUNT del text p.matcher("l'any 2024").find() // true (troba "2024")
3.3 Exemple complet: extracció de dates

El patró per a dates en format DD/MM/AAAA és un bon exemple de combinació de classes, rangs i quantificadors. El dia pot ser 01–31, el mes 01–12 i l'any 4 dígits:

import java.util.regex.*; public class ExtraccioDates { public static void main(String[] args) { String text = """ Avui, 15/09/2023, s'ha fet una reunió. La propera sessió serà el 01/10/2023, just abans del congrés del 05/11/2023. L'empresa va ser fundada el 08/03/1999. """; // DD: 01-09, 10-29, 30-31 / MM: 01-09, 10-12 / AAAA: 4 dígits String regex = "(0[1-9]|[12][0-9]|3[01])/(0[1-9]|1[0-2])/\\d{4}"; Pattern patro = Pattern.compile(regex); Matcher matcher = patro.matcher(text); System.out.println("Dates trobades:"); while (matcher.find()) { System.out.println(" " + matcher.group()); } } } // Sortida: // Dates trobades: // 15/09/2023 // 01/10/2023 // 05/11/2023 // 08/03/1999

4. JSON i Java

4.1 Què és JSON?

JSON (JavaScript Object Notation) és el format d'intercanvi de dades més usat avui dia. El trobaràs a tot arreu: respostes d'APIs web, fitxers de configuració, bases de dades document... Aprendre a llegir-lo i generar-lo és una habilitat fonamental per a qualsevol desenvolupador web.

L'estructura és senzilla: parells clau: valor entre claus, arrays entre claudàtors. Els valors poden ser cadenes, números, booleans, null, objectes o arrays.

{ "nom": "Anna Puig", "edat": 22, "activa": true, "email": "anna@example.com", "notes": [7.5, 8.0, 9.5], "adreca": { "carrer": "Gran Via", "codi_postal": "08080" } }
4.2 La llibreria Gson

Java no inclou suport natiu per a JSON. La llibreria més popular per processar JSON és Gson (de Google). Permet convertir objectes Java a JSON (serialització) i JSON a objectes Java (deserialització) de forma molt senzilla.

Afegir Gson al projecte (Maven): Inclou aquesta dependència al teu pom.xml:

<dependency>
  <groupId>com.google.code.gson</groupId>
  <artifactId>gson</artifactId>
  <version>2.10.1</version>
</dependency>

O descarrega el .jar directament de mvnrepository.com i afegeix-lo al classpath del projecte.
4.3 Serialització: Java → JSON
import com.google.gson.Gson; import com.google.gson.GsonBuilder; public class Alumne { private String nom; private int edat; private double nota; public Alumne(String nom, int edat, double nota) { this.nom = nom; this.edat = edat; this.nota = nota; } } public class Main { public static void main(String[] args) { Alumne a = new Alumne("Anna", 22, 8.5); // Gson bàsic Gson gson = new Gson(); String json = gson.toJson(a); System.out.println(json); // {"nom":"Anna","edat":22,"nota":8.5} // GsonBuilder: JSON formatat (pretty print) Gson gsonBonic = new GsonBuilder().setPrettyPrinting().create(); System.out.println(gsonBonic.toJson(a)); // { // "nom": "Anna", // "edat": 22, // "nota": 8.5 // } } }
4.4 Deserialització: JSON → Java
import com.google.gson.Gson; import com.google.gson.reflect.TypeToken; import java.util.List; import java.lang.reflect.Type; public class Main { public static void main(String[] args) { Gson gson = new Gson(); // JSON d'un objecte → objecte Java String jsonAlumne = "{\"nom\":\"Marc\",\"edat\":19,\"nota\":7.5}"; Alumne a = gson.fromJson(jsonAlumne, Alumne.class); System.out.println(a.getNom() + " — " + a.getNota()); // Marc — 7.5 // JSON d'un array → List Java String jsonArray = "[{\"nom\":\"Anna\",\"edat\":22,\"nota\":8.5}," + "{\"nom\":\"Laia\",\"edat\":20,\"nota\":9.0}]"; Type tipusLlista = new TypeToken<List<Alumne>>(){}.getType(); List<Alumne> alumnes = gson.fromJson(jsonArray, tipusLlista); alumnes.forEach(al -> System.out.println(al.getNom() + ": " + al.getNota())); // Anna: 8.5 // Laia: 9.0 } }
4.5 Llegir i escriure fitxers JSON
import com.google.gson.*; import java.io.*; import java.util.*; public class FitxerJSON { public static void main(String[] args) throws IOException { Gson gson = new GsonBuilder().setPrettyPrinting().create(); List<Alumne> alumnes = List.of( new Alumne("Anna", 22, 8.5), new Alumne("Marc", 19, 7.5) ); // Escriure al fitxer try (FileWriter fw = new FileWriter("alumnes.json")) { gson.toJson(alumnes, fw); } System.out.println("Fitxer escrit!"); // Llegir del fitxer try (FileReader fr = new FileReader("alumnes.json")) { Type tipus = new TypeToken<List<Alumne>>(){}.getType(); List<Alumne> llegits = gson.fromJson(fr, tipus); llegits.forEach(a -> System.out.println(a.getNom())); } } }

5. Exercicis — Expressions regulars

Ex. 1 Validació de matrícula espanyola Bàsic

Crea un programa que validi si una cadena correspon a una matrícula espanyola moderna: exactament 4 dígits seguits de 3 lletres majúscules, sense espais.

  1. Vàlids: 1234ABC, 0001BCD, 9876XYZ
  2. No vàlids: 123ABC, 12345ABC, 1234abc, 1234 ABC
  3. Demana la matrícula per teclat i mostra si és vàlida o no.
Bàsic CA 6.5 — matches(), quantificadors, classes de caràcters
✅ Solució
import java.util.Scanner; public class ValidacioMatricula { public static void main(String[] args) { Scanner sc = new Scanner(System.in); System.out.print("Introdueix una matrícula: "); String matricula = sc.nextLine().trim(); sc.close(); String regex = "\\d{4}[A-Z]{3}"; boolean valid = matricula.matches(regex); System.out.println(matricula + " → " + (valid ? "VÀLIDA ✓" : "NO VÀLIDA ✗")); } }
Ex. 2 Validació de codi postal espanyol Bàsic

Valida codis postals espanyols: 5 dígits, els dos primers entre 01 i 52 (les províncies espanyoles).

  1. Vàlids: 08080, 29012, 47001, 01001
  2. No vàlids: 99999 (fora de rang), 1234 (4 dígits), A2890 (lletra), 00001 (comença per 00)
  3. La regex ha de controlar que els dos primers dígits siguin 01–52.
Bàsic CA 6.5 — matches(), alternança |, rangs
✅ Solució
import java.util.Scanner; public class ValidacioCodiPostal { public static void main(String[] args) { Scanner sc = new Scanner(System.in); System.out.print("Introdueix un codi postal: "); String codi = sc.nextLine().trim(); sc.close(); // 0[1-9] = 01..09 | [1-4][0-9] = 10..49 | 5[0-2] = 50..52 String regex = "(0[1-9]|[1-4][0-9]|5[0-2])\\d{3}"; boolean valid = codi.matches(regex); System.out.println("Codi postal vàlid? " + valid); } }
Ex. 3 Validació de nom d'usuari Mitjà

Un nom d'usuari vàlid ha de: (a) Començar per una lletra. (b) Tenir entre 3 i 15 caràcters. (c) Contenir només lletres, dígits i guió baix.

  1. Vàlids: Usuari123, Xavi_99, abc
  2. No vàlids: 99User (comença per dígit), U (massa curt), abc@123 (caràcter no permès)
  3. Prova el teu programa amb almenys 5 casos, incloent-ne de vàlids i no vàlids.
Mitjà CA 6.5 — matches(), \w, quantificadors {n,m}
✅ Solució
public class ValidacioUsuari { public static void main(String[] args) { String[] casos = { "Usuari123", "Xavi_99", "abc", "99User", "U", "abc@123", "nom_molt_llarg_que_no_cap" }; // [a-zA-Z] = comença per lletra // \\w{2,14} = 2..14 caràcters més de paraula (total 3..15) String regex = "[a-zA-Z]\\w{2,14}"; for (String usuari : casos) { boolean valid = usuari.matches(regex); System.out.printf("%-30s → %s%n", usuari, valid ? "VÀLID ✓" : "NO VÀLID ✗"); } } }
Ex. 4 Extracció de dates d'un text Mitjà

Donat el text de l'exercici del Drive, cerca totes les dates en format DD/MM/AAAA i mostra-les per pantalla. Usa Pattern i Matcher amb find().

  1. El patró ha de validar correctament el dia (01–31) i el mes (01–12).
  2. Copia literalment el text del Drive al codi com a String text block.
  3. Mostra quantes dates s'han trobat i llista-les totes.
Mitjà CA 6.5 — Pattern, Matcher, find(), group()
✅ Solució
import java.util.regex.*; import java.util.*; public class ExtraccioDates { public static void main(String[] args) { String text = """ Avui, 15/09/2023, s'ha fet una reunió molt important. Ens han informat que la següent sessió serà el 01/10/2023, just abans del congrés del 05/11/2023. Recordeu que l'any passat, el 25/12/2022, vam tenir un esdeveniment especial. Les dates de l'examen final: - Primera convocatòria: 10/06/2024 - Segona convocatòria: 20/07/2024 L'empresa va ser fundada el 08/03/1999. El festival d'estiu, del 01/07/2025 al 20/08/2025. Última actualització: 30/01/2024. """; String regex = "(0[1-9]|[12][0-9]|3[01])/(0[1-9]|1[0-2])/\\d{4}"; Pattern patro = Pattern.compile(regex); Matcher matcher = patro.matcher(text); List<String> dates = new ArrayList<>(); while (matcher.find()) { dates.add(matcher.group()); } System.out.println("Dates trobades: " + dates.size()); dates.forEach(System.out::println); } }

6. Exercicis — JSON amb Gson

Ex. 5 Serialitzar i deserialitzar un alumne Bàsic

Crea la classe Alumne amb atributs nom, edat i nota. Serialitza un objecte a JSON i deserialitza el JSON resultant per verificar que les dades coincideixen.

  1. Crea un Alumne, converteix-lo a JSON amb Gson i mostra la cadena JSON.
  2. Converteix la cadena JSON de nou a objecte Alumne i mostra els atributs per comprovar que tot és correcte.
  3. Usa GsonBuilder().setPrettyPrinting() per formatar el JSON.
Bàsic CA 6.8 · CA 6.9 — Gson toJson / fromJson
✅ Solució
import com.google.gson.*; class Alumne { private String nom; private int edat; private double nota; public Alumne(String nom, int edat, double nota) { this.nom = nom; this.edat = edat; this.nota = nota; } public String getNom() { return nom; } public int getEdat() { return edat; } public double getNota() { return nota; } } public class Main { public static void main(String[] args) { Gson gson = new GsonBuilder().setPrettyPrinting().create(); // Serialització Alumne original = new Alumne("Anna", 22, 8.5); String json = gson.toJson(original); System.out.println("JSON generat:\n" + json); // Deserialització Alumne recuperat = gson.fromJson(json, Alumne.class); System.out.println("\nAlumne recuperat:"); System.out.println(" Nom: " + recuperat.getNom()); System.out.println(" Edat: " + recuperat.getEdat()); System.out.println(" Nota: " + recuperat.getNota()); } }
Ex. 6 Gestió d'alumnes amb fitxer JSON Avançat

Crea un programa que gestioni una llista d'alumnes persistent en un fitxer JSON. El programa ha de poder guardar i recuperar la llista entre execucions.

  1. En iniciar, llegeix el fitxer alumnes.json si existeix i carrega la llista. Si no existeix, comença amb una llista buida.
  2. Permet afegir nous alumnes des de teclat (nom, edat, nota). Valida que el nom no estigui buit i la nota entre 0 i 10.
  3. Mostra la llista completa d'alumnes amb la seva posició i nota.
  4. En sortir del programa, guarda la llista actualitzada al fitxer JSON.
Avançat CA 6.8 · CA 6.9 — Gson + FileReader/FileWriter + col·leccions
✅ Solució
import com.google.gson.*; import com.google.gson.reflect.TypeToken; import java.io.*; import java.lang.reflect.Type; import java.util.*; public class GestioAlumnesJSON { static final String FITXER = "alumnes.json"; static Gson gson = new GsonBuilder().setPrettyPrinting().create(); static List<Alumne> carregarAlumnes() { File f = new File(FITXER); if (!f.exists()) return new ArrayList<>(); try (FileReader fr = new FileReader(f)) { Type tipus = new TypeToken<List<Alumne>>(){}.getType(); List<Alumne> llista = gson.fromJson(fr, tipus); return llista != null ? llista : new ArrayList<>(); } catch (IOException e) { System.out.println("Error llegint fitxer: " + e.getMessage()); return new ArrayList<>(); } } static void guardarAlumnes(List<Alumne> alumnes) { try (FileWriter fw = new FileWriter(FITXER)) { gson.toJson(alumnes, fw); System.out.println("Dades guardades a " + FITXER); } catch (IOException e) { System.out.println("Error escrivint fitxer: " + e.getMessage()); } } public static void main(String[] args) { List<Alumne> alumnes = carregarAlumnes(); Scanner sc = new Scanner(System.in); boolean continuar = true; while (continuar) { System.out.println("\n1. Afegir alumne 2. Mostrar llista 3. Sortir"); System.out.print("Opció: "); int opcio = sc.nextInt(); sc.nextLine(); switch (opcio) { case 1: System.out.print("Nom: "); String nom = sc.nextLine().trim(); System.out.print("Edat: "); int edat = sc.nextInt(); System.out.print("Nota (0-10): "); double nota = sc.nextDouble(); sc.nextLine(); if (!nom.isEmpty() && nota >= 0 && nota <= 10) { alumnes.add(new Alumne(nom, edat, nota)); System.out.println("Alumne afegit."); } else { System.out.println("Dades no vàlides."); } break; case 2: if (alumnes.isEmpty()) { System.out.println("Llista buida."); break; } for (int i = 0; i < alumnes.size(); i++) { Alumne a = alumnes.get(i); System.out.printf("%d. %s (%d anys) — %.1f%n", i+1, a.getNom(), a.getEdat(), a.getNota()); } break; case 3: continuar = false; break; } } guardarAlumnes(alumnes); sc.close(); } }
